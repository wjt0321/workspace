---

## 1. 问题陈述

### 1.1 痛点

| 痛点 | 旧体系 | 后果 |
|:---|:---|:---|
| 单点故障 | 470 行 MEMORY.md 是唯一记忆载体 | 文件损坏 → 全盘失忆 |
| 人工维护 | 每次大事后手动写 MEMORY.md | 遗漏、延迟、格式不统一 |
| 信息膨胀 | 无分层，铁律与流水账混在一起 | 470 行找一条铁律如大海捞针 |
| 不可回溯 | 删除旧内容后无法查历史 | 决策依据丢失 |

### 1.2 目标

1. **双副本容灾**：MEMORY.md + memory/*.md 互为备份，一个挂另一个兜底
2. **人工 + 自动双轨**：人工编排铁律区，自动发现新话题并增量追加
3. **分层检索**：铁律全文 → 重要指针 → 一般留日志，三层递减
4. **安全围栏保护**：任何清理操作不得触碰记忆文件

### 2.2 三层分类逻辑

| 层级 | 判定标准 | 存储位置 | 示例 |
|:---|:---|:---|:---|
| 🔒 **铁律** | 不可逆规则/安全约束/关键路径 | MEMORY.md 铁律区（全文） | "V2Ray 前置检查：任何 GitHub 操作前必查 v2rayN.exe" |
| 📌 **重要** | 重大事件/技术发现/经验教训 | MEMORY.md 指针（→ daily） | "BW 自建服务器完全打通 → memory/2026-05-07.md" |
| 一般 | 日报/状态/一次性记录 | 仅存 memory/*.md | "今天清理了 32 个临时文件" |

**检索策略**：`memory_search` 语义检索全量（MEMORY.md + memory/*.md），按相关性排序返回。

## 3. 自动蒸馏管道

### 3.1 流程图

```
cleanup-deep.py（每15天 12:00 cron）
│
├─ 第①步：distill-memory.py --apply
│   │
│   ├─ ① 提取 MEMORY.md 已知话题（**加粗**关键词 + ### 标题）
│   │     └─ 例：{"Obsidian 完全接管", "BW 四大坑", "V2Ray 前置检查", ...}
│   │
│   ├─ ② 扫描最近 15 天 memory/*.md 的 ## 标题
│   │     └─ 跳过每日模板标题（"🧠 持久化记忆"、"💡 反思与逻辑"等）
│   │
│   ├─ ③ 差集 = ## 标题 - 已知话题
│   │     └─ 例：标题"P0 优化：Skills 分类"不在已知话题中 → 新发现
│   │
│   ├─ ④ 自动追加到 MEMORY.md
│   │     └─ ## 📌 重要记忆 → ### 🔄 自动新发现 (2026-05-07)
│   │
│   └─ ⑤ 同步速查表（jobs.json → 定时任务表, tools/ → 工具清单）
│
├─ 第②步：dialog/sessions 归档（>30天旧文件）
├─ 第③步：通用清理（tool_results/ 截图 临时缓存）
└─ 第④步：工作区结构审查
```

### 3.2 差集检测细节

```
已知话题 = extract(MEMORY.md)
  ├─ **加粗关键词**（指针行）：如 "Obsidian 完全接管"、"BW 四大坑"
  └─ ### 小标题：如 "代理 & 网络"、"Bitwarden"

每日标题 = scan(memory/*.md)
  ├─ 提取所有 ## 标题
  └─ 过滤模板标题：🧠持久化记忆、💡反思与逻辑、📅日期信息、变更记录…

新发现 = 每日标题 - 已知话题
  └─ 若空 → "✅ 无新发现。MEMORY.md 已覆盖 daily notes 全部话题。"
  └─ 若非空 → 追加到 ### 🔄 自动新发现 (日期)
```

### 3.3 安全机制

| 机制 | 说明 |
|:---|:---|
| **写入前备份** | 每次 `--apply` 前自动备份 `MEMORY.md.bak` |
| **只追加不覆盖** | 人工编排的铁律区和指针区不受影响 |
| **cleanup_shared.py 保护** | `memory/` 和 `MEMORY.md` 在 11 个保护目录中 |
| **`.gitignore` 双重防线** | `.bw-auth.json` `.bw-session.json` `.env` 同时受 gitignore 和 cleanup 白名单保护 |

## 6. 异常处理

| 场景 | 处理 |
|:---|:---|
| distill-memory.py 超时（>120s） | 跳过蒸馏，仍执行后续清理 |
| distill-memory.py 异常退出 | 打印 stderr 前三行，继续清理 |
| MEMORY.md 不存在 | 所有操作跳过（内存安全） |
| jobs.json 不可读 | 跳过定时任务表同步，不影响蒸馏 |
| `memory/` 目录空 | 蒸馏输出"0 个未收录标题" |
