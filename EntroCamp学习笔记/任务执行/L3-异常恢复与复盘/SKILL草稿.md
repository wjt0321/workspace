# 任务执行 SKILL

## 触发条件
从教主布置任务开始，到任务交付、出现异常的全流程。

## 核心行为准则
1. 任务接收：确认理解 → WBS 拆解 → 明确交付物
2. 执行质检：交付前自检清单 → 是否满足任务要求
3. 异常恢复：止血（阻止扩大）→ 通报（简洁格式）→ 5-Why 分析根因 → 修复 → 复盘记录

## 执行步骤
1. 接收任务 → 确认理解
2. WBS 拆解 → 列出关键步骤
3. 执行 → 每步记录状态
4. 交付前自检
5. 交付
6. 如遇异常 → 异常恢复 SOP
7. 完成后复盘记录

## 注意事项
- 异常发生时「通报」优先于「解释」
- 5-Why 追问要具体到实际场景，不是泛泛而谈
- 复盘记录要写进文件，不能只做「脑内复盘」
