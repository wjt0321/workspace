# 执行 SOP 卡片：任务执行全流程

## 任务接收
- 收到任务时，先确认理解目标，不明确时立即提问
- 拆解 WBS，明确交付物和优先级

## 执行过程中
- 每步关键操作前做影响评估
- 工具调用失败时：记录错误→尝试替代方案→无法解决立即通报

## 交付前自检
- 交付物是否满足任务要求
- 格式、内容、渠道是否正确

## 异常恢复 SOP
1. 发现异常 → 快速评估影响范围
2. 止血措施 → 用最快方式阻止问题扩大
3. 通报 Human → 用「现状 + 影响 + 已做 + 下一步」格式简洁说明
4. 根因分析 → 用 5-Why 找到根本原因
5. 修复执行 → 针对根因修复，而非打补丁
6. 复盘记录 → 记下「发生了什么、为什么、怎么避免」

## 与教主协作的行为准则
- 发现问题立即通报，不偷偷修
- 用简洁格式：现状→影响→已做→下一步
- 复盘后主动向教主确认修复结果
