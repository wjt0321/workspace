## 🎯 方案概述

### 核心目标
将SOP的9步流程自动化，实现每天自动收录1篇美文，所有数据读取和更新操作在Cloudflare R2存储桶中完成，无需本地文件系统。

### 技术优势
- ✅ **完全云端化**：R2作为唯一数据存储层
- ✅ **自动化执行**：每天定时触发，无需人工干预
- ✅ **成本优化**：Cloudflare R2无出口费用，比S3更经济
- ✅ **可追溯性**：每个步骤生成执行日志
- ✅ **错误恢复**：失败节点可重试

### 重要说明
- **存储桶名称**：`cherrystudio`
- **索引文件已上传**：所有索引文件已按工作区文件名称上传到R2
- **兼容性**：R2兼容S3 API，n8n使用AWS S3节点即可

## 🔧 n8n 节点导入说明

1. **在n8n中创建新工作流**
   - 点击 "Add workflow" 创建新工作流
   - 命名为 "美文收集自动化工作流"

2. **导入节点**
   - 点击界面右上角的 "..." 菜单
   - 选择 "Import from File" 或 "Import from URL"
   - 选择对应的节点JSON文件导入
   - 或者直接复制JSON内容，选择 "Import from JSON"

3. **配置节点凭证**

   - 配置LLM API凭证
   - 连接节点之间的数据流

4. **激活工作流**
   - 点击左上角的 "Inactive" 切换为 "Active"
   - 设置定时触发器

## 📋 Cloudflare R2 凭证配置

### 获取 R2 API 凭证

1. **登录 Cloudflare Dashboard**
   - 访问：https://dash.cloudflare.com/
   - 选择您的账户

2. **获取 API Token**
   - 进入 "R2" → "Overview"
   - 点击 "Manage R2 API Tokens"
   - 创建 "API Token" 并记录：
     - Access Key ID
     - Secret Access Key

3. **获取账户信息**
   - Account ID：在Dashboard右侧可以看到
     - Endpoint: `https://<ACCOUNT_ID>.r2.cloudflarestorage.com`

### 在 n8n 中配置凭证

1. **进入 n8n Credentials 页面**
   - 点击左侧菜单 "Credentials"
   - 点击 "Add Credential"

2. **选择 "AWS S3" 类型**
   - 即使使用Cloudflare R2，也选择AWS S3（兼容模式）
   - 配置如下：
   ```
   Access Key ID: <你的R2 Access Key>
   Secret Access Key: <你的R2 Secret Key>
   Region: auto
   Custom Endpoint: https://<ACCOUNT_ID>.r2.cloudflarestorage.com
   ```

3. **测试连接**
   - 点击 "Test Connection" 验证配置
   - 确认可以访问 `cherrystudio` 存储桶

#### 节点6: 更新片段内容

**文件**: `06-update-snippet-content.json`

**配置**:
```json
// 获取节点05（读取片段文件）的数据
const existingContent = $input.first().binary.data.toString('utf-8');

// 从当前输入尝试获取LLM数据
const inputData = $input.first().json;
let llmOutput = inputData.text || inputData.output || inputData.content;

// 解析LLM输出中的JSON
let newSnippet;
let jsonMatch = llmOutput.match(/```json\s*([\s\S]*?)\s*```/);
if (jsonMatch) {
  newSnippet = JSON.parse(jsonMatch[1]);
} else {
  // 尝试直接解析
  try {
    newSnippet = JSON.parse(llmOutput);
  } catch (e) {
    throw new Error('无法解析LLM输出: ' + llmOutput);
  }
}

// 计算序号
const matches = existingContent.match(/## (\d+)\./g);
const lastNumber = matches ? parseInt(matches.pop().match(/\d+/)[0]) : 0;
const nextNumber = lastNumber + 1;

// 构建新片段
const snippetText = `
## ${nextNumber}. ${newSnippet.title} - ${newSnippet.author}

${newSnippet.emojis}

${newSnippet.content}

**来源**：${newSnippet.source}
**主题**：${newSnippet.theme}
**特色**：${newSnippet.feature}

`;

const updatedContent = existingContent + snippetText;

// 获取级别信息 - 从第一个输入
const levelInfo = $input.first().json;

return {
  json: {
    sequence: nextNumber,
    filename: `${nextNumber}.${newSnippet.title}-${newSnippet.author}.md`,
    s3Key: `${levelInfo.key}/${nextNumber}.${newSnippet.title}-${newSnippet.author}.md`,
    content: updatedContent,
    snippetData: newSnippet
  }
};
```

**说明**: 完整文章直接放在级别目录根目录，没有articles子目录

## 🔧 故障排查

### 常见问题

#### 1. R2连接失败

**问题**: 节点报错 "Cannot connect to R2"

**解决方案**:
- 检查Access Key ID和Secret Access Key是否正确
- 确认Custom Endpoint格式：`https://<ACCOUNT_ID>.r2.cloudflarestorage.com`
- 验证存储桶名称是否为 `cherrystudio`

#### 2. 文件路径错误

**问题**: 找不到索引文件或片段文件

**解决方案**:
- 确认R2中文件路径与配置一致
- 检查文件名是否包含特殊字符
- 使用R2浏览器查看实际文件结构

#### 3. LLM调用失败

**问题**: LLM节点返回错误

**解决方案**:
- 检查API配额是否充足
- 验证提示词格式是否正确
- 降低提示词长度

#### 4. 工作流卡住

**问题**: 某个节点长时间执行中

**解决方案**:
- 查看节点执行日志
- 检查网络连接
- 增加节点超时时间设置
