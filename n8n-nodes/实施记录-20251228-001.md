### 节点04A: 提取片段数据

**节点类型**: Code
**状态**: ✅ 成功

**完整代码**:
```javascript
// 获取LLM返回的文本
const llmOutput = $input.first().json;
const llmText = llmOutput.output || llmOutput.text || JSON.stringify(llmOutput);

// 提取JSON部分（去除```json标记）
let jsonText = llmText;
const jsonMatch = llmText.match(/```json\s*([\s\S]*?)\s*```/) || llmText.match(/\{[\s\S]*\}/);
if (jsonMatch) {
  jsonText = jsonMatch[1] || jsonMatch[0];
}

// 解析JSON
let parsedData;
try {
  parsedData = JSON.parse(jsonText);
} catch (e) {
  throw new Error('JSON解析失败: ' + e.message);
}

// 只返回解析出的片段数据
return {
  json: {
    sequence: parsedData.sequence || 1,
    title: parsedData.title,
    author: parsedData.author,
    emojis: parsedData.emojis,
    content: parsedData.content,
    source: parsedData.source,
    theme: parsedData.theme,
    feature: parsedData.feature
  }
};
```

**作用**: 从LLM返回的markdown代码块中提取JSON数据

### 节点06: 更新片段内容

**节点类型**: Code
**状态**: ✅ 成功（输出JSON，但无binary字段）

**完整代码**:
```javascript
// 获取节点05传入的文件内容
const fileInput = $input.first();
const existingContent = fileInput.binary.data.toString('utf-8');

// 获取节点04C传入的完整数据
const data = fileInput.json;

// 从数据中提取片段信息
const snippetData = {
  title: data.title,
  author: data.author,
  emojis: data.emojis,
  content: data.content,
  source: data.source,
  theme: data.theme,
  feature: data.feature
};

// 从数据中提取级别信息
const levelInfo = {
  key: data.key,
  snippetFile: data.snippetFile,
  indexFile: data.indexFile,
  templateFile: data.templateFile
};

// 计算下一个序号
const matches = existingContent.match(/## (\d+)\./g);
const lastNumber = matches ? parseInt(matches.pop().match(/\d+/)[0]) : 0;
const nextNumber = lastNumber + 1;

// 构建新片段文本
const snippetText = `## ${nextNumber}. ${snippetData.title} - ${snippetData.author}

${snippetData.emojis}

${snippetData.content}

**来源**：${snippetData.source}
**主题**：${snippetData.theme}
**特色**：${snippetData.feature}

`;

// 拼接完整内容
const updatedContent = existingContent + snippetText;

// 生成文件路径
const s3Key = `${levelInfo.key}/${nextNumber}.${snippetData.title}-${snippetData.author}.md`;

// 返回结果
return {
  json: {
    sequence: nextNumber,
    filename: `${nextNumber}.${snippetData.title}-${snippetData.author}.md`,
    s3Key: s3Key,
    snippetFile: levelInfo.snippetFile,
    indexFile: levelInfo.indexFile,
    templateFile: levelInfo.templateFile,
    key: levelInfo.key,
    updatedContent: updatedContent,
    snippetData: snippetData
  }
};
```

**输出示例**:
```json
{
  "sequence": 1,
  "filename": "1.春-朱自清.md",
  "s3Key": "初中生美文/1.春-朱自清.md",
  "snippetFile": "初中生美文/初中生美文精选100篇.md",
  "indexFile": "初中生美文/美文收集索引.md",
  "updatedContent": "完整更新后的文件内容..."
}
```

**问题**:
- ✅ 成功输出JSON和updatedContent字符串
- ❌ **已知问题**: Code节点无法返回binary字段（n8n版本限制）

## 📊 技术要点总结

### 1. 数据传递策略

**问题**: LLM返回的JSON嵌套在markdown代码块中，且缺少levelInfo

**解决方案**:
- 节点03输出级别信息 → 节点04B Merge节点合并 → 节点04C提取并整合
- 确保每个节点都能访问完整的数据（片段数据+级别信息）

### 2. 表达式引用

**成功案例**:
- `{{ $json.snippetFile }}` - 访问当前节点的JSON字段
- `{{ $binary.data }}` - 访问S3下载的文件内容
- `$input.first().binary.data.toString('utf-8')` - Code节点中读取文件

### 3. 节点连接模式

**关键节点**:
- 节点03同时连接到节点04和节点04B（并行分支）
- 节点04B使用Merge模式合并两个输入
- 节点04C将数组转换为单一JSON对象

### 4. 文件路径规范

**确认的路径结构**:
- ✅ 片段文件: `级别目录/文件名.md`（如 `初中生美文/初中生美文精选100篇.md`）
- ✅ 索引文件: `级别目录/美文收集索引.md`
- ✅ 模版文件: `级别目录/美文赏析与教学通用模版.md`
- ❌ **不使用** `articles/` 子目录
