## 📦 准备工作

### 1. 确认R2存储桶

在开始之前，请确认您的Cloudflare R2存储桶已配置好：

```bash
✅ 存储桶名称: cherrystudio
✅ 已上传所有索引文件
✅ 已上传片段收集文件
⚠️  需要上传模版文件到 templates/ 目录
```

### 2. 获取R2 API凭证

1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com/)
2. 进入 R2 → Overview → Manage R2 API Tokens
3. 创建新的API Token，记录：
   - Access Key ID
   - Secret Access Key
   - Account ID (用于构建Endpoint)

### 3. 配置LLM API

准备您的LLM API凭证：
- OpenAI API Key
- 或其他兼容的LLM服务

### 4. 在n8n中配置凭证

#### 步骤1: 添加R2凭证

1. 打开n8n
2. 点击左侧菜单 **Credentials**
3. 点击 **Add Credential**
4. 选择 **AWS** 类型（R2兼容S3 API）
5. 填写信息：
   ```
   Credential Name: R2 Account
   Access Key ID: [您的R2 Access Key]
   Secret Access Key: [您的R2 Secret Key]
   ```
6. 点击 **Save**

#### 步骤2: 配置R2 Endpoint

在创建AWS S3节点时，需要设置Custom Endpoint：
```
https://<ACCOUNT_ID>.r2.cloudflarestorage.com
```

将 `<ACCOUNT_ID>` 替换为您的实际Account ID。

#### 步骤3: 添加LLM凭证

1. 点击 **Add Credential**
2. 选择您的LLM类型（如OpenAI）
3. 填写API Key
4. 点击 **Save**

## 🛠️ 手动创建节点

### Step 0️⃣: 操作前准备（4个节点）

#### 节点00: 定时触发器

1. 点击 **Add nodes**，搜索 **Cron**
2. 选择 **Cron Trigger**
3. 配置：
   - **Name**: 定时触发器
   - **Cron Expression**: `0 9 * * *`
4. 点击 **Save**

#### 节点01: 读取总索引

1. 搜索 **AWS S3** 节点
2. 配置：
   - **Name**: 读取总索引
   - **Operation**: Download
   - **Bucket Name**: `cherrystudio`
   - **Key**: `美文总索引.md`
   - **Credentials**: 选择您的R2凭证
3. 在 **Advanced** 中设置 **Custom Endpoint**:
   ```
   https://<ACCOUNT_ID>.r2.cloudflarestorage.com
   ```
4. 点击 **Save**

#### 节点02: LLM分析索引

1. 搜索 **OpenAI** 节点（或您的LLM节点）
2. 配置：
   - **Name**: LLM分析索引
   - **Model**: `gpt-4`（或您的模型）
   - **Prompt**:
     ```
     分析以下美文总索引内容，提取：
     1. 各文件夹现有文章数量和标题
     2. 已收录作家名单和作品分布
     3. 题材覆盖情况
     4. 潜在重复风险作品

     总索引内容：
     {{ $binary.data }}

     请以JSON格式返回分析结果。
     ```
3. 点击 **Save**

## 🔗 节点连接

### 连接规则

1. **线性连接**: 大部分节点按顺序连接
2. **条件分支**: IF节点有True/False两个输出
3. **Switch分支**: Switch节点有多个输出

### 完整连接图

```
00 → 01 → 02 → 03 → 04 → 05 → 06 → 07 → 08 → 09 → 10
                                          → 11 → 12 (IF)
                                              ├→ True → 14 → 15 → 16 → 17
                                              └→ False → 13 (STOP)
                                                          → 18 → 19 → 20 (SWITCH)
                                                              ├→ proceed → 22 → 23 → 24 → 25 → 26 → 27
                                                              └→ stop → 21 (STOP)
                                                                           → 28 → 29 (IF)
                                                                               ├→ True → 30 → 31
                                                                               └→ False → 31
                                                                                        → 32 → 33 → 34 → 35 → 36
```

### 如何连接IF节点

节点12 (版本检查) 是IF节点：

1. 点击 **节点12**
2. 您会看到两个输出点：
   - ✅ (图标) - True输出
   - ❌ (图标) - False输出
3. 将True输出连接到 **节点14**
4. 将False输出连接到 **节点13**

### 如何连接Switch节点

节点20 (去重决策) 是Switch节点：

1. 点击 **节点20**
2. 您会看到多个输出点（取决于规则数量）
3. 将 **proceed** 输出连接到 **节点22**
4. 将 **stop** 输出连接到 **节点21**

## ❓ 常见问题

### Q1: 节点连接后没有数据传递

**原因**: 可能是引用表达式错误

**解决**:
- 检查表达式中的节点名称是否正确
- 使用n8n的表达式编辑器验证
- 查看节点的执行数据

### Q2: AWS S3节点报错 "Cannot connect"

**原因**: R2凭证或Endpoint配置错误

**解决**:
1. 检查Access Key ID和Secret Access Key
2. 确认Custom Endpoint格式：
   ```
   https://<ACCOUNT_ID>.r2.cloudflarestorage.com
   ```
3. 测试凭证：点击凭证的 **Test** 按钮

### Q3: LLM节点返回错误

**原因**: 提示词格式或API配额问题

**解决**:
1. 检查提示词中的表达式语法
2. 验证API Key是否有效
3. 检查API配额是否充足
4. 尝试简化提示词

### Q4: Code节点执行失败

**原因**: JavaScript语法错误或引用数据不存在

**解决**:
1. 使用n8n的Code编辑器调试
2. 检查变量引用是否正确
3. 查看错误堆栈信息

### Q5: 文件路径找不到

**原因**: R2中的文件路径与配置不匹配

**解决**:
1. 在Cloudflare Dashboard中查看实际文件结构
2. 确认文件名（包括扩展名）完全一致
3. 检查路径分隔符（使用 `/`）

### Q6: 工作流执行时间过长

**原因**: LLM调用耗时或网络延迟

**解决**:
1. 在节点设置中增加超时时间
2. 使用更快的LLM模型
3. 检查网络连接

### Q7: 如何调试单个节点？

**方法**:
1. 点击要测试的节点
2. 点击 **Execute Node**（仅执行当前节点）
3. 查看输入输出数据
4. 使用n8n的日志功能

### Q8: 如何修改定时时间？

**方法**:
1. 点击 **定时触发器** 节点
2. 修改 **Cron Expression**
3. 常用示例：
   - `0 9 * * *` - 每天早上9点
   - `0 */6 * * *` - 每6小时一次
   - `0 9 * * 1` - 每周一早上9点
