## 🎯 核心功能(跨级别检测)

### 使用场景
在添加新内容前或定期维护时,检测并处理重复内容和序号冲突。

**分级系统特性:**
- **同级检测**: 在同一级别内检测重复(如小学生美文内部)，绝对不允许同一文件夹内重复
- **跨级关联**: 跨级别检测相同文章，允许不同文件夹出现相同文章，但需建立双向链接
- **智能建议**: 根据文章难度,建议更合适的收录级别

### 核心规则
1. **同一文件夹**: 绝对不允许出现相同文章(作品名+作者重复)
2. **跨文件夹**: 允许相同文章存在，但必须建立双向链接指向相应出处
3. **🚨 数量限制**: 同一作品(作品名+作者)在全库中不得超过2篇，包含2篇
4. **总索引预检**: 收录前必须查看`美文总索引.md`，确认重复情况和数量限制
5. **去重流程**: 先进行总索引预检 → 文件所在文件夹去重 → 跨文件夹数量检查 → 建立关联链接

**触发条件:**
- 添加新内容前(预检查) - **必须包含总索引预检**
- 用户说"检查{级别}重复"
- 用户说"全局去重检查"(检查所有级别)
- 定期维护(建议每月一次)

#### 预检流程
```python
def master_index_pre_check(title, author):
    """
    总索引预检 - 收录前必须执行
    检查拟收录作品的现有收录情况和数量限制
    """
    # 1. 读取美文总索引.md
    master_index = read_file("美文总索引.md")
    
    # 2. 搜索该作品的现有收录情况
    existing_entries = search_in_master_index(title, author)
    
    # 3. 统计跨文件夹数量
    cross_folder_count = len(existing_entries)
    
    # 4. 检查3篇限制
    if cross_folder_count >= 2:
        return {
            "status": "exceeded_limit",
            "message": f"该作品已达到2篇上限，不可再收录",
            "existing": existing_entries,
            "recommendation": "选择该作家的其他作品进行收录"
        }
    
    return {
        "status": "allowed",
        "current_count": cross_folder_count,
        "remaining_slots": 2 - cross_folder_count,
        "existing_entries": existing_entries,
        "action": "proceed_with_deduplication"
    }
```

### 检测维度1: 文章去重

#### 精确匹配
```python
def check_exact_duplicate(title, author, current_folder):
    """
    检查: 作品名+作者 完全相同
    规则: 
    1. 同一文件夹内不允许重复
    2. 跨文件夹允许但需建立链接
    3. 全库总数不得超过3篇限制
    4. 必须先执行总索引预检
    例如: 《春》+ 朱自清
    """
    # 0. 先执行总索引预检
    pre_check = master_index_pre_check(title, author)
    if pre_check["status"] == "exceeded_limit":
        return pre_check  # 直接返回超限错误
    
    key = f"{title}|{author}"
    
    # 1. 检查同一文件夹内是否重复
    if key in existing_articles[current_folder]:
        return {
            "status": "duplicate_same_folder",
            "message": "同一文件夹内已存在相同文章",
            "action": "remove_duplicate"
        }
    
    # 2. 检查其他文件夹是否存在
    cross_folder_matches = []
    for folder in existing_articles:
        if folder != current_folder and key in existing_articles[folder]:
            cross_folder_matches.append(folder)
    
    # 3. 检查跨文件夹数量限制
    total_count = len(cross_folder_matches) + 1  # +1为当前准备收录的
    if total_count > 2:
        return {
            "status": "exceeded_limit",
            "message": f"收录后将达到{total_count}篇，超过2篇限制",
            "existing_folders": cross_folder_matches,
            "recommendation": "选择该作家的其他作品进行收录"
        }
    
    if cross_folder_matches:
        return {
            "status": "duplicate_cross_folder",
            "message": f"其他文件夹已存在相同文章: {', '.join(cross_folder_matches)}",
            "action": "create_bidirectional_link",
            "target_folders": cross_folder_matches,
            "total_count": total_count,
            "remaining_slots": 2 - total_count
        }
    
    return {
        "status": "unique",
        "message": "文章唯一",
        "action": "add_article",
        "pre_check_status": pre_check
    }
```

#### 模糊匹配
```python
def check_fuzzy_duplicate(title1, title2):
    """
    检查: 作品名相似(可能是别名或简称)
    例如: 《春天》vs《春》, 《海燕》vs《燕子》
    """
    # 使用编辑距离算法
    distance = levenshtein_distance(title1, title2)
    similarity = 1 - (distance / max(len(title1), len(title2)))
    return similarity > 0.8  # 相似度阈值80%
```

**示例:**

```
文章2: 《春》- 朱自清

```

**目的:** 保持作家多样性,避免某个作家占比过高

```python
def check_author_distribution():
    """
    检查作家分布
    单个作家不应超过总数的10%
    """
    author_count = {}
    total_count = 0
    
    for article in articles:
        author = article['author']
        author_count[author] = author_count.get(author, 0) + 1
        total_count += 1
    
    # 检查占比
    warnings = []
    for author, count in author_count.items():
        ratio = count / total_count
        if ratio > 0.1:  # 超过10%
            warnings.append({
                'author': author,
                'count': count,
                'ratio': f"{ratio*100:.1f}%",
                'excess': count - int(total_count * 0.1)
            })
    
    return warnings
```

**示例:**

```
→ 超过阈值10%
→ 超出数量: 3篇

```

```yaml
1. 检测超限：现有2篇，收录后3篇
2. 立即停止：❌ 不可收录
3. 替代建议：选择该作家的其他作品
4. 重新验证替代作品的重复情况
```

### 方案4: 半自动解决(中风险)
```yaml
适用场景:
  - 相似但不完全相同的文章
  - 序号冲突但内容不一致

操作:
  - 生成建议
  - 需要用户确认
  - 执行用户选择

风险: 中
建议: 用户确认后执行
```

### 方案5: 人工处理(高风险)
```yaml
适用场景:
  - 复杂的版本差异
  - 不确定的相似度
  - 需要专业判断的情况

操作:
  - 详细说明情况
  - 提供多个选项
  - 完全由用户决策

风险: 高
建议: 仅提供信息,不自动执行
```
