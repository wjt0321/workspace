## 🎯 核心功能

### 使用场景
在美文收集工作流程的第7步执行，专门负责对所有已创建的文章进行全面的质量检查和问题修复。

**触发条件:**
- 美文收集工作流程的第7步
- 在所有文章格式化完成后执行
- 在建立链接之后、总索引更新之前执行

## ⚠️ 核心原则

### 质量检查铁律
- **必须全面检查**：每篇文章都要逐项检查
- **发现问题必须修复**：不能跳过任何问题
- **标准不可妥协**：必须100%符合对应级别模版要求
- **检查完成后验证**：修复后必须重新验证

## 🛠️ 完整质量检查流程

### 第一步：格式化完整性检查

**检查项目：**
1. **YAML元数据检查**：
   ```markdown
   - [ ] title: 必须包含完整作品名
   - [ ] source: 必须包含作者信息
   - [ ] created: 必须包含创建日期
   - [ ] tags: 必须包含对应级别标签
   - [ ] 完整版标签: 必须标注"完整版"
   ```

2. **标题区检查**：
   ```markdown
   - [ ] H1标题: 必须包含Emoji + 完整作品名 + 完整版标注
   - [ ] H3作者署名: 必须正确格式化作者信息
   - [ ] 开头Emoji: 必须添加2-3个主题相关Emoji组合
   ```

3. **核心板块检查**：
   ```markdown
   #### 小学生版（7板块）
   - [ ] 📖 原文正文
   - [ ] 🌟 美文赏析（写作特点+学习要点+优美词句）
   - [ ] 🎭 朗读指导
   - [ ] 🌈 小练笔
   - [ ] 📚 知识小卡片
   
   #### 初中生版（7板块）
   - [ ] 📖 原文正文
   - [ ] 🌟 美文赏析（艺术手法+主题思想+精彩语段）
   - [ ] 🎭 朗读与批注
   - [ ] 🌈 写作借鉴
   - [ ] 📚 文学常识 + 🎯 中考链接
   
   #### 高中生版（8板块）
   - [ ] 📖 原文正文
   - [ ] 🌟 文学赏析（艺术特色+主题意蕴+细读文本）
   - [ ] 🎭 批评视角
   - [ ] 🌈 写作借鉴
   - [ ] 📚 作家作品 + 🎯 高考链接
   - [ ] 🔗 知识网络
   
   #### 大学生版（10板块）
   - [ ] 📖 原文正文
   - [ ] 📝 版本校勘
   - [ ] 🌟 学术研究（文本细读+理论批评+风格学）
   - [ ] 🌐 语境研究
   - [ ] 📖 作家研究
   - [ ] 🔗 互文性研究
   - [ ] 📊 研究综述
   - [ ] 📚 参考文献
   
   #### 成人版（9板块）
   - [ ] 📖 原文正文
   - [ ] 💎 深度品读（文学之美+人生启示+金句）
   - [ ] 🎭 生活映照
   - [ ] 🌟 作者世界
   - [ ] 🍃 延伸阅读
   - [ ] ☕ 品读札记
   - [ ] 🎯 推荐人群 + 🌈 生活应用
   ```

### 第二步：正文Emoji穿插专项检查

**检查标准：**
1. **开头Emoji要求**：
   ```markdown
   - [ ] 正文开头已添加2-3个主题相关Emoji组合
   - [ ] Emoji与文章主题高度相关
   - [ ] 不影响阅读流畅性
   ```

2. **段落Emoji穿插**：
   ```markdown
   #### 统一标准（所有级别适用）
   - [ ] 每个段落都有Emoji穿插
   - [ ] Emoji密度统一（每2-3句话1个）
   - [ ] 所有Emoji都与句子内容相关
   - [ ] 句子定义：以任何标点符号结尾的完整语义单元（句号、问号、感叹号、分号、逗号等）
   - [ ] 不采用克制规则，保持一致的Emoji使用标准
   ```

3. **Emoji质量检查**：
   ```markdown
   - [ ] Emoji选择恰当，符合文章意境
   - [ ] 所有Emoji都与句子内容高度相关
   - [ ] 跨平台显示兼容性良好
   - [ ] 不使用生僻或可能引起歧义的Emoji
   - [ ] 统一按每2-3句话一个Emoji的标准执行
   ```

### 第三步：链接有效性验证

**检查项目：**
1. **内部链接检查**：
   ```markdown
   - [ ] 片段→完整文章链接可正常跳转
   - [ ] 完整文章→片段链接可正常跳转
   - [ ] 文章→文章相关链接可正常跳转
   - [ ] 索引文件链接可正常跳转
   ```

2. **链接格式检查**：
   ```markdown
   - [ ] 使用标准Obsidian链接格式：[[文件名]]
   - [ ] 文件名准确无误
   - [ ] 特殊字符处理正确
   - [ ] 路径相对关系正确
   ```

3. **链接完整性检查**：
   ```markdown
   - [ ] 所有必需的双向链接都已建立
   - [ ] 无孤立的文件（无入链或出链）
   - [ ] 跨文件夹重复文章的双向链接正确
   - [ ] 🔗标记正确标注重复文章
   ```

### 第四步：问题修复执行

**修复流程：**
1. **问题分类**：
   ```markdown
   🔴 严重问题：必须立即修复
   - 缺少必需板块
   - YAML元数据错误
   - 核心格式错误
   
   🟡 一般问题：建议修复
   - Emoji使用不当
   - 链接格式小问题
   - 内容格式不统一
   
   🟢 优化建议：可选修复
   - 文字表述优化
   - 布局美观度提升
   ```

2. **修复执行**：
   ```markdown
   - 按优先级逐项修复问题
   - 修复后立即验证结果
   - 记录修复过程和结果
   - 确保修复不引入新问题
   ```

## ✅ 完成检查清单

### 必须完成项目
- [ ] **所有文章已检查**：N篇文章全部完成质量检查
- [ ] **严重问题已修复**：所有严重级别问题都已解决
- [ ] **格式标准符合**：100%符合对应级别模版要求
- [ ] **Emoji使用合规**：符合统一标准和相关性要求
- [ ] **链接全部有效**：所有链接都可正常跳转
- [ ] **验证结果记录**：检查和修复过程已完整记录

### 质量标准
- **零容忍项目**：缺板块、YAML错误、格式错误
- **必须达标项目**：Emoji使用（统一标准）、链接有效性、内容完整性
- **持续改进项目**：表述优化、布局美观、用户体验

### Emoji统一标准
- **密度要求**：严格按每2-3句话一个Emoji执行
- **句子定义**：以任何标点符号结尾的完整语义单元
- **相关性要求**：所有Emoji必须与句子内容高度相关
- **统一性**：所有级别采用相同标准，无克制或差异化

- **及时性**：发现问题立即修复，不拖延
- **验证性**：修复后必须验证，确保有效
