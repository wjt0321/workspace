```yaml
author: AI Assistant
created: 2025-12-14
tags: [散文收集, 文学, 片段提取, 自动化]
```

## 🎯 核心功能

### 使用场景
当用户需要扩充散文片段库时,使用此skill自动收集著名作家的经典散文片段。

**触发条件:**
- 用户说"帮我收集N个新的散文片段"
- 用户说"添加一些经典美文片段"
- 用户说"扩充片段库"

## 🏗️ 执行流程

### Step 1: 识别目标级别并定位工作区

**分级系统(5个级别):**
```yaml
小学生美文: d:\workspace\小学生美文\
初中生美文: d:\workspace\初中生美文\
高中生美文: d:\workspace\高中生美文\
大学生美文: d:\workspace\大学生美文\
成人美文: d:\workspace\成人美文\
```

**用户需求识别:**
- 当用户说"收集小学美文"或未指定 → 默认`小学生美文`
- 当用户说"收集初中美文" → `初中生美文`
- 当用户说"收集高中美文" → `高中生美文`
- 当用户说"收集大学美文" → `大学生美文`
- 当用户说"收集成人美文" → `成人美文`

**工作目录定位:**
```bash
根据用户需求,定位到对应目录:
关键文件:
- 美文收集索引.md (索引文件)
```

### Step 2: 读取当前状态
1. 读取索引文件,确定:
   - 已使用的序号列表(1-100)
   - 已收录的作品+作者组合
   - 当前文件是否已满(100篇)
2. 如果已满100篇 → 创建新文件(如`中学生美文精选100篇.md`)

### Step 3: 随机选择作家和作品
**作家池(近现代中文散文大家):**

#### 经典大家(优先级⭐⭐⭐)
- 朱自清、老舍、冰心、巴金、郁达夫
- 鲁迅、沈从文、汪曾祺、丰子恺
- 叶圣陶、梁实秋、徐志摩、周作人

#### 散文名家(优先级⭐⭐)
- 林清玄、余秋雨、季羡林、史铁生
- 贾平凹、铁凝、毕淑敏、宗璞
- 杨朔、秦牧、刘白羽、魏巍

#### 当代作家(优先级⭐)
- 莫言、余华、王安忆、迟子建
- 张晓风、简媜、林文月、龙应台

**选择策略:**
1. 随机选择1位作家
2. 从该作家代表作中选择知名散文
3. 避免重复(检查"作品名+作者"是否已存在)
4. 保持多样性(单个作家片段数≤总数的10%)

### Step 4: 提取经典片段
**片段标准:**
- 长度: 按年龄段分级（与标准流程对齐）
  - 小学生: 150-300字
  - 初中生: 200-400字
  - 高中生: 300-500字
  - 大学生: 300-500字
  - 成人: 300-500字
- 完整性: 包含完整段落或场景
- 文采: 语言优美、意境深远
- 适配性: 适合目标年龄段

**提取技巧:**
- 选择文章的精华段落(通常是描写或抒情部分)
- 避免开头/结尾的叙述性段落
- 确保片段可独立阅读理解

### Step 5: 格式化片段
```markdown
#### **{序号}. 《{作品名}》 - {作者}**

> {片段内容开头Emoji组合}
> 
> {穿插Emoji的片段内容...}
> 
> {如果跨段,继续引用...}

**【推荐理由】** {50字以内的推荐语}

[[{完整文章文件名}]] {如果已收录完整版,添加双向链接}

---
```

**片段穿插Emoji规范（统一标准）:**
- **开头Emoji**: 添加2-3个与片段主题相关的Emoji组合
- **内容穿插**: 在关键描述、情感表达处适度添加Emoji
- **密度控制**: 统一按每2-3句话添加1个Emoji（所有级别相同）
- **句子定义**: 以任何标点符号结尾的完整语义单元
- **质量标准**: 增强可读性,不影响文学性
- **相关性要求**: 所有Emoji必须与句子内容高度相关

**示例:**
```markdown
#### **35. 《绿》 - 朱自清**

> 💚🌊
> 
> 梅雨潭闪闪的绿色招引着我们 🌿；我们开始追捉她那离合的神光了 ✨。揪着草 🌿，攀着乱石 ⛰️，小心探身下去 🧎‍♂️，又鞠躬过了一个石穹门 🏛️，便到了汪汪一碧的潭边了 💧。瀑布在襟袖之间 💦；但我的心中已没有瀑布了 🤫。我的心随潭水的绿而摇荡 💫。那醉人的绿呀！🌿 我若能裁你以为带 👗，我将赠给那轻盈的舞女 💃；她必能临风飘举了 🍃。

**【推荐理由】** 朱自清先生用极富想象力的语言描绘梅雨潭的绿,将静态色彩写得生动活泼,是学习色彩描写的绝佳范文。

[[12.绿-朱自清]]

---
```

### Step 6: 更新索引文件
在索引文件中添加:
```markdown
{序号}. 《{作品名}》 - {作者}【行号】
```

## ⚠️ 重要规则

### 去重机制
1. **作品级去重**: 检查"作品名+作者"组合
2. **序号去重**: 确保1-100序号不重复
3. **内容级去重**: 避免同一作品的不同片段重复

### 作家筛选规则
✅ **必须收录:**
- 近现代中文散文大家
- 有明确文学史地位
- 作品入选权威教材或文学史

❌ **严格排除:**
- 古典作者(如苏轼、王安石等)
- 外文作者(如海伦·凯勒、马克·汉林等)
- 网络作家、非文学人物

### 质量控制
- 片段必须来自作家的代表作或知名作品
- 优先选择教材常见作品(便于后续查找完整版)
- 避免过于深奥或晦涩的内容(根据目标年龄段)

## 🔍 智能推荐模式

### 根据现有片段推荐互补内容
分析已收录片段的:
- 题材分布(写景/状物/记事/抒情)
- 作家分布(是否某些作家过多)
- 年龄段适配性

**推荐策略:**
- 如果写景类过多 → 推荐记事/哲理类
- 如果某作家片段>10% → 推荐其他作家
- 如果低年级内容过多 → 推荐适合高年级的深度作品

## 🔧 故障排查

### 常见问题
1. **Q: 序号重复了怎么办?**
   - A: 使用自然数叠加(101, 102...)或检查索引文件修正

2. **Q: 找不到合适的片段?**
   - A: 扩大作家范围,或降低片段长度要求(150-300字)

3. **Q: 如何判断片段质量?**
   - A: 参考"推荐理由"标准:文采、技巧、适配性

4. **Q: 片段太短/太长?**
   - A: 灵活调整对应年龄段的长度标准，但必须保持完整性
     - 小学: 150-300字
     - 初中: 200-400字
     - 高中/大学/成人: 300-500字
