### 1.2 核心知识点详解

#### 1.2.1 Day 29：NLP基础与文本预处理（奠基模块）

**核心概念**：
1. **自然语言处理定义**：人工智能与语言学的交叉领域，使计算机能够理解、解释和生成人类语言
2. **NLP发展五阶段**：规则驱动→统计方法→深度学习→预训练模型→大语言模型时代（2018-2026）
3. **核心任务分类**：
   - 文本理解类：文本分类、情感分析、命名实体识别（NER）
   - 文本生成类：机器翻译、文本摘要、对话生成
   - 序列标注类：词性标注、句法分析

**关键技术**：
1. **文本预处理八步流程**：
   ```
   原始文本 → 数据清洗 → 分词 → 规范化 → 停用词去除 → 词形还原 → 特征提取 → 向量化
   ```
2. **中文分词技术**：jieba库三种模式（精确模式、全模式、搜索引擎模式）
3. **文本表示方法演进**：
   - 传统方法：词袋模型（BoW）、TF-IDF（词频-逆文档频率）
   - 现代方法：词向量（Word2Vec、GloVe、FastText）

#### 1.2.2 Day 30：词向量与Word2Vec（语义理解核心）

**核心突破**：解决传统One-Hot编码的三大缺陷（维度灾难、语义鸿沟、稀疏低效）

**Word2Vec两大模型架构**：
1. **CBOW模型**：用上下文预测中心词（训练快，适合高频词）
   - 数学形式：h = (1/2m) Σᵢ W₁·v(wₜ₊ᵢ)，P(wₜ|context) = softmax(W₂ᵀ·h)
2. **Skip-gram模型**：用中心词预测上下文（对低频词友好，语义捕捉强）
   - 数学形式：P(wₜ₊ⱼ|wₜ) = softmax(W₂ᵀ·W₁·v(wₜ))

**关键技术优化**：
1. **负采样（Negative Sampling）**：将多分类转为二分类，复杂度从O(V)降至O(k+1)
2. **层次Softmax（Hierarchical Softmax）**：基于哈夫曼树的概率计算，复杂度从O(V)降至O(log V)

**词向量神奇特性**：
- 线性类比关系：vec("国王") - vec("男人") + vec("女人") ≈ vec("女王")
- 语义相似度计算：余弦相似度、欧氏距离
- 高级演进：GloVe（全局统计+局部预测）、FastText（子词表示）

#### 1.2.3 Day 31：RNN与LSTM基础（序列建模核心）

**RNN核心创新**：引入隐藏状态作为"记忆"，处理序列数据的天然架构

**RNN数学原理**：
```
h_t = f(W_hh * h_{t-1} + W_xh * x_t + b_h)
y_t = g(W_hy * h_t + b_y)
```

**RNN四大架构**：
1. 一对一（One-to-One）：传统神经网络
2. 一对多（One-to-Many）：图像描述生成
3. 多对一（Many-to-One）：情感分析
4. 多对多（Many-to-Many）：机器翻译

**RNN致命缺陷**：梯度消失与梯度爆炸问题
- 数学本质：∂h_t/∂h_k = ∏_{i=k+1}^t (∂h_i/∂h_{i-1}) → 指数衰减/增长
- 后果：无法学习长期依赖关系

**LSTM三大创新**：
1. **细胞状态（Cell State）**：记忆高速公路，信息几乎无损流动
2. **门控机制**：遗忘门、输入门、输出门精确控制信息流
3. **加法更新**：C_t = f_t ⊙ C_{t-1} + i_t ⊙ C̃_t，避免梯度指数衰减

**GRU简化设计**：
- 更新门（z_t）控制新旧信息的混合
- 重置门（r_t）控制过去信息利用
- 参数减少约1/3，训练更快

#### 1.2.4 Day 32：文本分类实战（经典应用）

**文本分类完整六步流程**：
1. 需求定义与数据收集
2. 数据预处理与清洗
3. 特征提取与向量化
4. 模型选择与训练
5. 模型评估与优化
6. 部署与监控

**核心模型技术对比**：
| 模型 | 核心思想 | 优点 | 缺点 | 适用场景 |
|------|----------|------|------|----------|
| 朴素贝叶斯 | 贝叶斯定理，特征条件独立 | 训练快，小数据友好 | 独立性假设过于理想 | 垃圾邮件检测 |
| 支持向量机 | 最大间隔超平面 | 高维稳定，泛化强 | 训练复杂度高 | 新闻分类 |
| 逻辑回归 | 逻辑函数输出概率 | 可解释性强，计算高效 | 线性可分假设 | 情感二分类 |
| TextCNN | 一维卷积捕捉n-gram | 并行快，关键词组合 | 忽略长距离依赖 | 短文本分类 |
| LSTM | 序列建模，记忆机制 | 捕捉上下文信息 | 训练慢，难并行 | 长文本分析 |
| BERT | 自注意力全局建模 | 分类效果SOTA | 计算资源要求高 | 高精度场景 |

**评估指标体系**：
- 基础四指标：准确率、精确率、召回率、F1分数
- 多分类策略：宏平均、微平均、加权平均
- 实践选择：高风险场景重召回，内容审核重精确

#### 1.2.6 Day 34：文本生成入门（前沿探索）

**文本生成三大范式演进**：
1. **基于规则的方法**（1950s-1970s）：模板填充，规则驱动
2. **基于统计的方法**（1980s-2000s）：n-gram语言模型，统计机器翻译
3. **基于深度学习的方法**（2010s-至今）：
   - 序列到序列模型（Seq2Seq）
   - Transformer架构革命
   - 大语言模型时代（GPT系列）

**GPT核心原理**：
1. **自回归生成**：基于前文预测下一个词
2. **Transformer解码器**：多头自注意力 + 前馈网络
3. **预训练-微调范式**：大规模无监督预训练 + 任务特定微调

**生成策略对比**：
| 策略 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| 贪婪搜索 | 每一步选概率最大词 | 简单高效 | 易陷入局部最优 | 快速原型 |
| 集束搜索 | 保留k个候选序列 | 质量较好，可调节 | 计算量较大 | 机器翻译 |
| 随机采样 | 按概率分布随机采样 | 多样性高，创造性 | 可能不连贯 | 创意写作 |
| 核采样 | 仅从高概率词汇采样 | 平衡质量与多样性 | 超参数敏感 | 对话生成 |

**应用场景矩阵**：
| 应用领域 | 典型任务 | 技术特点 | 评估重点 |
|----------|----------|----------|----------|
| 创意写作 | 诗歌生成、小说续写 | 多样性、创造性 | 流畅性、新颖性 |
| 代码生成 | 程序合成、代码补全 | 逻辑性、正确性 | 语法正确、功能实现 |
| 对话系统 | 聊天机器人、客服助手 | 连贯性、相关性 | 回复质量、上下文理解 |
| 内容创作 | 新闻摘要、产品描述 | 准确性、完整性 | 信息覆盖、事实正确 |

### 2.1 单选题（5道，每题2分）

#### 题目1：中文分词的核心挑战是什么？
A. 词汇量过大
B. 没有空格作为天然分隔符
C. 语法结构复杂
D. 方言变体多样

**正确答案**：B
**答案解析**：中文与英文等拉丁语系语言最根本的区别在于中文书写时词汇之间没有空格分隔，如"我爱自然语言处理"需要算法识别"我"、"爱"、"自然语言"、"处理"的边界，这是中文分词技术的核心挑战。其他选项虽然也是中文处理的难点，但非分词特有的核心挑战。

#### 题目2：Word2Vec的Skip-gram模型训练目标是什么？
A. 用中心词预测上下文词
B. 用上下文词预测中心词
C. 学习词汇的共现矩阵
D. 构建词汇的语义层次结构

**正确答案**：A
**答案解析**：Skip-gram模型的核心思想是通过中心词来预测其上下文窗口内的词汇，其训练目标是最大化给定中心词条件下上下文词出现的概率。CBOW模型才是用上下文预测中心词，GloVe关注共现矩阵，层次结构是WordNet等词汇知识库的目标。

#### 题目3：普通RNN面临梯度消失问题的根本原因是什么？
A. 激活函数选择不当
B. 权重矩阵初始化问题
C. 反向传播中的链式乘法导致梯度指数衰减
D. 学习率设置过大

**正确答案**：C
**答案解析**：梯度消失的数学本质是在沿时间反向传播（BPTT）过程中，梯度需要通过链式法则乘以一系列雅可比矩阵∂h_i/∂h_{i-1}，每个矩阵包含激活函数导数和权重，当这些值的乘积小于1时，梯度会指数衰减到接近零。其他选项可能影响训练但不构成根本原因。

#### 题目4：情感分析任务中，处理类别不平衡问题的最有效方法是什么？
A. 随机删除多数类样本
B. 复制少数类样本
C. 使用类别权重调整损失函数
D. 综合采用过采样、欠采样和权重调整的组合策略

**正确答案**：D
**答案解析**：实际项目中，单一的过采样或欠采样往往效果有限，且容易导致过拟合或信息丢失。最有效的做法是综合策略：对多数类进行欠采样、对少数类进行SMOTE过采样，同时在损失函数中设置类别权重。这种组合策略在实践中被证明最为稳健。

#### 题目5：文本生成中，集束搜索（Beam Search）相比贪婪搜索的核心优势是什么？
A. 计算速度更快
B. 内存消耗更少
C. 考虑了多个候选序列，避免局部最优
D. 生成文本更长

**正确答案**：C
**答案解析**：集束搜索在每一步保留k个概率最高的候选序列，而不是只保留最优的一个（贪婪搜索），这样可以在全局范围内找到更优的生成结果，避免陷入局部最优。代价是计算量和内存消耗增加，但生成质量通常更好。

### 2.2 多选题（5道，每题3分）

#### 题目6：文本预处理中，中文特有的处理步骤包括哪些？（多选）
A. 分词处理
B. 繁简转换
C. 大小写统一
D. 新词发现
E. 词形还原

**正确答案**：A、B、D
**答案解析**：中文NLP特有的挑战包括：①分词（无空格分隔）；②繁简转换（两岸三地用字差异）；③新词发现（网络用语、领域术语）。大小写统一是英文特有，词形还原在中文中不显著（中文词汇无时态、单复数变化）。

#### 题目7：Word2Vec中采用的优化技术包括哪些？（多选）
A. 负采样（Negative Sampling）
B. 层次Softmax（Hierarchical Softmax）
C. Dropout正则化
D. 批量归一化（Batch Normalization）
E. 自适应学习率（Adam）

**正确答案**：A、B
**答案解析**：Word2Vec原始论文中主要提出两种优化技术应对大规模词表：负采样（将多分类转为二分类）和层次Softmax（基于哈夫曼树的概率计算）。Dropout、批量归一化和Adam是后来深度学习中的通用技术，非Word2Vec特有。

#### 题目8：LSTM中门控机制的功能包括哪些？（多选）
A. 决定遗忘哪些旧信息
B. 决定存储哪些新信息
C. 决定输出什么信息
D. 调整学习率
E. 正则化防止过拟合

**正确答案**：A、B、C
**答案解析**：LSTM的核心是三个门控：①遗忘门（f_t）决定从细胞状态丢弃什么信息；②输入门（i_t）决定将哪些新信息存入细胞状态；③输出门（o_t）基于细胞状态决定输出什么。学习率调整和正则化是训练策略，非门控机制功能。

#### 题目9：文本分类任务中，适用于二分类场景的评估指标包括哪些？（多选）
A. 准确率（Accuracy）
B. 精确率（Precision）
C. 召回率（Recall）
D. F1分数
E. 宏平均F1（Macro-F1）

**正确答案**：A、B、C、D
**答案解析**：二分类任务的标准评估指标包括：准确率、精确率、召回率、F1分数（精确率和召回率的调和平均）。宏平均F1是多分类场景的评估方法，用于计算多个类别的平均F1分数。

#### 题目10：文本生成的应用场景中，哪些任务特别适合使用随机采样策略？（多选）
A. 代码生成
B. 诗歌创作
C. 新闻摘要
D. 对话系统
E. 技术文档翻译

**正确答案**：B、D
**答案解析**：随机采样策略能产生更多样化、创造性的文本，适合：①诗歌创作等创意写作任务；②对话系统中需要丰富多样的回复。代码生成、新闻摘要、技术翻译更需要准确性和一致性，通常使用集束搜索等确定性更强的策略。

### 2.3 判断题（5道，每题2分）

#### 题目11：TF-IDF特征表示能够捕捉词汇之间的语义相似性。
**正确答案**：错误
**解析**：TF-IDF是基于词频统计的特征表示方法，只考虑词汇在文档中的出现频率和全局重要性，无法捕捉词汇间的语义关系（如同义词、反义词关系）。词向量（Word2Vec等）才能学习语义表示。

#### 题目12：GRU是LSTM的简化版本，参数量减少约1/3，但在某些任务上性能接近甚至超过LSTM。
**正确答案**：正确
**解析**：GRU将LSTM的遗忘门和输入门合并为更新门，同时合并细胞状态和隐藏状态，参数量显著减少。大量研究（如2014年Cho等人的论文）表明，在多数序列建模任务中，GRU性能与LSTM相当甚至有时更好，且训练更快。

#### 题目13：BERT模型在文本分类任务中，通常需要从头开始训练所有参数才能取得好效果。
**正确答案**：错误
**解析**：BERT的核心优势是预训练-微调范式。先在无标签的大规模语料上进行预训练（掩码语言建模和下一句预测），然后在具体任务（如文本分类）上进行微调，只需训练少量额外参数（如分类头），即可取得优异效果，无需从头训练。

#### 题目14：在情感分析中，否定词"不"通常应该被作为停用词直接去除。
**正确答案**：错误
**解析**：在情感分析任务中，否定词（"不"、"没有"、"非"等）对情感极性有重要影响（如"好"vs"不好"）。如果将其作为停用词去除，会导致语义改变和分类错误。情感分析需要专门处理否定词，而不是简单去除。

#### 题目15：集束搜索（Beam Search）的beam width参数k越大，生成文本的质量一定越高。
**正确答案**：错误
**解析**：虽然增大k通常能提高生成质量（考虑更多候选路径），但存在边际递减效应。当k过大时，计算成本显著增加，但质量提升有限，甚至可能因搜索空间过大而引入噪声。实践中需要平衡质量与效率，k通常取5-10。

    def load_stopwords(self):
        """加载中文停用词表"""
        # 方案1：使用内置停用词表
        stopwords = set(['的', '了', '在', '是', '我', '有', '和', '就', 
                        '不', '人', '都', '一', '一个', '上', '也', '很', 
                        '到', '说', '要', '去', '你', '会', '着', '没有', 
                        '看', '好', '自己', '这'])
        
        # 方案2：从文件加载（如果存在stopwords.txt）
        try:
            with open('stopwords.txt', 'r', encoding='utf-8') as f:
                stopwords = set([line.strip() for line in f])
        except FileNotFoundError:
            pass
            
        return stopwords
    
    def preprocess_text(self, text):
        """中文文本预处理"""
        # 1. 简单清洗：去除标点、数字等（根据任务调整）
        import re
        text = re.sub(r'[^\u4e00-\u9fa5]', ' ', text)  # 只保留中文
        
        # 2. 分词
        words = jieba.lcut(text)
        
        # 3. 去除停用词和单字
        filtered_words = [w for w in words if w not in self.stopwords and len(w) > 1]
        
        # 4. 返回处理后的文本（空格分隔）
        return ' '.join(filtered_words)
    
    def train(self, texts, labels):
        """训练文本分类模型"""
        # 1. 文本预处理
        processed_texts = [self.preprocess_text(t) for t in texts]
        
        # 2. 特征提取（TF-IDF）
        self.vectorizer = TfidfVectorizer(max_features=5000)
        X = self.vectorizer.fit_transform(processed_texts)
        
        # 3. 划分训练测试集
        X_train, X_test, y_train, y_test = train_test_split(
            X, labels, test_size=0.2, random_state=42
        )
        
        # 4. 训练朴素贝叶斯模型
        self.model = MultinomialNB()
        self.model.fit(X_train, y_train)
        
        # 5. 评估模型
        y_pred = self.model.predict(X_test)
        print("分类报告:")
        print(classification_report(y_test, y_pred))
        
        return self
    
    def predict(self, text):
        """预测新文本类别"""
        # 1. 预处理
        processed = self.preprocess_text(text)
        
        # 2. 特征转换
        features = self.vectorizer.transform([processed])
        
        # 3. 预测
        prediction = self.model.predict(features)[0]
        
        return prediction

### 3.2 常见困惑点识别

#### 高频困惑点

1. **词向量维度选择**：50维、100维、300维如何选择？
   - **解答**：小数据（<10万词）用50-100维，大数据用200-300维；任务简单用低维，复杂用高维；通过实验确定最优维度。

2. **RNN/LSTM/GRU选择**：实际项目中如何抉择？
   - **解答**：数据少选GRU（参数少），长序列选LSTM（记忆强），实时应用选GRU（计算快），精度优先选LSTM；先试GRU，效果不足再试LSTM。

3. **文本生成质量评估**：如何客观评估生成文本质量？
   - **解答**：采用BLEU、ROUGE等自动指标，辅以人工评估（流畅度、相关性、事实性）；实际应用中结合业务指标（用户满意度、点击率）。

4. **预训练模型选择**：BERT、ERNIE、RoBERTa等如何选择？
   - **解答**：中文任务优先ERNIE（中文优化），多语言任务选BERT，计算资源有限选DistilBERT，追求效果选最新大模型；通过小规模实验验证。

5. **类别不平衡处理**：多种方法如何组合使用？
   - **解答**：先尝试类别权重调整，效果不足再加过采样（SMOTE），最后考虑集成方法；避免单纯复制少数类样本导致过拟合。
