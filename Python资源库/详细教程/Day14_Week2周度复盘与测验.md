### 2. 异常处理机制（Day 9）

**核心概念**：优雅处理程序运行中的错误，提高程序健壮性

| 知识点 | 关键内容 | 应用场景 |
|--------|----------|----------|
| **异常捕获** | `try-except`结构 | 捕获特定类型的错误 |
| **异常类型** | `ValueError`、`FileNotFoundError`、`ZeroDivisionError`等 | 不同类型的错误处理 |
| **多个异常** | `except (Type1, Type2)` 或 多个`except`块 | 处理多种可能的错误 |
| **完整结构** | `try-except-else-finally` | 清理资源、无论异常与否都执行 |
| **自定义异常** | 继承`Exception`类创建新异常 | 特定领域错误类型 |

**最佳实践**：
- 尽量捕获具体的异常类型，而不是通用的`Exception`
- 在`finally`块中释放资源（文件、网络连接等）
- 自定义异常提供更清晰的错误信息和处理逻辑

### 选择题（每题5分，共20分）

**1. 关于文件操作，以下哪种说法是正确的？**

A. 使用`with open()`语句时，不需要手动调用`close()`方法  
B. 模式`"r+"`用于读取文件，不能写入  
C. 二进制文件使用`"w"`模式打开最合适  
D. `readlines()`方法返回的是一个字符串列表，每行包含换行符

**正确答案：A**  
**解析**：`with`语句是上下文管理器，会自动关闭文件，无需手动调用`close()`方法。  
**错误选项分析**：  
- B：`"r+"`模式支持读写，不只是读取  
- C：二进制文件应使用`"wb"`模式  
- D：`readlines()`返回的字符串包含换行符，这是正确的描述，但A是更重要的特性

**2. 异常处理的最佳实践是？**

A. 总是捕获通用的`Exception`类型，避免遗漏  
B. 在`try`块中尽量少放代码，只放可能出错的语句  
C. `finally`块只在需要清理资源时使用  
D. 自定义异常应该直接继承`BaseException`

**正确答案：B**  
**解析**：`try`块应只包含可能出错的代码，避免将正常代码也包含在内，提高代码可读性和维护性。  
**错误选项分析**：  
- A：应尽量捕获具体异常类型，避免隐藏潜在错误  
- C：`finally`块应始终用于确保资源释放，无论是否发生异常  
- D：自定义异常应该继承`Exception`类，而不是`BaseException`

**3. 关于Python模块导入，错误的是？**

A. `import numpy as np` 是为模块起别名  
B. `from math import *` 导入所有函数到当前命名空间  
C. `sys.path` 列表决定了Python查找模块的顺序  
D. `__init__.py` 文件在Python 3.8+版本中可有可无

**正确答案：D**  
**解析**：在Python 3.3+版本中，`__init__.py`文件对于标识包目录仍然需要，但可以是空文件。但在包中如果有特殊需求（如初始化代码或定义`__all__`变量），仍需要该文件。  
**错误选项分析**：  
- A：正确，这是常见的导入别名用法  
- B：正确，但不推荐在生产代码中使用  
- C：正确，`sys.path`是模块搜索路径列表

**4. 面向对象编程中，关于继承的说法正确的是？**

A. 单继承只能继承一个方法，不能继承属性  
B. `super()`函数用于调用父类的构造函数，不能在普通方法中使用  
C. 方法重写时，子类方法的参数列表必须与父类方法相同  
D. 多继承时，方法解析顺序完全由继承顺序决定

**正确答案：C**  
**解析**：方法重写（override）要求子类方法具有相同的名称和参数列表。  
**错误选项分析**：  
- A：单继承继承整个类，包括所有属性和方法  
- B：`super()`可以在任何方法中调用父类的方法，不仅限于构造函数  
- D：MRO使用C3算法，不仅考虑继承顺序，还考虑类层次关系

### 判断题（每题3分，共15分）

**5. 实例属性是属于具体对象的，不同对象的实例属性互不影响。**

**正确答案：✓ 正确**  
**解析**：实例属性是每个对象独有的，存储在对象自身的内存空间中，不同对象的同名实例属性是独立的。

**6. 在Python中，使用双下划线开头命名属性（如`__name`）可以完全防止外部访问。**

**正确答案：✗ 错误**  
**解析**：双下划线开头的属性会进行名称改写（name mangling），但不能完全防止访问。可以通过`对象._类名__属性名`的方式访问。

**7. `__str__()`方法返回的字符串主要用于调试，而`__repr__()`方法返回的字符串用于用户友好的显示。**

**正确答案：✗ 错误**  
**解析**：恰恰相反。`__str__()`返回用户友好的字符串，`__repr__()`返回开发者友好的字符串，应尽量返回能重建对象的表达式。

**8. 鸭子类型是指只要一个对象有`__call__()`方法，就可以被当作函数调用。**

**正确答案：✗ 错误**  
**解析**：鸭子类型是指"如果它走起来像鸭子，叫起来像鸭子，那么它就是鸭子"。在Python中，只要对象有相应的方法，就可以被当作某种类型使用，不限于`__call__()`方法。

**9. 使用`pip freeze > requirements.txt`可以生成当前环境所有依赖包的列表。**

**正确答案：✓ 正确**  
**解析**：这是pip的标准用法，用于记录项目依赖，便于在其他环境中复现。

### 代码补全题（15分）

**10. 补全以下银行账户类的代码，实现存款和取款功能：**

```python
class BankAccount:
    def __init__(self, account_holder, initial_balance=0):
        self.account_holder = account_holder
        self.__balance = initial_balance  # 私有属性
    
    def deposit(self, amount):
        # 补全存款方法
        if amount <= 0:
            print("存款金额必须大于0！")
            return False
        
        # 请在此处补全代码
        self.__balance += amount
        print(f"存款成功！当前余额：¥{self.__balance}")
        return True
    
    def withdraw(self, amount):
        # 补全取款方法
        if amount <= 0:
            print("取款金额必须大于0！")
            return False
        
        if amount > self.__balance:
            # 请在此处补全代码
            print(f"余额不足！当前余额：¥{self.__balance}")
            return False
        
        # 请在此处补全代码
        self.__balance -= amount
        print(f"取款成功！当前余额：¥{self.__balance}")
        return True
    
    def get_balance(self):
        # 返回当前余额
        return self.__balance
```

**解析要点**：
1. 存款方法的核心：`self.__balance += amount`
2. 取款前的余额检查：`amount > self.__balance`
3. 取款方法的核心：`self.__balance -= amount`
4. 私有属性的正确访问方式
5. 方法的返回值设计
