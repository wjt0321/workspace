| 知识点 | 关键内容 | 应用场景 |
|--------|----------|----------|
| **闭包三要素** | 嵌套函数、外部变量引用、返回内部函数 | 状态保持、函数工厂、记忆功能 |
| **装饰器本质** | 接收函数作为参数，返回新函数的可调用对象 | 日志记录、性能计时、权限验证 |

### 选择题（每题5分，共20分）

**1. 关于Python闭包，以下哪种说法是正确的？**

A. 闭包函数不能访问外部函数的变量  
B. 闭包会延长外部函数变量的生命周期  
C. 闭包只能用于装饰器实现  
D. 闭包会导致内存泄漏，应该避免使用

**正确答案：B**  
**解析**：闭包的核心特性是捕获并延长外部函数变量的生命周期，即使外部函数已经执行完毕，内部函数仍然可以访问这些变量。  
**错误选项分析**：  
- A：闭包的核心就是能够访问外部函数变量  
- C：闭包有独立的应用场景，如函数工厂、状态保持，不限于装饰器  
- D：合理使用闭包不会导致内存泄漏，不当使用才可能有问题

**2. 生成器与普通函数的主要区别是？**

A. 生成器使用`def`定义，普通函数使用`gen`定义  
B. 生成器只能使用`return`返回值  
C. 生成器使用`yield`暂停执行并返回值，下次从暂停处继续  
D. 生成器必须包含至少一个循环结构

**正确答案：C**  
**解析**：`yield`关键字使函数成为生成器，它可以在执行过程中暂停，下次调用时从暂停处继续执行，这是生成器的核心特性。  
**错误选项分析**：  
- A：生成器同样使用`def`定义，区别在于使用`yield`  
- B：生成器使用`yield`而不是`return`返回值  
- D：生成器不一定需要循环，简单的`yield`语句即可

**3. NumPy广播机制的主要作用是？**

A. 将数组从CPU传输到GPU加速计算  
B. 自动扩展不同形状数组以进行逐元素运算  
C. 在多核CPU上并行执行数组运算  
D. 将数组数据广播到网络上的其他计算机

**正确答案：B**  
**解析**：广播机制是NumPy的核心特性，它允许不同形状的数组进行逐元素运算，自动扩展较小数组的维度以匹配较大数组。  
**错误选项分析**：  
- A：描述的是GPU计算，不是广播机制  
- C：描述的是并行计算，广播是形状扩展  
- D：描述的是网络通信，与广播机制无关

**4. 在Pandas中，以下哪种方法最适合处理缺失值？**

A. 直接删除包含缺失值的行  
B. 使用`fillna()`填充缺失值  
C. 忽略缺失值，继续进行分析  
D. 将缺失值替换为字符串"missing"

**正确答案：B**  
**解析**：`fillna()`是Pandas专门用于填充缺失值的方法，支持多种填充策略（如均值、中位数、前向填充等），是最灵活和常用的缺失值处理方法。  
**错误选项分析**：  
- A：直接删除可能丢失重要数据，不是最佳选择  
- C：忽略缺失值可能导致分析错误  
- D：将数值型缺失值替换为字符串会改变数据类型，影响后续分析

### 判断题（每题3分，共12分）

**5. 装饰器本质上是一个接收函数作为参数，并返回新函数的高阶函数。**

**正确答案：✓ 正确**  
**解析**：装饰器的核心定义就是高阶函数：它接收一个函数作为输入，返回一个新的函数（通常是原函数的增强版本）。装饰器语法糖`@decorator`只是简化了这个过程。

**6. 生成器表达式`(x**2 for x in range(10))`会立即计算所有平方值并存储在内存中。**

**正确答案：✗ 错误**  
**解析**：生成器表达式是惰性求值的，它不会立即计算所有值，而是在迭代时逐个生成，内存占用恒定，与范围大小无关。

**7. NumPy数组的`reshape()`方法总会返回一个新数组，不会修改原数组。**

**正确答案：✓ 正确**  
**解析**：`reshape()`默认返回数组的新视图（共享数据），但如果新形状与原始数组不兼容（如要求连续内存），则可能返回副本。无论如何，原数组内容不变。

**8. 在机器学习中，训练集用于评估模型的最终性能，测试集用于调整模型参数。**

**正确答案：✗ 错误**  
**解析**：恰恰相反。训练集用于训练模型参数，验证集用于调整超参数，测试集用于评估最终性能，确保评估的客观性。

### 代码补全题（每题9分，共18分）

**9. 补全以下装饰器代码，实现函数执行时间统计功能：**

```python
import time

def timer_decorator(func):
    def wrapper(*args, **kwargs):
        # 补全计时开始代码
        start_time = time.time()
        
        # 执行原函数
        result = func(*args, **kwargs)
        
        # 补全计时结束和打印代码
        end_time = time.time()
        execution_time = end_time - start_time
        print(f"函数 {func.__name__} 执行耗时: {execution_time:.4f} 秒")
        
        return result
    return wrapper

# 使用示例
@timer_decorator
def calculate_sum(n):
    return sum(range(n))

# 测试
print(calculate_sum(1000000))
```

**解析要点**：
1. 使用`time.time()`获取开始时间戳
2. 计算结束时间与开始时间的差值得到执行时间
3. 格式化输出函数名和执行时间，保留4位小数
4. 装饰器结构正确：定义高阶函数，内部定义包装函数
