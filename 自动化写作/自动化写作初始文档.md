🚀 项目代号：智能美文策展人 (Smart Curator)
1. 项目愿景 (Project Vision)
打造一个私人专属的**“顶级杂志主编”**。它不仅仅是收集素材，更是对信息进行“精装修”。
 * 目标：将散落在网络上的优质文章（素材），转化为排版精美、易于阅读的 Markdown 笔记，自动归档至 Obsidian 知识库。
 * 核心价值：从“囤积链接”升级为“享受阅读”，建立高质量的个人知识储备。
2. 架构演进 (Architecture Evolution)
经过讨论，我们放弃了“辅助写作/改写”的路线，确立了**“策展与阅读体验”**为核心。
最终选定的架构 (V2.0)：
 * 输入端 (Input)：
   * A. 主动搜 (Keyword Search)：输入关键词，AI 自动寻找高质量文章。（未来迭代）
   * B. 链接洗 (Link Cleaning)：输入 URL，AI 自动提取纯净正文。（未来迭代）
 * 处理端 (Processing)：
   * 角色：Gemini 被设定为“杂志主编”。
   * 任务：阅读理解 -> 提取金句 -> 智能分段 -> Markdown 美化排版。
 * 输出端 (Storage)：
   * 目的地：本地 Obsidian 仓库 (D:\workspace\AI_Articles)。
   * 格式：自动命名、带元数据（标签、阅读时长）的 .md 文件。

4. 核心代码原型 (Prototype V0.1)
此脚本实现了**“文本美化 -> 本地归档”**的最小闭环。
环境要求：
 * Python 3.x
 * 库：google-generativeai
 * Obsidian 仓库路径：D:\workspace
代码逻辑：
 * 配置：填入 API Key 和本地路径。
 * AI 处理：将乱序文本发送给 Gemini，要求按“杂志主编”风格返回 Markdown。
 * 解析：提取第一行作为标题，剩余部分作为正文。
 * 保存：自动在 AI_Articles 目录下生成 .md 文件。
(完整代码请参考对话历史中的 curator.py 示例)
5. 提示词工程 (System Prompt)
为了保证输出质量，我们设计了如下系统指令：
> Role: 你是一个顶级杂志主编。
> Task: 将【原始文本】重新排版为优美的 Markdown 格式。
> Requirements:
>  * 阅读指引 (Callout)：开头必须包含预计阅读时间、3个核心标签、一句金句。
>  * 排版美化：使用二级标题分层，加粗重点，增加段落呼吸感。
>  * 输出规范：第一行纯标题，之后为正文，无其他废话。
>
