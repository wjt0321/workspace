# china-stock-analyst 幻觉问题审查报告

> 审查日期：2026年3月13日

## 一、问题概述

今天生成的股票报告中，推荐的 TOP3 股票股价出现严重错误：

| 股票 | 报告中的股价 | 问题描述 |
|:---|:---|:---|
| **中国电建** (601669) | 7.19元（涨停） | 区间价格被误用 |
| **中国能建** (601868) | 3.23元（涨停） | 可能是历史价或预测价 |
| **节能风电** (601016) | 3.10-4.03元 | 范围过大，无确定值 |

---

## 二、根本原因分析

### 1. 数据获取层的问题（SKILL.md 第 97-110 行）

技能文档虽然规定了数据提取原则：
- 优先使用"股票名称 + 股票代码 + 最新价"关键词组合
- **仅提取**"最新价/现价/收盘价/报价"后的价格
- **禁止**将止损位、历史价、目标价写入当前价格字段

**但实际执行中存在漏洞：**
- 搜索时未强制要求"今日/最新"日期限定
- 搜索结果中常混有"预测目标价"、"止损位"、"压力位"等价格
- 当多个价格同时出现时，提取规则不够精确

### 2. 专家鉴别 Agent 流于形式（stock-identity-auditor.md）

该 Agent 本应负责"代码-名称-价格一致性校验，发现冲突即触发阻断或降级"，但从报告第 206-218 行看：

```
| 中国电建 | ✅ 通过 | 7.19元 < 10元 | 无 |
| 节能风电 | ✅ 通过 | 3.10~4.03元 < 10元 | 无 |
| 中国能建 | ✅ 通过 | 3.23元 < 10元 | 无 |
```

**问题在于：**
- 它只验证了"股价 < 10元"这个条件
- **没有验证股价是否与当日真实收盘价匹配**
- **没有校验价格区间的合理性**（如 3.10~4.03 元范围过大）
- 缺少实时数据源的交叉验证

### 3. 报告生成层的校验缺失（generate_report.py）

查看代码结构发现：
- 数据解析函数存在
- 但缺少**强制校验环节**：提取的股价必须与当日收盘价一致才能写入报告
- 缺失时应该降级为"观察"或标注"数据待核实"

## 三、具体错误案例分析

### 案例1：节能风电（601016）

**报告数据**：3.10-4.03元（涨停）
**问题**：价格区间过大（相差近1元），说明：
- 系统可能提取了多个价格（开盘价、最高价、最低价等）
- 未明确选择"收盘价/最新价"

### 案例2：中国电建（601669）

**报告数据**：7.19元（涨停）
**问题**：
- 7.19元是当天涨停价，但未标注是否为收盘价
- 未提供这一价格的数据来源和时间戳

### 案例3：中国能建（601868）

**报告数据**：3.23元（涨停）
**问题**：
- 需要核实这是否为当日真实收盘价
- 可能是历史数据或估算价格

## 四、修复建议

### 建议1：强化数据获取（高优先级）

```markdown
# 搜索关键词必须包含：
"股票代码 + 今日收盘价" 或 "股票代码 + 最新价 + 2026年3月13日"

# 示例：
"601669 中国电建 今日收盘价 2026年3月13日"
"601868 中国能建 最新价 今日"
```

### 建议2：增强专家鉴别 Agent 校验规则（中优先级）

stock-identity-auditor.md 应增加：
- **收盘价校验**：获取后与权威来源（东方财富/同花顺）交叉验证
- **价格合理性检查**：单日波动超过 15% 需要确认是否为涨停/跌停
- **区间检查**：价格区间范围不超过 0.5 元，否则标记为"数据不确切"

### 建议3：报告生成层强制校验（高优先级）

generate_report.py 应增加：
```python
def _validate_price_consistency(stock_code, extracted_price):
    # 1. 检查价格是否在合理范围（与昨日收盘价偏差 > 20% 需警告）
    # 2. 检查是否有多个价格混用（开盘/最高/最低/收盘）
    # 3. 缺失确切价格时，标签上限为"观察"
```

## 五、结论

**问题根源**：不是单一环节的失误，而是**数据获取 → 专家校验 → 报告生成**三层都存在校验漏洞，导致错误数据层层通过。

**最核心的问题**：
1. 搜索时未强制限定"今日/最新收盘价"
2. 专家鉴别 Agent 只做了形式化的" < 10元"检查，没有真正校验价格真实性
3. 报告生成层缺少强制的数据一致性校验
