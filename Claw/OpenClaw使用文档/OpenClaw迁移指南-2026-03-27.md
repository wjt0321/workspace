# OpenClaw 跨平台迁移指南（Windows / macOS / Linux）

> 适用版本：OpenClaw 中文分支（`openclaw-cn`）
> 更新日期：2026-03-30
> 整理人：旺财（首席大祭司）

---

## 前言

这是一份从旧机器迁移到新机器的实战手册，覆盖 Windows、macOS、Linux 三个平台。  
迁移目标是：配置可复用、通道可恢复、定时任务可继续、工作区不丢失。

---

## 一、先搞清楚要迁移什么

| 类型 | Linux/macOS 默认路径 | Windows 默认路径 | 迁移方式 |
|------|----------------------|------------------|----------|
| 核心配置 | `~/.openclaw/openclaw.json` | `%USERPROFILE%\.openclaw\openclaw.json` | 直接复制 |
| 技能库 | `~/.openclaw/skills/` | `%USERPROFILE%\.openclaw\skills\` | 直接复制 |
| 会话记录 | `~/.openclaw/agents/` | `%USERPROFILE%\.openclaw\agents\` | 直接复制 |
| 定时任务 | `~/.openclaw/cron/` | `%USERPROFILE%\.openclaw\cron\` | 直接复制 |
| 工作空间 | `~/clawd/` | `%USERPROFILE%\clawd\` | 直接复制 |
| 插件编译产物 | `app/extensions/*/dist/` | `app\extensions\*\dist\` | 需要在新机重新编译 |

> 最常见坑：插件只复制源码但没编译 `dist`，导致飞书等渠道插件加载失败，cron 推送静默失败。

## 二、迁移前准备（旧机器执行）

### 2.1 记录当前状态（必须）

#### Linux/macOS

```bash
openclaw-cn --version
openclaw-cn status --all
openclaw-cn channels status --probe
openclaw-cn cron list
```

#### Windows（PowerShell）

```powershell
openclaw-cn --version
openclaw-cn status --all
openclaw-cn channels status --probe
openclaw-cn cron list
```

### 2.2 停止网关，防止迁移中写入

#### Linux/macOS

```bash
openclaw-cn gateway stop
```

#### Windows（PowerShell）

```powershell
openclaw-cn gateway stop
```

### 2.3 打包迁移数据

#### Linux/macOS（推荐 tar 打包）

```bash
mkdir -p ~/openclaw-migrate-backup
tar -czf ~/openclaw-migrate-backup/openclaw-backup-$(date +%Y%m%d).tar.gz \
  ~/.openclaw \
  ~/clawd
```

#### Windows（PowerShell 压缩）

```powershell
New-Item -ItemType Directory -Force "$env:USERPROFILE\openclaw-migrate-backup" | Out-Null
Compress-Archive -Path "$env:USERPROFILE\.openclaw","$env:USERPROFILE\clawd" `
  -DestinationPath "$env:USERPROFILE\openclaw-migrate-backup\openclaw-backup.zip" -Force
```

## 三、目标机器环境安装（按系统）

### 3.1 Linux 安装

```bash
# 安装 Node.js 24+
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
export NVM_DIR="$HOME/.nvm"
source "$NVM_DIR/nvm.sh"
nvm install 24
nvm use 24

# 安装 OpenClaw 中文版
npm i -g openclaw-cn@latest --registry=https://registry.npmmirror.com
```

### 3.2 macOS 安装

```bash
# 安装 Node.js 24+
brew install nvm
mkdir -p ~/.nvm
export NVM_DIR="$HOME/.nvm"
source "$(brew --prefix nvm)/nvm.sh"
nvm install 24
nvm use 24

# 安装 OpenClaw 中文版
npm i -g openclaw-cn@latest --registry=https://registry.npmmirror.com
```

### 3.3 Windows 安装

#### 方案 A（推荐）：WSL2 + Ubuntu（兼容性最好）

```powershell
wsl --install
wsl --install -d Ubuntu-24.04
wsl --shutdown
```

然后进入 Ubuntu，按 Linux 章节安装。

#### 方案 B：Windows 原生 PowerShell

1. 安装 Node.js 24+（官网安装包或 nvm-windows）。
2. 打开新 PowerShell，执行：

```powershell
npm i -g openclaw-cn@latest --registry=https://registry.npmmirror.com
```

## 四、恢复数据到新机器

### 4.1 Linux/macOS 恢复

```bash
mkdir -p ~/
tar -xzf /你的备份路径/openclaw-backup-YYYYMMDD.tar.gz -C /
```

如果你是通过目录复制而不是 tar：

```bash
cp -a /备份目录/.openclaw ~/
cp -a /备份目录/clawd ~/
```

### 4.2 Windows 恢复（PowerShell）

```powershell
Expand-Archive -Path "D:\backup\openclaw-backup.zip" -DestinationPath "$env:USERPROFILE" -Force
```

## 五、插件重编译（关键步骤）

迁移后必须在新机器上重新编译你实际使用的插件：

```bash
cd /app/extensions/feishu
npm install
npx tsc
```

其他插件同理：

```bash
cd /app/extensions/qqbot && npx tsc
cd /app/extensions/discord && npx tsc
```

验证编译产物：

```bash
ls /app/extensions/feishu/dist/
```

若 `dist/` 为空或不存在，渠道会加载失败。

## 六、重启与验证（新机器执行）

### 6.1 重启网关

```bash
openclaw-cn gateway restart
```

### 6.2 运行诊断

```bash
openclaw-cn doctor
openclaw-cn status --all
openclaw-cn channels status --probe
```

### 6.3 验证定时任务

```bash
openclaw-cn cron list
```

如果你需要巡检投递状态：

```bash
for f in ~/.openclaw/cron/runs/*.jsonl; do
  job=$(basename "$f" .jsonl)
  last=$(tail -1 "$f" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('deliveryStatus'), d.get('status'))")
  echo "$job: $last"
done
```

理想状态应为 `delivered ok`。

## 七、Windows / macOS / Linux 差异与配置建议

### 7.1 路径差异

- Linux/macOS 使用 `~/.openclaw` 与 `~/clawd`
- Windows 使用 `%USERPROFILE%\.openclaw` 与 `%USERPROFILE%\clawd`

### 7.2 服务管理差异

- Linux：通常用 `systemd --user`
- macOS：通常由 launchd 或应用托管
- Windows：建议 WSL2 环境运行，避免服务控制行为差异

### 7.3 网络与防火墙

- 放行网关端口（默认 `18789`）
- 若启用控制 UI，建议仅允许局域网或反向代理入口访问

### 7.4 编码与换行

- Windows 原生环境可能遇到 CRLF / UTF-8 问题
- 建议统一 UTF-8，并在脚本仓库中固定 `.gitattributes`

## 八、常见问题

### Q1：迁移后提示 `Unknown channel: feishu`

大概率是插件未编译或编译失败。先检查 `dist/`，再重启网关。

### Q2：Windows 下 `gateway restart` 报服务相关错误

建议切到 WSL2 方案运行，兼容性更稳定。

### Q3：Git 推送提示认证失败

重新配置 PAT，并确认凭证存储正常。

### Q4：cron 都在跑，但没有消息送达

优先检查渠道插件是否加载成功，再检查目标用户 ID 类型和权限配置。

## 九、迁移后必做清单

1. 运行 `openclaw-cn doctor`
2. 运行 `openclaw-cn channels status --probe`
3. 实测每个渠道至少一条收发消息
4. 检查关键 cron 任务一次完整投递
5. 检查工作区 Git fetch/push
6. 保留最近一次可用备份至少 7 天
