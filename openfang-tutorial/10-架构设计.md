# OpenFang 架构设计

## 概述

OpenFang 采用**模块化内核设计**，由 14 个 Rust Crate 组成，总计 137,728 行代码。

> [!important] 设计原则
> - **模块化**：每个 Crate 职责单一
> - **高性能**：Rust 原生实现
> - **可扩展**：清晰的接口边界
> - **安全优先**：零信任架构

## 14个 Crate 详解

### 1. openfang-kernel

**职责**：核心编排层

```
openfang-kernel
├── orchestration    # Agent协调
├── workflows        # 工作流引擎
├── metering         # 计量系统
├── rbac             # 基于角色的访问控制
├── scheduler        # 任务调度
└── budget_tracking  # 预算追踪
```

**主要功能**：
- Agent 生命周期管理
- 工作流编排
- 权限控制
- 定时任务调度
- Token/成本预算管理

**职责**：消息通道适配器

## 扩展机制

### 自定义工具

```rust
// 实现 Tool trait
impl Tool for MyCustomTool {
    fn name(&self) -> &str { "my_tool" }
    
    fn description(&self) -> &str {
        "My custom tool description"
    }
    
    fn parameters(&self) -> Vec<Parameter> {
        vec![
            Parameter::required("input", "Input parameter")
        ]
    }
    
    async fn execute(&self, params: Value) -> Result<Value> {
        // 工具逻辑
        Ok(json!({"result": "success"}))
    }
}

// 注册工具
runtime.register_tool(Box::new(MyCustomTool::new()));
```

### 自定义通道

```rust
// 实现 ChannelAdapter trait
impl ChannelAdapter for MyChannel {
    async fn send(&self, message: Message) -> Result<()> {
        // 发送逻辑
        Ok(())
    }
    
    async fn receive(&self) -> Result<Vec<Message>> {
        // 接收逻辑
        Ok(vec![])
    }
    
    fn capabilities(&self) -> ChannelCapabilities {
        ChannelCapabilities {
            supports_markdown: true,
            supports_files: true,
            max_message_length: 4096,
        }
    }
}
```
