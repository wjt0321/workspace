# OpenFang 开发指南

## 开发环境设置

### 系统要求

| 工具 | 版本要求 |
|------|----------|
| Rust | 1.75+ |
| Cargo | 最新版 |
| Git | 2.0+ |

### 克隆仓库

```bash
git clone https://github.com/RightNow-AI/openfang.git
cd openfang
```

### 安装依赖

```bash
# 安装 Rust（如果尚未安装）
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 更新到最新稳定版
rustup update stable
```

---

## 构建

### 构建工作空间

```bash
# 构建所有 Crate
cargo build --workspace --lib

# 构建发布版本
cargo build --workspace --release
```

### 构建特定 Crate

```bash
# 只构建 kernel
cargo build -p openfang-kernel

# 只构建 runtime
cargo build -p openfang-runtime
```

## 测试

### 运行所有测试

```bash
# 运行全部测试（1,767+）
cargo test --workspace
```

### 运行特定测试

```bash
# 运行特定 Crate 的测试
cargo test -p openfang-kernel

# 运行特定测试
cargo test test_agent_lifecycle

# 运行带标记的测试
cargo test --workspace -- --ignored  # 运行被忽略的测试
```

### 测试覆盖率

```bash
# 安装 tarpaulin
cargo install cargo-tarpaulin

# 生成覆盖率报告
cargo tarpaulin --workspace --out Html
```

## 编码规范

### Rust 风格指南

遵循官方 Rust 风格指南：

```rust
// 函数命名：snake_case
fn calculate_total_price(items: &[Item]) -> u32 {
    // ...
}

// 类型命名：PascalCase
struct UserSession {
    id: SessionId,
    created_at: DateTime<Utc>,
}

// 常量命名：SCREAMING_SNAKE_CASE
const MAX_RETRIES: u32 = 3;

// 文档注释
/// 计算总价
/// 
/// # Arguments
/// * `items` - 商品列表
/// 
/// # Returns
/// 总价格（分）
fn calculate_total_price(items: &[Item]) -> u32 {
    // ...
}
```

### 错误处理

使用 `thiserror` 定义错误类型：

```rust
use thiserror::Error;

#[derive(Error, Debug)]
pub enum AgentError {
    #[error("Agent not found: {0}")]
    NotFound(String),
    
    #[error("Invalid configuration: {0}")]
    InvalidConfig(String),
    
    #[error("LLM error: {0}")]
    LlmError(#[from] LlmError),
}
```

### 异步代码

```rust
// 使用 tokio 运行时
#[tokio::main]
async fn main() -> Result<()> {
    // ...
}

// 异步函数
async fn process_message(msg: Message) -> Result<Response> {
    // ...
}
```

## 提交代码

### 分支策略

```
main        # 主分支，稳定版本
├── develop # 开发分支
│   ├── feature/xxx  # 功能分支
│   ├── fix/xxx      # 修复分支
│   └── docs/xxx     # 文档分支
```

### 提交消息格式

```
<type>(<scope>): <description>

[optional body]

[optional footer]
```

**类型**：
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具

**示例**：

```
feat(runtime): add support for Ollama provider

- Implement Ollama API client
- Add configuration options
- Include integration tests

Closes #123
```

### Pull Request 流程

1. Fork 仓库
2. 创建功能分支
3. 提交更改
4. 运行测试和 Clippy
5. 创建 Pull Request

```bash
# 1. Fork 并克隆
git clone https://github.com/YOUR_USERNAME/openfang.git

# 2. 创建分支
git checkout -b feature/my-feature

# 3. 提交更改
git add .
git commit -m "feat: add my feature"

# 4. 运行检查
cargo test --workspace
cargo clippy --workspace --all-targets -- -D warnings
cargo fmt --all -- --check

# 5. 推送并创建 PR
git push origin feature/my-feature
```

## 调试

### 日志

使用 `tracing` 进行日志记录：

```rust
use tracing::{info, warn, error, debug};

fn process_request(req: Request) {
    debug!("Processing request: {:?}", req);
    
    match handle(req) {
        Ok(result) => info!("Request processed successfully"),
        Err(e) => error!("Request failed: {}", e),
    }
}
```

### 日志级别

```bash
# 设置日志级别
RUST_LOG=debug cargo run

# 特定模块日志
RUST_LOG=openfang_runtime=trace cargo run
```

### 调试构建

```bash
# 构建调试版本
cargo build

# 运行调试版本
./target/debug/openfang --debug
```

## 发布

### 版本号规则

遵循语义化版本：

```
MAJOR.MINOR.PATCH

MAJOR: 不兼容的API变更
MINOR: 向后兼容的功能新增
PATCH: 向后兼容的问题修复
```

### 发布检查清单

- [ ] 更新版本号
- [ ] 更新 CHANGELOG
- [ ] 运行所有测试
- [ ] 运行 Clippy
- [ ] 检查文档
- [ ] 创建 Git 标签
- [ ] 构建发布版本
- [ ] 发布到 GitHub Releases

```bash
# 发布流程
cargo test --workspace
cargo clippy --workspace --all-targets -- -D warnings
cargo build --workspace --release

git tag v0.1.1
git push origin v0.1.1
```
