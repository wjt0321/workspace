# OpenFang 迁移指南

## 概述

OpenFang 提供完整的迁移工具，支持从多个主流 Agent 框架迁移。

> [!important] 支持的迁移源
> - OpenClaw
> - LangChain
> - AutoGPT

---

## 从 OpenClaw 迁移

### 一键迁移

```bash
# 迁移所有内容 — agents、memory、skills、configs
openfang migrate --from openclaw

# 从指定路径迁移
openfang migrate --from openclaw --path ~/.openclaw

# 预览模式（不实际执行）
openfang migrate --from openclaw --dry-run
```

### 迁移内容

| 内容 | 迁移状态 | 说明 |
|------|----------|------|
| Agents | ✅ 完全支持 | 配置和行为保持一致 |
| Memory | ✅ 完全支持 | 对话历史完整保留 |
| Skills | ✅ 完全支持 | SKILL.md 原生兼容 |
| Config | ✅ 完全支持 | 配置自动转换 |
| Channels | ✅ 完全支持 | 通道配置迁移 |

### 配置映射

```toml
# OpenClaw config.toml
[agent]
name = "assistant"
model = "gpt-4"

# OpenFang config.toml
[agent.default]
name = "assistant"
model = "gpt-4o"
```

### ClawHub 兼容

OpenFang 原生读取 SKILL.md 格式，与 ClawHub 市场完全兼容：

```bash
# ClawHub 技能可直接使用
openfang skill install https://clawhub.dev/skills/xxx

# 已安装的技能自动迁移
# ~/.openclaw/skills/ → ~/.openfang/skills/
```

### API 兼容

OpenFang 提供 OpenAI 兼容 API，现有工具无需修改：

```bash
# OpenClaw API
curl http://localhost:3000/v1/chat/completions

# OpenFang API（兼容）
curl http://localhost:4200/v1/chat/completions
```

## 从 LangChain 迁移

### 概念映射

| LangChain | OpenFang |
|-----------|----------|
| Chain | Hand |
| Agent | Agent |
| Tool | Tool |
| Memory | Memory (SQLite + Vector) |
| Document | Knowledge Entry |

### 迁移步骤

```bash
# 1. 导出 LangChain 配置
python export_langchain_config.py --output langchain_export.json

# 2. 迁移到 OpenFang
openfang migrate --from langchain --config langchain_export.json

# 3. 验证迁移结果
openfang agent list
openfang memory stats
```

### 代码迁移

**LangChain Python**:

```python
from langchain.agents import initialize_agent
from langchain.llms import OpenAI

llm = OpenAI(temperature=0)
agent = initialize_agent(tools, llm, agent="zero-shot-react-description")
agent.run("What is the weather?")
```

**OpenFang 等效**:

```bash
# CLI
openfang chat default
> What is the weather?

# 或使用 API
curl -X POST http://localhost:4200/v1/chat/completions \
  -d '{"model": "default", "messages": [{"role": "user", "content": "What is the weather?"}]}'
```

### Memory 迁移

```bash
# 导出 LangChain memory
python -c "
from langchain.memory import ConversationBufferMemory
import json
memory = ConversationBufferMemory()
# ... load memory ...
print(json.dumps(memory.load_memory_variables({})))
" > langchain_memory.json

# 导入到 OpenFang
openfang memory import --file langchain_memory.json --format langchain
```

## 从 AutoGPT 迁移

### 概念映射

| AutoGPT | OpenFang |
|---------|----------|
| Goal | Hand task |
| Command | Tool |
| Memory | Memory + Knowledge |
| Workspace | File storage |
| Plugin | Skill |

### 迁移步骤

```bash
# 1. 导出 AutoGPT 配置
python -m autogpt --export-config autogpt_export.json

# 2. 迁移到 OpenFang
openfang migrate --from autogpt --config autogpt_export.json

# 3. 验证
openfang hand list
```

### Goals → Hands 转换

AutoGPT Goals:

```json
{
  "goals": [
    "Research AI market trends",
    "Write a report",
    "Save to file"
  ]
}
```

OpenFang Hand 配置:

```toml
[hand]
name = "autogpt-migrated"

[hand.goals]
primary = "Research AI market trends and generate report"

[hand.settings]
output_format = "markdown"
save_results = true
```

## 常见问题

### Q: 迁移后原系统还能用吗？

> [!tip] 回答
> 可以。迁移是复制操作，不会修改原系统数据。可以同时运行两个系统进行对比测试。

### Q: 迁移失败怎么办？

```bash
# 查看详细错误
openfang migrate --from openclaw --verbose

# 只迁移部分内容
openfang migrate --from openclaw --only agents
openfang migrate --from openclaw --only memory
openfang migrate --from openclaw --only skills
```

### Q: 如何回滚？

```bash
# 迁移前自动备份
ls ~/.openfang/backups/

# 恢复备份
openfang restore --backup ~/.openfang/backups/pre-migration-20240315/
```

### Q: 配置差异如何处理？

```bash
# 查看配置差异
openfang config diff --source openclaw

# 手动合并
openfang config merge --source openclaw --strategy interactive
```
