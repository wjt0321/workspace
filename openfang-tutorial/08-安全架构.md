# OpenFang 安全架构

## 概述

OpenFang 不把安全作为事后补充——**每一层都可以独立测试，没有单点故障**。

> [!important] 安全理念
> 从零构建的16层安全防护，而非事后打补丁。

## 第一层：WASM 双计量沙箱

### 功能说明

工具代码在 WebAssembly 沙箱中运行，配备双重计量机制：

| 机制 | 说明 |
|------|------|
| **Fuel计量** | 限制指令执行数量 |
| **Epoch中断** | 时间片中断机制 |
| **看门狗线程** | 终止失控代码 |

### 工作原理

```
┌─────────────────────────────────────────────────────────┐
│              WASM Sandbox                               │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  Tool Code (WASM)                                       │
│       │                                                  │
│       ↓                                                  │
│  ┌─────────────┐     ┌─────────────┐                    │
│  │ Fuel Meter  │     │ Epoch Timer │                    │
│  │ 10M units   │     │  100ms slice│                    │
│  └──────┬──────┘     └──────┬──────┘                    │
│         │                   │                            │
│         └───────┬───────────┘                            │
│                 ↓                                        │
│         ┌─────────────┐                                  │
│         │ Watchdog    │                                  │
│         │ Thread      │                                  │
│         └──────┬──────┘                                  │
│                │                                         │
│                ↓                                         │
│         超时 → 强制终止                                  │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 配置

```toml
[sandbox]
enabled = true
backend = "wasm"

[sandbox.limits]
fuel = 10000000        # 最大指令数
epoch = "100ms"        # 时间片
memory = "64MB"        # 最大内存
```

## 第二层：Merkle 哈希链审计追踪

### 功能说明

每个操作都加密链接到前一个操作。篡改任何一个条目，整个链都会断裂。

### 工作原理

```
┌─────────────────────────────────────────────────────────┐
│              Merkle Hash Chain                          │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  Genesis Block                                          │
│  └─ hash: 0x000...                                      │
│                                                          │
│  Block 1                                                │
│  ├─ prev_hash: 0x000...                                 │
│  ├─ action: "web_search('AI')"                          │
│  ├─ timestamp: 2024-03-15T10:30:00Z                     │
│  └─ hash: SHA256(prev_hash + action + timestamp)        │
│                                                          │
│  Block 2                                                │
│  ├─ prev_hash: [Block 1 hash]                           │
│  ├─ action: "memory_store('result')"                    │
│  ├─ timestamp: 2024-03-15T10:30:05Z                     │
│  └─ hash: SHA256(prev_hash + action + timestamp)        │
│                                                          │
│  ...                                                    │
│                                                          │
│  篡改 Block 1 → 所有后续 Block hash 失效                 │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 验证审计链

```bash
# 验证审计链完整性
openfang audit verify

# 查看审计日志
openfang audit log --session <session-id>

# 导出审计日志
openfang audit export --format json
```

## 第三层：信息流污染追踪

### 功能说明

标签在执行过程中传播——从源头到终点追踪秘密数据。

### 工作原理

```
┌─────────────────────────────────────────────────────────┐
│              Taint Tracking                             │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  Source (API Key)                                       │
│  Label: SECRET                                          │
│       │                                                  │
│       ↓                                                  │
│  Variable Assignment                                    │
│  key = get_api_key()  →  key: SECRET                    │
│       │                                                  │
│       ↓                                                  │
│  String Operation                                       │
│  header = "Bearer " + key  →  header: SECRET            │
│       │                                                  │
│       ↓                                                  │
│  Sink Check                                             │
│  log(header)  →  ❌ BLOCKED: SECRET flowing to LOG sink │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 标签类型

| 标签 | 说明 |
|------|------|
| `SECRET` | API密钥、密码等敏感数据 |
| `USER_INPUT` | 用户输入数据（防注入） |
| `UNTRUSTED` | 外部数据源 |
| `SANITIZED` | 已清理的数据 |

## 第六层：秘密零化

### 功能说明

`Zeroizing<String>` 类型在不再需要时自动擦除内存中的API密钥。

### 工作原理

```rust
// 内部实现
struct Zeroizing<String> {
    data: Vec<u8>,
}

impl Drop for Zeroizing<String> {
    fn drop(&mut self) {
        // 用零覆盖内存
        for byte in self.data.iter_mut() {
            *byte = 0;
        }
        // 防止编译器优化
        std::sync::atomic::compiler_fence(Ordering::SeqCst);
    }
}
```

### 使用场景

```rust
// API密钥自动零化
let api_key = Zeroizing::new(get_api_key());
let client = Client::new(&api_key);
// api_key 离开作用域后自动擦除
```

```

│              OFP Mutual Authentication                  │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  Node A                        Node B                   │
│    │                             │                       │
│    │──── nonce_A ──────────────►│                       │
│    │                             │                       │
│    │◄──── nonce_B ──────────────│                       │
│    │                             │                       │
│    │──── HMAC(key, nonce_B) ───►│                       │
│    │                             │                       │
│    │◄──── HMAC(key, nonce_A) ───│                       │
│    │                             │                       │
│    │  恒定时间验证 ✓             │                       │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

## 第十一层：子进程沙箱

### 功能说明

`env_clear()` + 选择性变量传递。进程树隔离，跨平台终止。

### 工作原理

```rust
// 启动子进程
Command::new("tool")
    .env_clear()  // 清除所有环境变量
    .env("PATH", "/safe/path")  // 仅传递安全变量
    .env("HOME", "/sandbox/home")
    .spawn()
```

### 进程树隔离

```
┌─────────────────────────────────────────────────────────┐
│              Process Tree Isolation                     │
├─────────────────────────────────────────────────────────┤
│                                                          │

│      │                                                   │
│      ├── worker_1 (sandboxed)                           │
│      │       └── tool_process                           │
│      │                                                   │
│      ├── worker_2 (sandboxed)                           │
│      │       └── tool_process                           │
```

## 第十五层：路径遍历防护

### 功能说明

规范化路径，防止符号链接逃逸。`../` 在这里不起作用。

### 防护机制

```rust
fn validate_path(base: &Path, requested: &Path) -> Result<PathBuf> {
    // 1. 规范化路径
    let canonical = requested.canonicalize()?;
    
    // 2. 检查是否在基础目录内
    if !canonical.starts_with(base.canonicalize()?) {
        return Err(PathTraversalError);
    }
    
    // 3. 检查符号链接
    if is_symlink_escaping(&canonical, base)? {
        return Err(SymlinkEscapeError);
    }
    
    Ok(canonical)
}
```

### 攻击示例

```
请求: /safe/dir/../../../etc/passwd
规范: /etc/passwd
检查: ❌ 不在 /safe/dir 内

请求: /safe/dir/symlink_to_root/etc/passwd
检查: ❌ 符号链接逃逸
```

## 第十六层：GCRA 速率限制

```
├─────────────────────────────────────────────────────────┤

│  └─ 每Agent: 50 请求/分钟                               │
│                                                          │
│  成本感知:                                              │
│  ├─ web_search: 成本 1                                  │
│  ├─ llm_call: 成本 10                                   │

│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 配置

```toml
[rate_limit]
enabled = true

[rate_limit.global]
requests = 1000
period = "1m"

[rate_limit.per_ip]
requests = 100
period = "1m"

[rate_limit.per_agent]
requests = 50
period = "1m"

[rate_limit.costs]

llm_call = 10
file_write = 5

```

## 安全最佳实践

### 1. 最小权限原则



```bash

openfang audit export --format json
```

```bash
# 查看被阻止的操作
openfang audit events --blocked

# 查看速率限制命中
openfang stats rate-limits
```
