# 📚 美文收集标准流程

## 📅 版本信息
- **版本**：1.0.0
- **创建日期**：2025-12-15
- **适用范围**：小学生美文、初中生美文、高中生美文、大学生美文、成人美文
- **创建人**：AI Assistant
- **使用软件**：Obsidian

## 🌟 核心原则
1. **严格遵循工作流程**：不可跳过或颠倒顺序
2. **强制执行模版**：100%符合对应级别的通用模版
3. **保证内容质量**：收录原文完整版，执行三步验证法
4. **建立完整链接**：片段与完整文章双向链接
5. **定期质量检查**：确保所有文章符合标准
6. **🚨 总索引更新铁律**：任何收录工作完成后都必须更新`美文总索引.md`，这是不可违背的强制要求
7. **基础四项检查**：任何文件操作必须进行以下四项正确性检查：
   - 序号连续准确性
   - 文章到片段、片段到文章的双向链接
   - 索引正确性
   - 文章清洗去重
   *（注：去重可在过程中进行，但相同文章清除后必须补足数量）*

## 🏗️ 完整工作流程（8个步骤）

> **⚠️ 重要提醒**：Step 8（总索引更新）是强制性的最后步骤，任何收录工作都必须以更新`美文总索引.md`作为结束，这是不可违背的铁律！

### Step 1️⃣: 收集片段 (prose-snippet-collector)

#### 📚 执行内容
- 创建或更新 `{级别}美文精选100篇.md` 或 `{级别}美文精选100篇（二）.md` 等续辑
- 收集N个经典片段（小学150-300字，初中200-400字，高中300-500字）
- 每个片段包含：序号、作品名、作者、内容引用、推荐理由

#### ✅ 输出格式
```markdown
#### **{序号}. 《{作品名}》 - {作者}**

> {开头Emoji组合}
> 
> {穿插Emoji的片段内容...}
> 
> {如果跨段,继续引用...}

**【推荐理由】** {50字以内的推荐语}

[[完整文章的文件名]] {如果已收录完整版,添加Obsidian双向链接}

---
```

#### 🎯 关键点
- **开头Emoji组合**：2-3个主题相关Emoji
- **正文穿插Emoji**：每2-3句话1个，与内容高度贴合
- **片段长度**：小学150-300字，初中200-400字，高中300-500字
- **必须来自近现代中文散文大家**
- **避免重复**：检查作品名+作者组合

#### 📂 输出文件
- `d:\workspace\{级别}美文\{级别}美文精选100篇.md` 或 `{级别}美文精选100篇（二）.md`

### Step 2️⃣: 更新索引 (prose-index-manager)

#### 📚 执行内容
- 更新 `美文收集索引.md`
- 记录片段列表
- 更新统计信息

#### ✅ 输出格式
```markdown
## 📊 收录统计

- **总计篇数**：{数量}篇
- **已完成**：{数量}篇
- **待补充**：{数量}篇
- **最后更新**：{YYYY-MM-DD}

## 📖 片段列表

### 🌟 已收录片段

#### **{序号}. 《{作品名}》 - {作者}**
- **文件位置**：`{文件名}.md`
- **行号范围**：{行号}
- **主题**：{主题}
- **特色**：{特色}
```

#### 🎯 关键点
- 记录片段序号和作品信息
- 更新收录统计（总计篇数、片段数、完整文章数）
- 标记状态（待收集完整版）

#### 📂 输出文件
- `d:\workspace\{级别}美文\美文收集索引.md`

### Step 3️⃣: 搜索原文 (prose-fulltext-hunter)

#### 📚 执行内容
- 为每个片段搜索完整版原文
- **强制执行三步验证法**：
  1. 搜索完整版关键词
  2. **必须**搜索"课文与原文区别"
  3. 版本验证（字数、结构、特征）

#### ✅ 核查清单
```markdown
- [ ] 已搜索"课文与原文区别"
- [ ] 已确认是否存在删改
- [ ] 已对比字数
- [ ] 确认为完整版 ✅
```

#### 🎯 关键点
- **铁律**：宁可不收录，也不能收录删减版
- 高风险作品必查：丰子恺《白鹅》、郑振铎《燕子》等
- 优先选择文集版本

#### 📂 输出
- 版本核查报告（记录在笔记中）
- 完整版原文内容

### Step 4️⃣: 格式化文章 (prose-article-formatter)

#### 📚 执行内容
- **必须先读取对应级别的模版文件**
- 严格按照模版创建完整文章
- 100%符合模版要求（YAML+Emoji+所有板块）

#### ✅ 小学/初中/高中/大学/成人模版共通要求
1. **YAML 元数据**：必须包含title、source、created、tags
2. **标题区**：H1含Emoji+完整版标注，H3含作者
3. **原文正文**：
   - 开头添加2-3个主题相关Emoji组合
   - 使用H3 + Emoji分段
   - 正文穿插Emoji，密度适中（每2-3句话1个）
   - 分段逻辑清晰
4. **核心板块**：根据级别包含对应板块
5. **文件命名**：`{序号}.{标题}-{作者}.md`

#### 🎯 关键点
- **铁律**：任何新收录文章必须100%符合模版，不得缺项
- 必须包含所有Emoji
- YAML必须包含：title、source、created、tags、完整版标签
- 原文标注字数

#### 📂 输出文件
- `d:\workspace\{级别}美文\{序号}.{作品名}-{作者}.md`
- 例如：`01.散步-莫怀戚.md`

### Step 5️⃣: 去重检查 (prose-deduplicator)

#### 📚 执行内容
- **总索引预检**：收录前必须查看`美文总索引.md`，确认重复情况和数量限制
- **同一文件夹内去重**：检查当前文件夹内所有文章
- **跨文件夹关联**：检查其他级别文件夹的相同文章
- **数量限制检查**：确保同一作品全库收录不超过2篇
- 四级去重检查：
  1. **总索引核查**：查看`美文总索引.md`中的收录情况
  2. **文件名去重**：检查序号是否重复
  3. **作品去重**：检查"作品名+作者"组合是否重复
  4. **内容去重**：检查是否有相同内容

#### ✅ 检查清单
```markdown
## 收录前检查
- [ ] 已查看《美文总索引》，确认拟收录作品的现有情况
- [ ] 确认该作品全库收录数量未超过3篇限制
- [ ] 如需重复收录，已准备建立双向链接

## 文件夹内检查  
- [ ] 序号无重复（1-100）
- [ ] 作品名+作者在同一文件夹内无重复
- [ ] 片段与完整文章一致

## 跨文件夹检查
- [ ] 跨文件夹相同文章已建立双向链接
- [ ] 双向链接格式正确（**相关文章**：[[文件夹/文件名|显示文本]]）
- [ ] 《美文总索引》已更新重复关系记录
```

#### 🎯 关键点

##### 📋 收录前强制检查规则
- **总索引查重**：收录任何文章前必须先查看`美文总索引.md`，确认重复情况
- **同一文件夹规则**：绝对不允许同一文件夹内出现相同文章（作品名+作者重复）
- **跨文件夹数量限制**：同一作品（作品名+作者）在全库中不得超过2篇，包含2篇
- **超限处理**：如发现某作品已达3篇上限，必须选择其他作品替代，不得强行收录

##### 🔗 跨文件夹重复处理规则
- **允许重复条件**：不同文件夹可出现相同文章，但不得超过3篇限制
- **强制双向链接**：所有跨文件夹重复文章必须建立互相的双向链接
- **链接格式**：使用`**相关文章**：[[文件夹/文件名|显示文本]]`格式
- **链接位置**：在文章开头"返回片段"链接下方添加相关文章链接

##### 📊 去重检查流程
1. **预检查**：查看`美文总索引.md`，确认拟收录作品的现有收录情况
2. **数量核查**：如已有收录，统计跨文件夹数量，确保不超过3篇限制
3. **文件夹去重**：检查当前文件夹内是否有重复，如有重复立即处理
4. **跨文件夹关联**：为重复文章建立双向链接
5. **索引更新**：更新`美文总索引.md`，记录新的重复关系

##### 📈 数量限制示例
- ✅ **允许**：《背影》- 朱自清 在初中、高中两个文件夹中收录（2篇）
- ❌ **禁止**：《背影》- 朱自清 再在小学美文中收录（将达到3篇，超限）
- 🔄 **替代**：选择朱自清的其他作品如《绿》《荷塘月色》等进行收录

##### 🚨 违规处理
- **发现超限**：立即停止收录，选择替代作品
- **遗漏链接**：补建所有跨文件夹重复文章的双向链接
- **索引不符**：更新`美文总索引.md`，确保信息准确

#### 📂 输出
- 去重检查报告（如有问题）
- 跨文件夹关联报告（相同文章的双向链接情况）
- `美文总索引.md`更新（新增重复关系记录）
- 数量限制合规报告（确保未超过3篇限制）

### Step 6️⃣: 建立链接 (prose-link-weaver)

#### 📚 执行内容
- 建立片段→完整文章的双向链接
- 建立文章之间的相关链接
- 更新索引文件的链接

#### ✅ 链接类型
1. **片段→完整文章**：在片段下方添加Obsidian双向链接 `[[完整文章的文件名]]`
2. **完整文章→片段**：在相关作品中添加返回链接 `[[片段收集文件名]]`
3. **文章→文章**：在相关作品中添加同主题文章的Obsidian双向链接
4. **索引链接**：更新索引文件的所有链接为Obsidian双向链接格式

#### 🎯 关键点
- 使用标准Obsidian链接格式：`[[文件名]]`
- 链接仅使用文件名，不包含路径
- 空链接应直接清除
- 确保所有链接可点击跳转
- 兼容Obsidian的双向链接功能

#### 📂 需要更新的文件
- `{级别}美文精选100篇.md`
- 所有完整文章的 `.md` 文件
- `美文收集索引.md`

### Step 7️⃣: 质量检查与修复

#### 📚 执行内容
- 检查所有文章的正文格式
- 确保符合prose-article-formatter/SKILL.md要求
- 修复任何不符合标准的内容

#### ✅ 质量检查清单
```markdown
## 格式化完整性检查清单

### 必需元素(不得缺项)
- [ ] YAML 元数据(title, source, created, tags)
- [ ] H1 标题(含Emoji +完整版标注)
- [ ] H3 作者署名
- [ ] 📖 原文正文(分段+Emoji)
- [ ] 🌟 美文赏析(3个子板块)
- [ ] 🎭 朗读指导(2个子板块)
- [ ] 🌈 小练笔(步骤式)
- [ ] 📚 知识小卡片

### 正文Emoji穿插专项检查
- [ ] 开头添加了2-3个主题相关Emoji组合
- [ ] 每个段落都有适度Emoji穿插
- [ ] Emoji密度符合年龄段要求
- [ ] 所有Emoji都与句子内容相关
- [ ] 没有影响阅读流畅性
```

#### 🎯 关键点
- 所有文章必须经过质量检查
- 不符合标准的文章必须立即修复
- 定期进行批量检查

### Step 8️⃣: 总索引更新 (prose-master-index-updater) 🚨 **铁律步骤**

#### 📚 执行内容
- **🚨 强制更新总索引**：收录完成后必须更新`美文总索引.md`，这是不可违背的铁律
- **同步各库信息**：确保总索引反映各文件夹的最新状态
- **维护统计数据**：更新总体统计、分库统计、重复关系等信息

> **⚠️ 铁律声明**：此步骤为强制性要求，任何收录工作如果没有完成总索引更新，都视为未完成。无论收录多少篇文章，都必须执行此步骤！

#### ✅ 更新清单
```markdown
## 总索引更新检查清单

### 基础信息更新
- [ ] 总体统计数据已更新（总文章数、涉及作家数等）
- [ ] 分库统计表已更新（各文件夹文章数量、完成度）
- [ ] 最后更新时间已修改为当前日期

### 文章信息同步
- [ ] 新收录文章已添加到对应文件夹分类中
- [ ] 按作者分类已更新（新作家或新作品）
- [ ] 按题材分类已更新（确保分类准确）
- [ ] 跨文件夹重复关系已记录（如有新增重复）

### 检索功能维护
- [ ] 按首字母检索已更新（新文章标题）
- [ ] 快速检索功能正常（链接有效性）
- [ ] 各库索引链接正常（指向各文件夹索引）

### 质量保证检查
- [ ] 所有新增链接可正常跳转
- [ ] 🔗标记正确标注重复文章
- [ ] 统计数据与实际文件一致
- [ ] 分类逻辑科学合理
```

#### 🎯 关键点
- **🚨 铁律执行**：任何收录工作完成后都必须更新总索引，这是不可违背的强制要求，不得遗漏
- **信息同步**：总索引必须准确反映各文件夹的最新状态
- **数据一致性**：统计数据必须与实际文件完全一致
- **链接有效性**：所有新增链接必须经过测试，确保可正常跳转
- **及时更新**：收录工作完成当日必须完成总索引更新
- **🚨 违规后果**：如果发现收录工作完成后未更新总索引，视为工作流程执行失败

#### 📂 输出文件
- `美文总索引.md`（根目录）
- 总索引更新报告（记录更新内容和验证结果）

#### 🔄 更新流程
1. **数据收集**：从各文件夹索引收集最新信息
2. **统计计算**：重新计算总体统计和分库统计
3. **分类整理**：按文件夹、作者、题材重新整理
4. **重复检查**：确认跨文件夹重复关系
5. **链接验证**：测试所有新增链接的有效性
6. **质量检查**：确保信息准确性和完整性

## ⚠️ 注意事项

### 1. 不可跳过步骤
- ❌ **错误**：直接创建完整文章，跳过片段收集
- ✅ **正确**：严格按照1→2→3→4→5→6→7的顺序

### 2. 不可颠倒顺序
- ❌ **错误**：先格式化文章，再收集片段
- ✅ **正确**：先收集片段，最后进行质量检查

### 3. 每个步骤都要完成
- ❌ **错误**：格式化文章后就停止，不建立链接
- ✅ **正确**：必须执行完Step 7才算完成

### 4. 必须验证版本
- ❌ **错误**：直接使用找到的第一个版本
- ✅ **正确**：严格执行三步验证法

### 5. 必须符合模版
- ❌ **错误**：缺少板块、Emoji或YAML
- ✅ **正确**：100%符合对应级别模版

### 6. 必须进行质量检查
- ❌ **错误**：创建文章后不检查
- ✅ **正确**：执行Step 7进行质量检查与修复

## 🔗 双向链接格式

正确的双向链接格式应为：
```
[[文件名]]
```

示例：
```
[[02.提醒幸福-毕淑敏]]
[[经典散文片段]]
```

### 链接规范
1. **文件名格式**：直接使用目标文件的完整文件名（包括扩展名.md）
2. **同一目录**：链接仅使用文件名，不包含路径
3. **跨目录**：如果跨目录引用，使用相对路径或完整文件名
4. **空链接处理**：空的双向链接（如[[ ]]）应直接清除
5. **目标文件**：链接目标必须是实际存在的.md文件

### 错误链接处理
- 无法找到目标文件的链接应直接清除
- 格式错误的链接应修复为正确格式
- 空链接应直接清除
