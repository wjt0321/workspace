# Vibe Kanban 完整教程

> [!info] 工具概述
> Vibe Kanban 是一个专为 AI 编码代理设计的开源项目管理工具，将混乱的 AI 开发工作流转化为清晰的看板界面。

## 为什么需要 Vibe Kanban？

> [!warning] 传统 AI 编码的问题
> - 大量终端输出，看不懂进度
> - 代码混乱，难以审查
> - Git 操作复杂，容易冲突
> - 任务不可控，不知道 AI 在做什么

> [!success] Vibe Kanban 的解决
> - 可视化看板，一眼看清状态
> - 隔离环境运行，互不干扰
> - 自动处理 Git，自动提交 PR
> - 实时监控，随时干预

## 核心特性

| 特性 | 功能说明 |
|------|----------|
| 🤖 多代理支持 | 同时运行 Claude Code、Codex、Gemini CLI、Amp |
| 📋 看板视图 | Trello 风格的可视化任务管理 |
| 🔒 隔离环境 | 每个任务独立运行，互不干扰 |
| 🔄 Git 集成 | 自动提交、分支管理、PR 创建 |
| 👀 代码审查 | 变更对比、审查评论、质量检查 |
| ☁️ 云端协作 | 团队实时同步（Beta） |

## 安装与启动

> [!tip] 快速启动
> ```bash
> npx vibe-kanban
> ```

| macOS | `brew install vibe-kanban` |

> - Node.js 18+

## 工作流程

> [!example] 工作流程
> 1. 创建任务 → 2. 分配 AI → 3. 隔离执行 → 4. 监控进度 → 5. 代码审查 → 6. 合并 PR

## 核心功能详解

### 1. 创建项目

> [!todo] 添加现有仓库
> 1. 点击左侧「+」按钮
> 2. 选择「添加现有仓库」
> 3. 选择本地 Git 路径
> 4. 设置名称和描述

> [!todo] 创建新项目
> 1. 点击左侧「+」按钮
> 2. 选择「创建新项目」
> 3. 选择模板（空白/Node.js/Python/React）
> 4. 自动初始化 Git 仓库

### 2. 创建任务

```text
┌────────────────────────────────────────────────────────────┐
│  创建任务                                                   │
├────────────────────────────────────────────────────────────┤
│  标题      [实现用户登录功能                    ]           │
│  描述      [请添加用户认证功能...]               ]           │
│  优先级    [● 高  ○ 中  ○ 低]                                 │
│  负责人    [🤖 Claude Code                    ▼]           │
│  标签      [#auth #feature #urgent             ×]           │
│  截止日期  [2026-02-15                       ▼]           │
│                                    [取消]    [创建任务]     │
└────────────────────────────────────────────────────────────┘
```

### 3. 分配 AI 代理

> [!info] 支持的 AI 代理

| 代理 | 特点 | 适用场景 |
|------|------|----------|
| Claude Code | 企业级安全 | 敏感项目 |
| Gemini CLI | 强大推理 | 研究分析 |
| Codex | 快速生成 | 原型开发 |
| Amp | 本地优先 | 隐私优先 |

### 4. 任务状态流转

> [!question] 状态说明

```
    ┌─────────┐
    │  待办   │  等待开始的任务
    └────┬────┘
         │ 开始任务（分配 AI）
         ↓
    ┌─────────┐
    │ 进行中  │  AI 代理正在隔离环境中执行
    │ ⚡ AI   │  实时显示终端输出
    └────┬────┘
         │
         ├─→ 暂停/取消
         └─→ 提交代码 ──→ 审查中
    ┌─────────┐
    │ 审查中  │  等待人工或自动审查
    │ 👀 AI   │  显示代码变更
    └────┬────┘
         │
         ├─→ 请求修改（打回进行中）
         └─→ 批准合并 ──→ 已完成
    ┌─────────┐
    │ 已完成  │  任务完成
    │ ✅      │
    └─────────┘
```

### 7. Git 操作

> [!note] 分支命名规范
> ```
> 格式：{类型}/{任务编号}-{简短描述}
> 
> 类型前缀：
> - feat/     新功能
> - fix/      Bug 修复
> - refactor/ 代码重构
> 
> 示例：
> - feat/123-add-user-authentication
> - fix/456-fix-login-redirect-loop
> ```

> [!bug] 冲突解决流程
> 1. Vibe Kanban 自动检测冲突
> 2. 打开冲突面板
> 3. 选择解决方式：
>    - 保留当前版本 (HEAD)
>    - 保留传入版本 (feature 分支)
>    - 手动编辑合并
> 4. 解决完成，继续合并

## 最佳实践

### 任务设计原则

> [!success] ✅ 好任务示例
> - "实现用户登录功能"
> - "添加密码重置邮件"
> - "修复登录重定向循环"

> [!failure] ❌ 差任务示例
> - "实现用户系统并修复各种问题"

> [!important] 任务粒度建议
> - 过小 → 难以追踪价值
> - **适中 → 1-4 小时 ⭐ 推荐**
> - 过大 → 难以管理，容易偏离

### AI 代理使用技巧

> [!quote] 好的指令示例
> "这是一个使用 React + TypeScript 的电商项目，位于 /workspace/shop-app。请添加用户登录功能，使用现有的认证系统模式，在 src/auth 目录下实现。遵循 ESLint 规则和项目代码风格。"

> [!warning] 不好的指令
> "添加登录功能"

> [!help] 监控与干预
> 发现 AI 偏离方向时：
> 1. 暂停任务
> 2. 明确指出问题："你在修改错误文件，请回到 src/auth 目录"
> 3. 给出正确方向："使用现有的 auth 服务模式"

## 常见问题

### 任务执行失败

> [!faq] 排查步骤
> 1. 查看终端输出，找错误信息
> 2. 常见原因：
>    - 依赖安装失败 → 检查网络
>    - API 配额耗尽 → 等待重置
>    - 代码冲突 → 先合并主分支
>    - 权限不足 → 重新认证
> 3. 修复后点击「重试」

### Git 冲突解决

> [!summary] 冲突类型与策略
> - 单文件冲突 → 快速解决（选择版本）
> - 多文件冲突 → AI 自动解决
> - 重大分歧 → 人工审查讨论

### AI 代理选择

> [!info] 选择指南
> | 场景 | 推荐代理 | 原因 |
> |------|----------|------|
> | 企业项目（需权限控制） | Claude Code | 企业级安全 |
> | 研究探索（需推理） | Gemini CLI | 强大推理 |
> | 快速原型（效率优先） | Codex | 快速生成 |
> | 基础设施（隐私优先） | Amp | 本地优先 |
