# AI网文小说生成系统 - 需求文档

## 1. 项目概述

### 1.1 项目背景
开发一个基于AI的智能小说创作助手系统，能够自动生成高质量的网络小说，支持完整的创作流程：世界观构建、角色设计、职业体系设定、大纲生成、章节续写、情节分析等功能。

### 1.2 项目目标
- 创建一个完整的AI小说创作平台
- 支持多AI模型（智谱GLM、MiniMax，通过 Anthropic 兼容接口）
- 保留 OpenAI/Claude 原生接口支持（可选）
- 实现流式生成，避免超时问题
- 建立长期记忆系统，保持故事连贯性
- 提供可视化的创作管理界面
- 轻量级部署，适合个人或小团队使用（3-5人）

## 2. 功能需求

### 2.1 项目管理模块

#### FR-PM-001 创建项目
**优先级**: P0（必须）
**描述**: 用户可以创建新的小说项目，包含基本信息录入
**输入字段**:
- 书名（title）：必填，3-30字
- 简介（description）：必填，50-500字
- 主题（theme）：必填，核心主题描述
- 类型（genre）：必填，类型标签（玄幻/都市/仙侠/科幻等）
- 目标字数（target_words）：可选，默认30万字
- 叙事视角（narrative_perspective）：可选，第三人称/第一人称

**业务流程**:
```
用户输入基本信息
    ↓
AI引导生成世界观 → 职业体系 → 角色 → 大纲
    ↓
项目创建完成
```

#### FR-PM-002 项目列表
**优先级**: P0（必须）
**描述**: 显示用户所有项目，支持搜索和筛选
**功能**:
- 列表展示：书名、类型、创建时间、进度
- 搜索：按书名搜索
- 筛选：按类型筛选

#### FR-PM-003 项目详情
**优先级**: P0（必须）
**描述**: 查看项目详细信息和创作进度
**显示内容**:
- 项目基本信息
- 已生成章节数量和字数
- 已设计角色数量
- 大纲进度
- 操作按钮（继续创作、编辑设置等）

### 2.2 世界观生成模块

#### FR-WB-001 世界观自动生成
**优先级**: P0（必须）
**描述**: 基于项目基本信息，AI自动生成完整的世界观设定
**输出结构**:
```json
{
  "time_period": "时间背景与社会状态（300-500字）",
  "location": "空间环境与地理特征（300-500字）",
  "atmosphere": "感官体验与情感基调（300-500字）",
  "rules": "世界规则与社会结构（300-500字）"
}
```

**类型适配规则**:
- 现代都市：聚焦城市环境、行业文化、社会现状
- 玄幻仙侠：修炼规则、灵气环境、门派势力
- 历史古代：时代特征、礼教制度、阶级分化
- 科幻：科技水平、社会形态、文明转折

**质量要求**:
- 世界观必须支撑主题
- 为简介中的情节提供合理背景
- 设定之间逻辑自洽

#### FR-WB-002 世界观编辑
**优先级**: P1（重要）
**描述**: 用户可以手动编辑和调整世界观设定
**功能**:
- 文本编辑
- 版本历史
- 一键还原

### 2.4 角色管理模块

#### FR-CH-001 批量角色生成
**优先级**: P0（必须）
**描述**: 基于世界观和职业体系，批量生成小说角色
**角色类型**:
- 主角（protagonist）：至少1个
- 配角（supporting）：多个
- 反派（antagonist）：可选
- 组织（organization）：高影响力组织

**输出结构**:
```json
[
  {
    "name": "角色姓名",
    "age": 25,
    "gender": "男/女",
    "is_organization": false,
    "role_type": "protagonist",
    "personality": "性格特点（100-200字）",
    "background": "背景故事（100-200字）",
    "appearance": "外貌描述（50-100字）",
    "traits": ["特长1", "特长2"],
    "relationships_array": [
      {
        "target_character_name": "已有角色名",
        "relationship_type": "关系类型",
        "intimacy_level": 75,
        "description": "关系描述"
      }
    ],
    "career_assignment": {
      "main_career": "主职业名",
      "main_stage": 2,
      "sub_careers": [{"career": "副职业名", "stage": 1}]
    }
  }
]
```

**数量要求**:
- 每批生成5个角色
- 必须包含1个主角
- 精确控制数量，不多不少

#### FR-CH-002 角色关系管理
**优先级**: P1（重要）
**描述**: 可视化管理角色之间的关系网络
**功能**:
- 关系图谱可视化
- 关系类型：家族/社交/职业/敌对
- 关系强度：-100到100
- 组织成员管理

#### FR-CH-003 自动角色引入
**优先级**: P2（参考）
**描述**: 在续写过程中，预测剧情发展并智能引入新角色
**功能**:
- 预测性分析：是否需要新角色
- 角色规格生成
- 批量创建新角色
- 自动建立关系

### 2.5 大纲管理模块

#### FR-OL-001 开篇大纲生成
**优先级**: P0（必须）
**描述**: 生成小说开篇的大纲规划
**输出结构**:
```json
[
  {
    "chapter_number": 1,
    "title": "章节标题",
    "summary": "章节概要（300-500字）",
    "scenes": ["场景1", "场景2"],
    "characters": ["角色1", "角色2"],
    "key_points": ["要点1", "要点2"],
    "emotion": "情感基调",
    "goal": "叙事目标"
  }
]
```

**开篇要求**:
- 完成开局设定和世界观展示
- 引入主要角色，建立初始关系
- 埋下核心矛盾和悬念钩子
- 为后续剧情发展打下基础

#### FR-OL-002 大纲续写
**优先级**: P0（必须）
**描述**: 基于已有章节，续写后续大纲
**输入参数**:
- 起始章节号
- 续写章节数量
- 剧情阶段（开端/发展/高潮/结局）
- 故事发展方向

**功能要求**:
- 自然衔接前文
- 考虑已有伏笔和钩子
- 遵循角色成长轨迹

#### FR-OL-003 大纲展开
**优先级**: P1（重要）
**描述**: 将单个大纲节点展开为多个章节规划
**策略选项**:
- balanced：平衡展开
- climax：侧重高潮
- detail：详细展开

**输出结构**:
```json
[
  {
    "sub_index": 1,
    "title": "章节标题",
    "plot_summary": "剧情摘要（200-300字）",
    "key_events": ["事件1", "事件2", "事件3"],
    "character_focus": ["角色A", "角色B"],
    "emotional_tone": "情感基调",
    "narrative_goal": "叙事目标",
    "conflict_type": "冲突类型",
    "estimated_words": 3000
  }
]
```

### 2.6 章节生成模块

#### FR-CH-001 单章生成
**优先级**: P0（必须）
**描述**: 基于大纲生成单个章节的正文内容
**输入**:
- 章节大纲
- 角色信息
- 上一章结尾（衔接锚点）
- 写作风格
- 目标字数

**输出**:
- 纯文本章节内容
- 目标字数 ±10% 浮动

**上下文策略**:
| 章节范围 | 衔接锚点 | 角色信息 | 记忆 | 故事骨架 |
|---------|---------|---------|------|---------|
| 第1章 | 无 | 全部角色 | 无 | 无 |
| 第2-10章 | 300字 | 相关角色 | 无 | 无 |
| 第11-50章 | 500字 | 相关角色 | 3条 | 无 |
| 第51章+ | 500字 | 相关角色 | 5条 | 有 |

**质量要求**:
- 严格按照大纲推进
- 自然承接上一章结尾
- 保持角色性格一致
- 禁止使用AI常见总结语

#### FR-CH-002 批量章节生成
**优先级**: P1（重要）
**描述**: 批量生成多个章节
**功能**:
- 批量选择大纲
- 并发生成控制
- 进度显示
- 失败重试

#### FR-CH-003 章节重写
**优先级**: P1（重要）
**描述**: 根据反馈意见重新生成章节
**输入**:
- 原章节内容
- 修改要求（用户输入 + AI分析建议）
- 重新生成次数

### 2.7 情节分析模块

#### FR-PA-001 章节分析
**优先级**: P1（重要）
**描述**: 分析已生成章节的剧情要素
**分析维度**:
1. 剧情钩子（Hooks）
   - 类型：悬念/情感/冲突/认知
   - 强度评分（1-10）
   - 出现位置
   - 原文关键词

2. 伏笔分析（Foreshadows）
   - 新埋伏笔
   - 回收旧伏笔
   - 巧妙性评分

3. 冲突分析（Conflict）
   - 冲突类型
   - 冲突各方
   - 强度评分（1-10）
   - 解决进度

4. 情感曲线（Emotional Arc）
   - 主导情绪
   - 情感强度（1-10）
   - 情绪变化轨迹

5. 角色状态追踪
   - 心理状态变化
   - 关系变化
   - 职业变化（可选）

6. 质量评分
   - 节奏把控（pacing）
   - 吸引力（engagement）
   - 连贯性（coherence）
   - 整体质量（overall）

7. 改进建议
   - 与分数关联的针对性建议

**输出结构**:
```json
{
  "hooks": [
    {
      "type": "悬念",
      "content": "描述",
      "strength": 8,
      "position": "中段",
      "keyword": "原文关键词"
    }
  ],
  "foreshadows": [...],
  "conflict": {...},
  "emotional_arc": {...},
  "character_states": [...],
  "scores": {
    "pacing": 6.5,
    "engagement": 5.8,
    "coherence": 7.2,
    "overall": 6.5
  },
  "suggestions": [
    "改进建议1",
    "改进建议2"
  ]
}
```

### 2.8 记忆系统模块

#### FR-MS-001 语义检索
**优先级**: P1（重要）
**描述**: 基于向量数据库实现语义检索
**技术要求**:
- Embedding模型：paraphrase-multilingual-MiniLM-L12-v2
- 向量库：ChromaDB
- 支持中文语义匹配

**功能**:
- 按内容语义搜索相关记忆
- 按重要性阈值过滤
- 按章节范围过滤
- 按记忆类型过滤

#### FR-MS-002 伏笔追踪
**优先级**: P2（参考）
**描述**: 自动追踪未回收的伏笔
**功能**:
- 标记伏笔状态：已埋下/已回收
- 智能提醒即将到期伏笔
- 伏笔时间线可视化

#### FR-MS-003 故事骨架
**优先级**: P2（参考）
**描述**: 为长篇小说（50章+）提供故事骨架
**功能**:
- 每10章采样一次章节摘要
- 构建完整的故事发展脉络
- 生成章节索引

## 3. 非功能需求

### 3.1 性能需求
| 指标 | 要求 |
|------|------|
| 页面加载时间 | < 2秒 |
| API响应时间 | < 3秒 |
| 章节生成速度 | 500字/秒（流式） |
| 并发用户数 | 支持3-5用户（轻量级部署） |
| 同时生成任务 | 最多5个并行AI请求 |

### 3.2 可用性需求
- Docker 单容器一键部署（或本地直接运行）
- SQLite 数据库，无需额外安装
- 错误提示清晰
- 操作可撤销
- 进度实时显示（SSE）

### 3.3 可扩展性需求
- 支持添加新的AI提供商（Anthropic 兼容接口）
- 默认使用智谱GLM-4-Flash（低成本），可切换MiniMax或原生Claude
- 支持自定义Prompt模板
- 插件系统（MCP，可选）

### 3.4 安全需求
- API密钥加密存储
- 用户数据隔离
- SQL注入防护
- XSS防护
