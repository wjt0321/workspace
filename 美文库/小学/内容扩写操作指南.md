# 内容扩写操作指南

## 🎯 适用范围
本指南适用于为现有文档集合进行内容扩写和整合的AI协作场景。

---

## 📋 标准工作流程

### 第一阶段：准备与分析
1. **探索文件结构**
   - 使用 `Glob` 工具识别目标文件夹和文件
   - 了解现有文档的组织结构和命名规律

2. **读取原始内容**
   - 使用 `Read` 工具完整读取目标文件
   - 分析现有内容的格式规范、分类体系、写作风格

3. **结构化分析**
   - 确定文章分类方式和数量分配
   - 分析标准格式要求（标题、作者、正文、推荐理由等）
   - 识别语言风格特点和表达方式

### 第二阶段：规划与创作
4. **内容分类规划**
   - 根据新增内容主题确定分类归属
   - 保持分类逻辑的一致性和合理性
   - 更新各分类的文章数量统计

5. **内容创作标准**
   - 严格按照现有格式创作新内容
   - 保持语言风格和表达方式的一致性
   - 确保每篇内容都有明确的教育价值和推荐理由
   - 遵循由物及人、托物言志的写作思路
   - 尽量选用文学大家的作品选篇

### 第三阶段：整合与实施
6. **渐进式整合**
   - 优先采用文件重建而非复杂修改
   - 重新计算和分配所有文章编号
   - 更新标题、说明文字和统计数据

7. **格式统一检查**
   - 验证标题格式一致性
   - 确认推荐理由格式规范
   - 检查分段和标记符号

### 第四阶段：完善与交付
8. **配套内容更新**
   - 更新学习指导部分
   - 调整推荐阅读列表
   - 更新使用说明和教学建议

9. **文件版本管理**
   - 不要创建新的完整版本文件，而是在原有基础上扩写。‘序号，标题，学习指南，使用建议’等要依据原文逻辑做相应调整
   - 采用清晰的版本命名规则

---

## 🔧 工具使用策略

### 优先级原则
1. **专用工具优先**
   - Read/Write/Edit > 系统命令
   - Glob/Grep > find/search

2. **操作安全原则**
   - 复杂修改前先创建备份
   - 避免可能导致格式破坏的批量操作
   - 优先创建新文件而非修改复杂结构

3. **并行处理原则**
   - 独立操作可并行执行
   - 依赖操作需按顺序执行



## 📝 内容创作标准

### 格式规范
- **标题格式**: `#### **N. 《标题》- 作者**`
- **正文**: 采用引用格式 `> `
- **推荐理由**: `**【推荐理由】**...`
- **分段**: 使用 `---` 分隔不同文章

### 内容要求
- **教育价值**: 每篇文章都要有明确的教育意义
- **年龄适配**: 内容适合目标读者群体
- **积极导向**: 传递正能量，培养美好品格
- **语言优美**: 文字生动形象，富有感染力

### 写作技巧
- **修辞运用**: 合理使用拟人、比喻、排比等手法
- **结构清晰**: 层次分明，逻辑严密
- **由表及里**: 从具体事物引申到人生哲理
- **情景交融**: 写景与抒情完美结合

## 🎯 协作原则

### 用户需求理解
- **完整性要求**: 确保新内容完全融入现有体系
- **一致性要求**: 保持原有风格和格式不变
- **实用性导向**: 内容适合目标用户群体使用
- **可维护性**: 便于后续继续扩写和修改
- **艺术性**：尽量选用文学大师的选篇

### 交付标准
- 明确的版本命名规则
- 清晰的使用说明

## 📚 模板化操作

### 标准模板结构
```
# 文档标题
- 分类一：文章数量
  - 文章1
  - 文章2
- 分类二：文章数量
  - 文章1
  - 文章2
- 学习指导
- 使用建议
```

### 文件命名规范
- 基础格式：`[内容名称]精选[N]篇（[版本]）.md`
- 版本递进：整合版 → 完整版 → 扩展版
- 清晰体现文章数量和版本信息
