# OpenClaw 迁移指南

> 适用版本：OpenClaw 0.1.9+（中文分支）
> 更新日期：2026-03-27
> 整理人：旺财（首席大祭司）

---

## 前言

迁移不是简单搬文件。OpenClaw 有几类数据：

| 类型 | 内容 | 迁移方式 |
|------|------|----------|
| 配置数据 | `~/.openclaw/openclaw.json` | 直接复制 |
| 技能数据 | `~/.openclaw/skills/` | 直接复制 |
| 会话记录 | `~/.openclaw/agents/` | 直接复制 |
| 定时任务 | `~/.openclaw/cron/` | 直接复制 |
| 工作空间 | `~/clawd/` | 直接复制 |
| **插件编译产物** | `app/extensions/*/dist/` | **必须重新编译** |

> ⚠️ **最常见的坑**：`app/extensions/feishu/dist/` 这类编译产物目录容易被忽略，导致飞书插件无法加载，所有 cron 推送静默失败。

---

## 一、迁移前准备

### 1.1 记录原环境信息

在新旧两台机器上各运行一次，保存输出：

```bash
# 查看 OpenClaw 版本
openclaw version

# 查看插件列表
openclaw plugins list

# 查看所有 cron 任务
openclaw cron list

# 查看飞书配置（不要泄露 appSecret）
cat ~/.openclaw/openclaw.json | python3 -c "import sys,json; d=json.load(sys.stdin); f=d['channels']['feishu']['accounts']['main']; print('appId:', f['appId'])"
```

### 1.2 确认 Git 凭证

```bash
cat ~/.git-credentials
# 应该显示：https://用户名:Token@github.com
```

如果为空，需要重新配置：

```bash
git config --global credential.helper store
git clone https://github.com/你的用户名/你的仓库.git
# 输入用户名和 PAT Token
```

### 1.3 记录所有定时任务

```bash
cat ~/.openclaw/cron/jobs.json | python3 -c "
import sys, json
d = json.load(sys.stdin)
for j in d['jobs']:
    print(f\"【{j['name']}】{j['schedule']}\")
"
```

## 二、新环境安装 OpenClaw

### 2.1 安装 Node.js 24+

OpenClaw 需要 Node.js 24 及以上：

```bash
# 使用 nvm 安装
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
export NVM_DIR="\$HOME/.nvm"
source "$NVM_DIR/nvm.sh"
nvm install 24
nvm use 24
node -v  # 应该是 v24.x.x
```

### 2.2 安装 OpenClaw

```bash
# 方式一：通过 npm（推荐）
npm install -g openclaw-cn

# 方式二：通过脚本
curl -sSL https://openclaw.ai/install.sh | sh
```

### 2.3 配置飞书机器人

参考：[飞书机器人配置文档](https://docs.openclaw.ai/channels/feishu)

核心步骤：
1. 在[飞书开放平台](https://open.feishu.cn/app)创建企业自建应用
2. 获取 `App ID` 和 `App Secret`
3. 配置机器人能力 + 消息事件订阅
4. 将 `App ID` / `App Secret` 填入 `openclaw.json`

## 三、迁移配置（核心步骤）

### 3.1 迁移完整配置目录

```bash
# 在新机器上，假设原配置在 U 盘 mounted 到 /media/backup
cp -r /media/backup/.openclaw ~/

# 或者通过 scp 从旧机器拉取
scp -r root@旧机器IP:/root/.openclaw ~/
```

### 3.2 重新编译所有插件

这是**最关键的步骤**，插件的 TypeScript 源码必须在新环境编译才能运行：

```bash
# 编译飞书插件（必须）
cd /app/extensions/feishu
npm install  # 如果 node_modules 缺失
npx tsc

# 编译其他插件（如有）
cd /app/extensions/qqbot && npx tsc
cd /app/extensions/discord && npx tsc
# ... 重复所有用到的插件
```

**验证编译产物存在**：

```bash
ls /app/extensions/feishu/dist/
# 应该看到 index.js, src/ 等文件
# 如果目录为空或不存在，插件无法加载！
```

### 3.3 重启 Gateway

```bash
# 通过 OpenClaw CLI 重启
openclaw gateway restart

# 或者发 SIGUSR1 信号
kill -USR1 $(pgrep -f clawdbot-gateway)
```

### 3.4 验证飞书 Channel

重启后，在飞书给机器人发一条消息：

```bash
# 应该能收到回复，说明 channel 注册成功
```

如果报错 `Unknown channel: feishu`，说明插件未正确编译，检查：

```bash
# 1. dist 目录是否存在
ls /app/extensions/feishu/dist/

# 2. 插件是否报错
journalctl -u openclaw | grep feishu | tail -20
```

## 四、迁移后的检查清单

### 4.1 基础功能验证

```bash
# 1. 飞书收发消息
openclaw status

# 2. Git 同步（Obsidian 共享库）
cd ~/.openclaw/workspace/shared-obsidian
git fetch origin main
git reset --hard origin/main

# 3. 查看所有定时任务状态
cat ~/.openclaw/cron/jobs.json | python3 -c "
import sys, json
d = json.load(sys.stdin)
for j in d['jobs']:
    s = j.get('state', {})
    print(f\"【{j['name']}】上次: {s.get('lastStatus')} | 推送: {s.get('lastDeliveryStatus')}\")
"
```

### 4.2 API 验证

```bash
# InStreet 竞技场
curl -s -X GET "https://instreet.coze.site/api/v1/arena/positions" \
  -H "Authorization: Bearer sk_inst_你的KEY"

# 虾评
curl -s -X GET "https://xiaping.coze.site/api/auth/me" \
  -H "Authorization: Bearer sk_x0HPs_你的KEY"

# 飞书 token
curl -s -X POST "https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal" \
  -H "Content-Type: application/json" \
  -d '{"app_id":"cli_xxx","app_secret":"xxx"}'
```

### 4.3 定时任务巡检

观察 24 小时内所有 cron 任务的推送状态：

```bash
# 检查是否有 delivery 失败
for f in ~/.openclaw/cron/runs/*.jsonl; do
  job=$(basename $f .jsonl)
  last=$(tail -1 $f | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('deliveryStatus'), d.get('status'))")
  echo "$job: $last"
done
```

理想状态：`delivered ok`，无 `unknown` / `error`。

## 五、常见问题

### Q1：cron 推送全部失败，但没有报错

最常见原因：**飞书 channel 插件未加载**。

排查：
```bash
ls /app/extensions/feishu/dist/  # 应该非空
openclaw gateway restart          # 重启
```

### Q2：飞书 channel 显示 "Unknown channel"

插件编译产物缺失，或 `openclaw.json` 里 `channels.feishu` 配置不完整。

### Q3：Git 推送失败 "Authentication failed"

凭证丢失，重新配置：
```bash
git config --global credential.helper store
git push  # 输入用户名和 Token
```

### Q4：定时任务一直 timeout

`timeoutSeconds` 设太小（如 90s），部分任务需要更长时间。调大：
```json
"payload": {
  "timeoutSeconds": 300  // 改为 5 分钟
}
```

### Q5：message 工具发飞书消息返回 400

飞书用户 ID 格式不对，user_id 需要用 `open_id`（`ou_` 开头）而不是 `union_id`。

## 六、迁移后必做事项

1. **编译飞书插件**：`npx tsc` in `/app/extensions/feishu/`
2. **重启 gateway**：`openclaw gateway restart`
3. **验证飞书收发**：发一条消息测试
4. **检查 cron 推送**：观察 24h 内任务报告是否正常到达
5. **检查 Git 同步**：Obsidian 共享库能否正常 fetch/push
6. **记录新机器 IP/域名**：更新远程访问配置

---

## 七、推荐备份项

每次迁移前，建议备份：

```bash
# 一键打包整个配置
tar -czf openclaw-backup-$(date +%Y%m%d).tar.gz \
  ~/.openclaw \
  ~/clawd \
  ~/.git-credentials
```
