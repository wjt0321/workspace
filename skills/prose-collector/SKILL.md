# 📚 美文收集自动化系统

> 🚀 **一键触发，全自动完成美文收集9步SOP流程**

本文档是完整的 **Obsidian 美文收集技能**，整合了9步标准工作流程，支持5个级别的美文收录。

---

## 📋 系统概览

### 5级美文体系

| 级别 | 目录 | 片段长度 | 正文长度 | 定位 |
|------|------|----------|----------|------|
| 🌱 小学生美文 | `小学生美文/` | 150-300字 | 300-1500字 | 基础阅读、兴趣培养 |
| 📚 初中生美文 | `初中生美文/` | 200-400字 | 800-2000字 | 文学鉴赏、写作技巧 |
| 📕 高中生美文 | `高中生美文/` | 300-500字 | 1500-5000字 | 文学批评、思辨思维 |
| 📘 大学生美文 | `大学生美文/` | 300-500字 | 1500-5000字 | 学术研究、理论批评 |
| 📗 成人美文 | `成人美文/` | 300-500字 | 不限 | 人生智慧、审美品味 |

### 9步工作流程

```
Step 0 → Step 1 → Step 2 → Step 3 → Step 4 → Step 5 → Step 6 → Step 7 → Step 8
  ⬆️                                                    ⬆️
  🔴 预查重（必须）                              🚨 总索引更新（铁律）
```

## 🚀 快速开始

### 触发方式

```bash
"帮我收集3篇成人美文"
"收集5篇小学生美文"
"按照SOP流程收集高中生美文"
```

### 执行示例

```markdown
用户输入：帮我收集2篇初中生美文

执行流程：
Step 0: 预查重 → 检查 [[美文总索引]]，制定收集策略
Step 1: 片段收集 → 生成2个片段（200-400字）
Step 2: 索引更新 → 更新 [[初中生美文/美文收集索引]]
Step 3: 原文搜索 → 三步验证确认完整版
Step 4: 文章格式化 → 使用模板创建完整文章
Step 5: 去重检查 → 四级验证
Step 6: 双向链接 → 建立片段↔文章链接
Step 7: 质量检查 → 完整性验证
Step 8: 总索引更新 → 更新 [[美文总索引]]（铁律）
```

## 📖 Step 0: 预查重（必须）

### 执行内容

1. **读取总索引**
   ```markdown
   [[美文总索引]]
   ```

2. **分析现有情况**
   - 各文件夹文章数量
   - 作家分布
   - 题材覆盖
   - 跨文件夹重复关系

3. **识别风险**
   - 🔴 高风险：已达2篇上限的作品
   - 🟡 中风险：已收录1篇的作品
   - 🟢 低风险：未收录的作品

4. **制定策略**
   - 推荐收录清单
   - 避免收录清单
   - 作家平衡策略

### 输出格式

```markdown
## 🔍 Step 0 预查重报告

### 📊 当前状态
- 总文章数：{N}篇
- 涉及作家：{N}位

### ⚠️ 重复风险
- 🔴 高风险：{作品列表}
- 🟡 中风险：{作品列表}

### ✅ 推荐收录
- {作品名} - {作者}（理由）
```

---

## 📖 Step 1: 片段收集

### 执行内容

1. **识别目标级别**
   - 从用户输入中提取级别
   - 定位对应工作目录

2. **读取索引文件**
   ```markdown
   [[{级别}美文/美文收集索引]]
   ```

3. **选择作家和作品**
   - 优先选择近现代散文大家
   - 避免已收录作品
   - 保持多样性（单个作家≤10%）

4. **提取片段**
   - 按级别标准提取合适长度
   - 选择文章精华段落
   - 确保片段可独立阅读

5. **格式化片段**

   ```markdown
   #### **{序号}. 《{作品名}》 - {作者}**

   > {开头Emoji组合}
   >
   > {穿插Emoji的片段内容...}

   **【推荐理由】** {推荐语}

   [[{完整文章文件名}]] {如已有完整版}

   ---
   ```

### Emoji规范

- **开头Emoji**：2-3个主题相关Emoji
- **内容穿插**：每2-3句话1个Emoji
- **句子定义**：以任何标点符号结尾的完整语义单元
- **相关度**：必须与句子内容紧密相关

---

## 📖 Step 2: 索引更新

### 执行内容

1. **读取索引文件**
   ```markdown
   [[{级别}美文/美文收集索引]]
   ```

2. **更新统计**
   - 总计篇数
   - 已完成篇数
   - 待补充篇数
   - 最后更新日期

3. **添加片段记录**
   ```markdown
   #### **{序号}. 《{作品名}》 - {作者}**
   - **文件位置**：`{片段文件名}.md`
   - **行号范围**：{行号}
   - **主题**：{主题分类}
   - **特色**：{特色描述}
   ```

4. **检查序号**
   - 寻找可用序号
   - 避免序号冲突

---

## 📖 Step 3: 搜索原文（三步验证）

### 执行内容

#### 第一步：搜索完整版关键词

```bash
# 搜索组合（按优先级）
"{作品名}" "{作者}" 原文完整版
"{作品名}" "{作者}" 未删减版
"{作品名}" "{作者}" 文集版
```

#### 第二步：版本对比（🚨最关键）

**必须执行的搜索：**
```bash
"{作品名}" 课文与原文区别
"{作品名}" 教材删改
"{作品名}" 课文版 vs 原文版
```

#### 第三步：版本验证

- **字数验证**：完整版通常比课文版更长
- **结构验证**：检查开头/结尾是否完整
- **特征验证**：是否包含背景/深刻结局

### 核心理念

> 🚨 **宁可不收录，也不能收录删减版！**

### 经典作品删改清单

| 作品 | 作者 | 删减风险 | 核查关键词 |
|------|------|----------|------------|
| 《白鹅》 | 丰子恺 | 🔴 高 | 抗战、被吃掉 |
| 《燕子》 | 郑振铎 | 🔴 高 | 海燕、乡愁 |
| 《背影》 | 朱自清 | 🟡 中 | 祖母、家道中落 |
| 《绿》 | 朱自清 | 🟡 中 | 什刹海、对比 |

## 📖 Step 4: 文章格式化

### 执行内容

1. **读取模板**

   根据级别选择模板：

   | 级别 | 模板文件 |
   |------|----------|
   | 小学生 | `[[模板-小学生美文]]` |
   | 初中生 | `[[模板-初中生美文]]` |
   | 高中生 | `[[模板-高中生美文]]` |
   | 大学生 | `[[模板-大学生美文]]` |
   | 成人 | `[[模板-成人美文]]` |

2. **创建YAML元数据**

   ```yaml
   ---
   title: {文章标题}
   author: {作者}
   source: {出处}
   created: {YYYY-MM-DD}
   level: {级别}
   tags:
     - {级别}美文
     - 散文
     - {作者}
     - 完整版
   ---
   ```

3. **格式化标题**

   ```markdown
   # {Emoji} {文章标题}(完整版)
   ### —— {作者}

   ---
   ```

4. **格式化正文**

   - 分段标注Emoji小标题
   - 穿插Emoji（每2-3句话1个）
   - 开头添加Emoji组合

5. **生成必需板块**

   根据级别生成对应板块（详见模板）

### 文件命名

```
{序号}.{标题}-{作者}.md
示例：12.绿-朱自清.md
```

---

## 📖 Step 5: 去重检查

### 执行内容

#### 检测维度

1. **总索引预检**（强制）
   - 检查[[美文总索引]]
   - 确认全库收录数量≤2篇

2. **同级检测**
   - 同一文件夹内不允许重复

3. **跨级检测**
   - 跨文件夹允许重复，但必须建立链接

4. **序号检测**
   - 检查序号是否冲突
   - 文件名/索引/片段标记是否一致

### 数量限制

- 🚨 **全库上限**：同一作品（作品名+作者）≤2篇
- ❌ **同级禁止**：同一文件夹内绝对不允许重复
- ✅ **跨级允许**：不同文件夹可重复，需建立双向链接

### 输出格式

```markdown
## 🔍 去重检查报告

### ✅ 通过项
- 总索引预检：{通过/未通过}
- 同级去重：{通过/未通过}
- 序号检查：{通过/未通过}

### ⚠️ 需处理
- 跨文件夹关联：{数量}对
- 建议操作：{描述}
```

---

## 📖 Step 6: 建立链接

### 执行内容

1. **片段 → 完整文章链接**

   在片段推荐理由后添加：
   ```markdown
   [[{序号}.{标题}-{作者}|→ 查看完整版]]
   ```

2. **完整文章 → 片段链接**

   在文章YAML后、标题前添加：
   ```markdown
   > 📚 **片段收录**: [[片段文件名|返回片段]]
   ```

3. **完整文章 → 完整文章链接**

   相关作品之间建立链接

4. **索引链接更新**

   更新[[美文总索引]]和各文件夹索引

### 链接格式

```markdown
[[文件名]]                    # 链接到文件
[[文件名|显示文本]]           # 自定义显示文本
[[文件名#标题]]                # 链接到文件中的特定标题
```

---

## 📖 Step 7: 质量检查

### 检查清单

#### 必需元素检查

- [ ] YAML元数据完整（title, author, source, created, tags）
- [ ] H1标题含Emoji和"完整版"标注
- [ ] H3作者署名
- [ ] 原文正文分段+Emoji
- [ ] 所有必需板块齐全
- [ ] 双向链接已建立

#### Emoji规范检查

- [ ] 开头Emoji组合（2-3个）
- [ ] 每段都有Emoji穿插
- [ ] 密度符合标准（每2-3句话1个）
- [ ] Emoji与内容相关

#### 链接有效性检查

- [ ] 片段→文章链接可跳转
- [ ] 文章→片段链接可跳转
- [ ] 索引链接可跳转

---

## 📖 Step 8: 总索引更新（铁律）

### 🚨 强制执行

此步骤为**强制性要求**，任何收录工作没有完成总索引更新都视为未完成！

### 执行内容

1. **更新总体统计**

   ```markdown
   ## 📊 总体统计
   - **总文章数**：{最新总数}篇
   - **涉及作家数**：{最新数量}位
   - **最后更新**：{当前日期}
   ```

2. **更新分库统计**

   | 文件夹 | 已收录 | 完成率 |
   |--------|--------|--------|
   | 小学生美文 | {N}篇 | {N}% |
   | 初中生美文 | {N}篇 | {N}% |
   | 高中生美文 | {N}篇 | {N}% |
   | 大学生美文 | {N}篇 | {N}% |
   | 成人美文 | {N}篇 | {N}% |

3. **更新文章列表**

   - 按文件夹分类
   - 按作者分类
   - 按题材分类
   - 按首字母检索

4. **更新重复关系**

   ```markdown
   ## 🔗 跨文件夹重复关系
   - 《背影》- 朱自清：初中版 ↔ 高中版
   ```

### 完成确认

```markdown
## ✅ Step 8 完成确认

- [x] 总体统计已更新
- [x] 分库统计已更新
- [x] 文章列表已同步
- [x] 重复关系已记录
- [x] 链接已验证

🎉 **工作流程完整结束！**
```

    path: 高中生美文/

    template: 模板-大学生美文.md

    path: 成人美文/

### 去重规则

```yaml
deduplication:
  maxCrossLevelCopies: 2      # 全库同一作品上限
  sameLevelDuplicateAllowed: false  # 同级禁止重复
  linkRequired: true          # 跨级需建立链接
```

### Emoji规则

```yaml
emoji:
  openingCount: "2-3"         # 开头Emoji数量
  inlineDensity: "每2-3句话1个"  # 正文穿插密度
  sentenceDefinition: "以任何标点符号结尾的完整语义单元"
```

1. ❌ 跳过Step 0预查重
