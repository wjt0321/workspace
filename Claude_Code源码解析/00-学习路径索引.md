## 关键设计模式总结

### 可直接借鉴
- **Fork 子 agent**：tool 定义完全相同 → prompt cache 命中最大化
- **工具权限分层**：sync/async agent 工具权限独立配置
- **Context fork**：文件状态缓存克隆，隔离不泄漏
- **退出码 + stderr 双通道泄漏防护**

### 架构参考
- **Subagent 生命周期**：finally 块清理所有资源（MCP/hooks/files/background tasks）
- **Prompt 增强链**：层层叠加（base → CLAUDE.md → agent 定制 → 前置 skill）
- **Recursive 防护**：fork 链路上禁止再 fork（消息内容 + context 双保险）
- **Token budget**：Explore/Plan 排除 CLAUDE.md，节省 5-15 Gtok/week
