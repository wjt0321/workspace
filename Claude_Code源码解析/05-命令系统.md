## 5.2 命令注册流程

### 导入所有命令

```typescript
// 直接导入（始终可用）
import commit from './commands/commit.js'
import diff from './commands/diff/index.js'
import compact from './commands/compact/index.js'
// ... 还有 80+ 个

// 条件导入（Feature Gate）
const coordinatorModeModule = feature('COORDINATOR_MODE')
  ? require('./commands/coordinatorMode.js')
  : null

const assistantCommand = feature('KAIROS')
  ? require('./commands/assistant/index.js').default
  : null
```

### 命令对象结构

```typescript
interface Command {
  name: string           // 命令名，如 'commit'
  description: string    // 描述，供 /help 使用
  options?: Option[]    // 子选项
  
  // 执行函数
  run: (args: string[], context: CommandContext) => Promise<void>
}
```

### 注册到 Commander

```typescript
// 在 getCommands() 中统一注册
export function getCommands(): Command[] {
  return [
    { name: 'commit', run: commitHandler, ... },
    { name: 'diff', run: diffHandler, ... },
    { name: 'review', run: reviewHandler, ... },
    // ... 所有命令
  ]
}
```

## 5.6 命令与工具的区别

| 维度 | Slash Command | Tool |
|------|---------------|------|
| **触发方式** | 用户主动 `/cmd` | AI 主动调用 |
| **执行者** | 用户（用户决定） | AI（AI 决定） |
| **上下文** | 通常需要用户输入参数 | 自动从对话获取 |
| **典型用途** | 初始化操作、配置 | 完成任务步骤 |
| **例子** | `/commit`, `/review` | `Bash`, `Read`, `Edit` |

## 5.7 命令优先级

### 优先级定义

```typescript
const MAX_PRIORITY = 100
const DEFAULT_PRIORITY = 50

// 高优先级：init, help 等基础命令
// 普通优先级：大多数命令
// 低优先级：实验性命令
```

### 优先级队列

```typescript
export function getCommandsByMaxPriority(): Command[] {
  return commands
    .sort((a, b) => (b.priority || 50) - (a.priority || 50))
}
```
