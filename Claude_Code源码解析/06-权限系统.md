## 6.1 权限系统概述

Claude Code 的权限系统是它最核心的安全机制之一：

```
┌─────────────────────────────────────────────────────────────┐
│                      权限检查流程                             │
│                                                             │
│  工具调用请求 ──► 权限模式判断 ──► 规则匹配 ──► 用户决策    │
```

## 6.2 权限模式（Permission Mode）

### 模式类型

| 模式 | 说明 | 行为 |
|------|------|------|
| `default` | 默认模式 | 每次询问用户 |
| `plan` | 计划模式 | 只允许查看，不允许修改 |
| `acceptEdits` | 自动接受编辑 | 允许文件修改，不弹窗 |
| `bypassPermissions` | 绕过权限 | 所有操作都允许（危险） |
| `dontAsk` | 不询问 | 静默拒绝危险操作 |
| `auto` | 自动模式（内部） | AI 根据规则自动决定 |

### 模式配置

## 6.3 权限规则（Permission Rules）

### 规则类型

```typescript
// 规则来源
type RuleSource = 'default' | 'project' | 'user' | 'enterprise'

// 规则结构
interface PermissionRule {
  tool: string           // 工具名，如 'Bash'
  pattern?: string      // 匹配模式，如 'rm *'
  action: 'allow' | 'deny' | 'ask'
  source: RuleSource
}
```

### 规则示例

```typescript
// 默认允许的命令
const defaultAllowRules = [
  { tool: 'Read', action: 'allow' },
  { tool: 'Glob', action: 'allow' },
  { tool: 'Grep', action: 'allow' }
]

// 危险命令需要询问
const askRules = [
  { tool: 'Bash', pattern: 'rm*', action: 'ask' },
  { tool: 'Bash', pattern: 'git push', action: 'ask' },
  { tool: 'Write', action: 'ask' }
]

// 始终拒绝
const denyRules = [
  { tool: 'Bash', pattern: 'sudo*', action: 'deny' },
  { tool: 'Bash', pattern: 'curl*password*', action: 'deny' }
]
```

## 6.4 权限检查流程

```typescript
async function checkPermission(
  toolName: string,
  toolInput: object,
  context: ToolPermissionContext
): Promise<PermissionResult> {
  
  // 1. 检查绕过权限
  if (context.mode === 'bypassPermissions') {
    return { result: 'bypass' }
  }
  
  // 2. 检查始终允许规则
  if (matchRule(toolName, toolInput, context.alwaysAllowRules)) {
    return { result: 'allow' }
  }
  
  // 3. 检查始终拒绝规则
  if (matchRule(toolName, toolInput, context.alwaysDenyRules)) {
    return { result: 'deny' }
  }
  
  // 4. 根据模式决定
  if (context.mode === 'dontAsk') {
    return { result: 'deny' }
  }
  
  if (context.mode === 'acceptEdits' && isWriteOperation(toolName)) {
    return { result: 'allow' }
  }
  
  // 5. 需要询问用户
  return { result: 'ask' }
}
```

## 6.5 危险模式检测

### 6.5.1 文件系统危险模式

```typescript
// src/utils/permissions/dangerousPatterns.ts
const DANGEROUS_PATTERNS = [
  // 删除操作
  /^rm\s/,
  /^rmdir\s/,
  /^del\s/,
  
  // 系统修改
  /^chmod\s/,
  /^chown\s/,
  /^sudo\s/,
  
  // 网络操作
  /^curl.*-d.*password/,
  /^wget.*--post-data/,
  
  // 危险重定向
  />\s*\/dev\/sda/,
  />\s*\/etc\/passwd/
]
```

### 6.5.2 Bash 命令分类

```typescript
// 根据危险程度分类
type BashCategory = 'safe' | 'warning' | 'dangerous'

function categorizeBashCommand(cmd: string): BashCategory {
  if (DANGEROUS_PATTERNS.some(p => p.test(cmd))) {
    return 'dangerous'
  }
  if (cmd.includes('sudo') || cmd.includes('rm')) {
    return 'warning'
  }
  return 'safe'
}
```

## 6.6 自动模式（Auto Mode）

### 分类器决策

```typescript
// 使用 AI 分类器自动决定权限
class PermissionClassifier {
  async classify(
    toolName: string,
    toolInput: object,
    context: ToolPermissionContext
  ): Promise<'allow' | 'ask' | 'deny'> {
    
    // 构造提示
    const prompt = `
      工具: ${toolName}
      参数: ${JSON.stringify(toolInput)}
      命令: ${context.command}
      
      判断是否安全执行？
    `
    
    // 调用分类器
    const response = await callClassifier(prompt)
    return response.decision
  }
}
```

## 6.10 安全设计原则

### 1. 默认拒绝

```typescript
// 未明确允许的，默认拒绝
if (!isExplicitlyAllowed(toolName, toolInput)) {
  return { result: 'ask' }
}
```

### 2. 渐进式信任

```
新用户 ──► 每次询问 ──► 学习模式 ──► 信任模式
  │          │            │            │
  │          ▼            ▼            ▼
  │        Ask         部分允许     大部分允许
  │                                     │
  └─────────────────────────────────────┘
              可随时回退
```
