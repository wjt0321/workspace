## 3.1 核心概念

Claude Code 的上下文分为两类：

| 类型 | 函数 | 说明 |
|------|------|------|
| **System Context** | `getSystemContext()` | 系统级信息（Git 状态等） |
| **User Context** | `getUserContext()` | 用户级信息（.md 文档等） |

两者都被 **memoize** 缓存，缓存在会话期间有效。

---

## 3.2 系统上下文（System Context）

### 包含内容

```typescript
const systemContext = {
  gitStatus: `...`,        // Git 状态快照
  cacheBreaker?: string    // 缓存破坏器（内部使用）
}
```

### Git 状态采集

```typescript
const gitStatus = await Promise.all([
  getBranch(),              // 当前分支
  getDefaultBranch(),       // 主分支
  execFileNoThrow('git', ['status', '--short']),  // 状态
  execFileNoThrow('git', ['log', '--oneline', '-n', '5']),  // 最近提交
  execFileNoThrow('git', ['config', 'user.name'])  // 用户名
])
```

### 状态截断

```typescript
// Git 状态超过 2000 字符则截断
const truncatedStatus = 
  status.length > MAX_STATUS_CHARS
    ? status.substring(0, MAX_STATUS_CHARS) + '\n... (truncated)'
    : status
```

## 3.3 用户上下文（User Context）

### 包含内容

```typescript
const userContext = {
  claudeMd: string | null,  // 从 .md 文件读取的内容
  currentDate: string       // 当前日期
}
```

### Claude.md 文件

Claude Code 会扫描工作目录下的 `.md` 文件：

```typescript
// 扫描 .md 文件
const claudeMd = getClaudeMds(
  filterInjectedMemoryFiles(await getMemoryFiles())
)
```

**作用**：用户可以在项目根目录放置 `.md` 文件，AI 会自动读取并作为上下文参考。

## 3.4 上下文注入流程

```
                    ┌─────────────────┐
                    │  启动会话       │
                    └────────┬────────┘
                             │
                             ▼
                    ┌─────────────────┐
                    │  getSystemContext()│
                    │  (Git 状态)      │
                    └────────┬────────┘
                             │
                             ▼
                    ┌─────────────────┐
                    │  getUserContext()│

                    └────────┬────────┘
                             │
                             ▼
                    ┌─────────────────┐
                    │  合并到 Messages│
                    │  prependUserContext()│
                    └────────┬────────┘
                             │
                             ▼
                    ┌─────────────────┐
                    │  发送给 API     │
                    └─────────────────┘
```

## 3.5 缓存机制

### memoize 使用

```typescript
// 缓存整个会话期间的结果
export const getSystemContext = memoize(async (): Promise<{...}> => {
  // 只在第一次调用时执行
  // 后续调用直接返回缓存值
})

export const getUserContext = memoize(async (): Promise<{...}> => {
  // 同上
})
```

### 缓存清除

```typescript
// 缓存破坏器 - 强制刷新
export function setSystemPromptInjection(value: string | null): void {
  systemPromptInjection = value
  // 清除缓存
  getUserContext.cache.clear?.()
  getSystemContext.cache.clear?.()
}
```

## 3.6 上下文与 System Prompt

上下文最终会被注入到 System Prompt：

```typescript
// 在 query.ts 中
const systemContext = await getSystemContext()
const userContext = await getUserContext()

// prependUserContext 和 appendSystemContext 负责注入
const messages = prependUserContext(messages, userContext)
const messages = appendSystemContext(messages, systemContext)
```
