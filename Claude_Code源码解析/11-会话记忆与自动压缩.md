## 一、Session Memory（会话记忆）

### 机制
Session Memory 是**后台自动记忆提取服务**，在对话过程中周期性运行一个 **forked subagent**，把当前对话的关键信息提取到会话专属的 markdown 文件里。

### 核心设计
```
用户消息 + AI 回复
       ↓
  周期性触发（token 阈值）
       ↓
  forked extraction subagent 启动
       ↓
  分析最新消息 → 提取关键信息
       ↓
  写入会话专属 .md 文件
       ↓
  不阻塞主对话
```

### 关键参数（可配置）
| 参数 | 含义 |
|------|------|
| `minimumMessageTokensToInit` | 积累多少 token 后开始第一次提取 |
| `minimumTokensBetweenUpdate` | 两次提取之间最少积累多少 token |
| `toolCallsBetweenUpdates` | 两次提取之间最少多少次工具调用 |
| `toolCallsBetweenUpdates` | 同上（另一种计数方式） |

### 与我的 memory-extractor.sh 的区别
| | Claude Code Session Memory | 我的 memory-extractor.sh |
|---|---|---|
| 触发 | 自动（token 阈值） | 手动调用 |
| 执行 | forked subagent（不阻塞） | 主 agent 执行 |
| 持久化 | 会话专属文件（当前会话） | 可选升权到 MEMORY.md |
| 阻塞 | 非阻塞 | 阻塞主对话 |

### 借鉴价值
可以做一个**轻量版 Session Memory**：在心跳时顺便检查 token 积累量，够阈值就自动提取关键内容到会话专属文件。

## 二、Auto-Compact（自动压缩）

### Claude Code 的三层压缩

**1. MicroCompact（微压缩）**
- 仅压缩工具结果（tool_result）
- 只保留最近 3 个工具结果
- 其他替换为 `[Old tool result content cleared]`
- 图片限制 2000 token
- **只压缩这些工具**：file_read, bash, grep, glob, web_search, web_fetch, file_edit, file_write

**2. Time-based MC（基于时间的压缩）**
- 按时间周期压缩，不是按 token 阈值
- 配置驱动（`timeBasedMCConfig.ts`）

**3. Full Compact（完全压缩）**
- 调用 LLM 生成摘要
- 替换整个历史为一段摘要
- Forked agent 执行，不阻塞主对话

### Token 预算系统
```
有效上下文窗口 = 模型最大上下文 - 20,000（保留给摘要输出）
                        ↓
        AUTOCOMPACT_BUFFER_TOKENS = 13,000（自动压缩阈值）
        WARNING_THRESHOLD_BUFFER_TOKENS = 20,000（警告阈值）
        ERROR_THRESHOLD_BUFFER_TOKENS = 20,000（错误阈值）
```

### 断路器设计
```typescript
// 连续 3 次压缩失败后停止重试
// 原因：超过上下文限制的会话无法通过压缩恢复，继续尝试浪费资源
const MAX_CONSECUTIVE_AUTOCOMPACT_FAILURES = 3
```

### 图像处理
```typescript
// 用户图片在压缩前剥离，防止 API 调用本身也超限
stripImagesFromMessages(messages)
// 替换为文字标记："[Image: 用户发送了一张图片]"
```

### Skill 文件截断
```typescript
// Skills 可以很大（verify=18.7KB, claude-api=20.1KB）
// 压缩后重新注入时，每个 skill 最多保留 5,000 token
// 5 个 skills × 5K = 25K token 总预算
POST_COMPACT_MAX_TOKENS_PER_SKILL = 5_000
POST_COMPACT_SKILLS_TOKEN_BUDGET = 25_000
```

## 四、最值得借鉴的 3 个设计

### 🥇 微压缩策略（MicroCompact）
**可落地**：OpenClaw 的每日会话压缩可以区分"微压缩"和"完全压缩"：
- 微压缩：只清理旧工具结果，保留对话结构
- 完全压缩：LLM 生成摘要，替换整个历史

### 🥈 Token 预算 + 三层阈值
**可落地**：给 OpenClaw 的上下文管理加上：
- 警告阈值（剩余 20K token 时提示）
- 危险阈值（剩余 10K token 时强制压缩）
- 断路器（连续失败 3 次停止自动重试）

### 🥉 图像预处理剥离
**可落地**：压缩前自动剥离用户发送的大图片，只保留文字标记，防止图片撑满上下文。
