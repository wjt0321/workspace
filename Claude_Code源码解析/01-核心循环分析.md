## 1.1 核心循环的本质

Claude Code 的核心循环其实非常简洁：

```
┌─────────────────────────────────────────────────────────────┐
│                      核心循环                                │
│                                                             │
│  1. User Input → Messages[]                                │
│  2. Send to Claude API                                      │
│  3. Receive Response                                        │
│  4. stop_reason == "tool_use" ?                            │
│     ├── YES → Execute Tools → Append Results → Loop        │
│     └── NO  → Return to User                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**这就是最基础的 Agent Loop**，Claude Code 在这个基础上层层封装。

## 1.3 核心循环代码结构

### 第一层：初始化

```typescript
// 创建预算追踪器
const budgetTracker = createBudgetTracker(params.taskBudget)

// 规范化消息格式
const normalizedMessages = normalizeMessagesForAPI(params.messages)

// 构建 API 配置
const queryConfig = await buildQueryConfig(...)
```

### 第二层：循环主体

```typescript
// 主循环 - 核心是 Claude API 调用
while (true) {
  // 1. 调用 Claude API
  const response = await callClaudeAPI(messages, config)
  
  // 2. 检查响应类型
  if (response.stop_reason === 'tool_use') {
    // 工具调用模式
    const toolResults = await executeTools(response.content)
    messages.push(response)           // 添加 AI 响应
    messages.push(toolResultMessage) // 添加工具结果
  } else {
    // 文本响应模式 - 循环结束
    return response
  }
}
```

### 第三层：工具执行

```typescript
// 工具执行由 runTools() 负责
const toolResults = await runTools(
  response.content.filter(c => c.type === 'tool_use'),
  params.canUseTool,
  params.toolUseContext
)
```

## 1.4 关键类型定义

### 消息类型

```typescript
type Message = UserMessage | AssistantMessage | ToolResultMessage

interface UserMessage {
  type: 'user'
  content: string | ContentBlock[]
}

interface AssistantMessage {
  type: 'assistant'
  content: ContentBlock[]
  stop_reason: 'end_turn' | 'tool_use' | 'stop_sequence'
}

interface ToolResultMessage {
  type: 'tool_result'
  content: ToolResultBlock[]
}
```

### 工具调用

```typescript
interface ToolUseBlock {
  type: 'tool_use'
  id: string
  name: string  // 工具名称，如 'Bash'、'Read'、'Write'
  input: object // 工具参数
}
```

## 1.6 Feature Gate（编译时开关）

Claude Code 使用 Bun 的编译时特性开关：

```typescript
// 编译时决定是否包含某功能
const reactiveCompact = feature('REACTIVE_COMPACT')
  ? require('./services/compact/reactiveCompact.js')
```

## 1.7 流式处理

Claude Code 使用生成器模式实现流式响应：

```typescript
// 返回异步生成器，实现流式输出
export async function* query(params): AsyncGenerator<StreamEvent> {
  for await (const event of stream) {
    yield event  // 实时 yield 每个事件
  }
}
```

**StreamEvent 类型**：
```typescript
type StreamEvent = 
  | { type: 'content_block_delta', delta: string }
  | { type: 'message_delta', usage: Usage }
  | { type: 'error', error: string }
```

## 1.9 关键发现

### 1. 循环控制简洁
```typescript
while (true) {
  const response = await callAPI(...)
  if (response.stop_reason === 'end_turn') break
  const results = await executeTools(...)
  messages.push(...results)
}
```

### 2. 消息驱动
- 所有状态都存在 `messages[]` 数组里
- 工具结果作为 user message 追加
- 下一轮直接用扩展后的消息数组继续

### 3. 编译时优于运行时
- Feature gate 在编译时消除死代码
- 运行时零开销的条件功能

### 4. 生成器模式
- 用 `AsyncGenerator` 实现流式输出
- 每一块内容实时 yield
