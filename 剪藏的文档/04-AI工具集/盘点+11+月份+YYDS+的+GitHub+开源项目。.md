**Open Notebook**


**Open Notebook 是 Google NotebookLM 的一个开源替代品，现在 11.4K+ 的 Star 了。**

### 这个开源项目是一个灵活、隐私保护很强的 AI 笔记和研究工具。

![Image 3](https://mmbiz.qpic.cn/sz_mmbiz_png/ePw3ZeGRruz6XfrlrjNsvYbU05QnNorx9Ygzicfs72TNUT0PKcsiabWFKibk4OPcDH2PvRJ0qmMcmhoYcuT038Z6w/640?wx_fmt=png&from=appmsg#imgIndex=3)

### 它不仅复刻了 NotebookLM 的核心体验，比如基于文档的问答，还解除了对单一模型供应商的依赖，可以自由选择 AI 后端。

而且它具备多模态处理能力，能处理 PDF、视频、音频、网页等多种格式的资料。除了基础的 RAG 问答外。

它还集成了播客生成功能、全文及向量搜索，并提供了完整的 REST API 以供扩展。

![Image 4](https://mmbiz.qpic.cn/sz_mmbiz_png/ePw3ZeGRruz6XfrlrjNsvYbU05QnNorxSdUVWDtuHebWtfaicmVsF99ykIEx5JuSXcSZJ4j08PzsJ7hlwNndELw/640?wx_fmt=png&from=appmsg#imgIndex=4)

```Plain Text
开源地址: https://github.com/lfnovo/open-notebook
```

**AI 黑客智能体：Strix**

**Strix 模拟真实黑客的行为，现在已经 15K+ 的 Star 了。能够自动化地对应用程序进行安全扫描、漏洞发现和验证。**

与传统的静态扫描工具不同，Strix 采用动态的、智能体驱动（Agentic）的方法，旨在减少误报并提供通过实际攻击验证（PoC）的漏洞报告。

**GUI 组件库**


**GPUI Component 是一个基于 Rust 语言的 GUI 组件库，现在 8.7K 的 Star 了，它是建立在 Zed 编辑器开源的高性能 UI 框架 GPUI 之上的。**

该项目的目标是为 Rust 开发者提供一套现代、美观且高性能的通用 UI 组件，极大地降低使用 GPUI 开发跨平台桌面应用的门槛。

![Image 6](https://mmbiz.qpic.cn/sz_mmbiz_png/ePw3ZeGRruz6XfrlrjNsvYbU05QnNorx5xiaNqaERvf5kmWz0jp6FT9qLDzrs4XPdicQdvQibfWeuphlRfLVgDqAw/640?wx_fmt=png&from=appmsg#imgIndex=6)

该库提供了 60 多个现成的 UI 组件，设计风格灵感来源于 shadcn/ui，支持亮色/暗色主题切换和自定义主题。

它继承了 GPUI 的高性能特性，包含虚拟列表、Dock 布局系统、Markdown 渲染、图表以及高性能的代码编辑器组件。

它旨在提供类似 Web 开发的便捷体验，同时保持原生应用的速度。

![Image 7](https://mmbiz.qpic.cn/sz_mmbiz_png/ePw3ZeGRruz6XfrlrjNsvYbU05QnNorxpJgl35IuwZUTntF0IbvlCibE36an2wLGtyWzExykG9uE1NhavGVYn4Q/640?wx_fmt=png&from=appmsg#imgIndex=7)

```Plain Text
项目链接: https://github.com/longbridge/gpui-component
```

**API 客户端：Yaak**


Yaak 是一款极其直观、现代化的桌面 API 客户端，被视为 Postman 或 Insomnia 的有力竞争者。现在 16K+ 的 Star 。

![Image 8](https://mmbiz.qpic.cn/sz_mmbiz_png/ePw3ZeGRruz6XfrlrjNsvYbU05QnNorxacK1DcXRerZ5aDdUhyK2ZVibiaA1efsomuBDEy61ov3zGNXMw0XSoQEQ/640?wx_fmt=png&from=appmsg#imgIndex=8)

它最大的特点是保护隐私和离线优先，为你提供一个快速、无干扰且无云端锁定的 API 调试环境。


Yaak 基于 Tauri、Rust 和 React 构建，体积小巧且启动迅速。它支持 REST、GraphQL、WebSocket、Server Sent Events (SSE) 和 gRPC 等多种协议。

![Image 9](https://mmbiz.qpic.cn/sz_mmbiz_png/ePw3ZeGRruz6XfrlrjNsvYbU05QnNorxLIsNHWdIiaMpC6icsFTibgyTSOByWHZW5FRWvINLV3JjFP9aiaIpSaTvYw/640?wx_fmt=png&from=appmsg#imgIndex=9)

功能上，它支持导入 Postman/OpenAPI 集合、环境与变量管理、Cookie 管理以及通过插件系统进行扩展。

最重要的是，它没有强制登录和遥测，所有数据存储在本地，且易于通过 Git 进行版本控制。

![Image 10](https://mmbiz.qpic.cn/sz_mmbiz_png/ePw3ZeGRruz6XfrlrjNsvYbU05QnNorxVWuJMuyqNPpBXoFVQCMqribYHCwpaPz9a7GbVNBfRqy1ictKwWa6jXmw/640?wx_fmt=png&from=appmsg#imgIndex=10)

```Plain Text
项目链接: https://github.com/mountain-loop/yaak
```

07

**Agent Lightning**

**Agent Lightning 是微软研究院推出的一个框架，被誉为点亮 AI 智能体的终极训练师。**

**解决 AI Agent 开发中的优化难问题：如何像训练神经网络一样，系统性地优化 Agent 的提示词（Prompt）、工具使用策略和决策流程，而无需大规模重写代码。**

![Image 11](https://mmbiz.qpic.cn/sz_mmbiz_png/ePw3ZeGRruz6XfrlrjNsvYbU05QnNorxicK1wnia2BLo4Pqoz8byJLmotFZFaxWtdibpjtJWYicYLdn2fpUibmjwIYg/640?wx_fmt=png&from=appmsg#imgIndex=11)

该框架采用几乎零代码修改的设计理念，兼容 LangChain、AutoGen、CrewAI 等主流 Agent 框架。

它通过 LightningStore 收集 Agent 的运行轨迹，并利用强化学习、自动提示词优化等算法来分析这些轨迹，进而更新 Agent 的资源。

它构建了一个从推理到反馈再到更新的闭环优化流。


![Image 12](https://mmbiz.qpic.cn/sz_mmbiz_png/ePw3ZeGRruz6XfrlrjNsvYbU05QnNorx5Uz0oEQnlTJFJBt4nz27lTV6QPDga7300z19VUqtmwFdX9mjBiaWlEA/640?wx_fmt=png&from=appmsg#imgIndex=12)

Agent Lightning 适用于构建复杂 AI 智能体系统的开发者和研究人员。

当你的 Agent 表现不稳定、任务完成率低时，通过该工具可以引入系统性的优化算法，自动调整 Agent 的行为策略，而无需手动反复调试 Prompt。

```Plain Text
项目链接: https://github.com/microsoft/agent-lightning
```

**Claude Cookbooks**


Claude Cookbooks 是 Anthropic 官方维护的，汇集了大量使用 Claude 大模型的实战案例、Jupyter Notebooks 和最佳实践指南。

现在在 GitHub 上已经快 30K 的 Star 了。

![Image 13](https://mmbiz.qpic.cn/sz_mmbiz_png/ePw3ZeGRruz6XfrlrjNsvYbU05QnNorx9ErhVA7uJiaiaCn1ZdID0vnCCbuoeEa0IXtJNpib94ialH2xDBs70HvH6g/640?wx_fmt=png&from=appmsg#imgIndex=13)

它就像一本食谱，为开发者提供了从入门到精通的各种烹饪方法，展示了如何高效、有趣地利用 Claude 的能力。

这个开源项目内容丰富，涵盖了从基础的 API 调用到复杂的 Agent 构建。主要包括：图像视觉处理、工具使用、长文本处理、RAG、以及最新的扩展思维等高级特性。

![Image 14](https://mmbiz.qpic.cn/sz_mmbiz_png/ePw3ZeGRruz6XfrlrjNsvYbU05QnNorxBbkd99swb49d2ib16AB44IEy2mcHN4ialypTDzgYs4zHvnZFXib9KhicIg/640?wx_fmt=png&from=appmsg#imgIndex=14)

代码主要使用 Python 编写，结构清晰，方便开发者直接复制粘贴或进行微调。对于任何使用 Claude API 的开发者来说，这是必读的参考资料。

无论你是想了解如何优化 Prompt、如何让 Claude 操作外部数据库，还是想构建复杂的自动化工作流，这里都能找到经过官方验证的高质量代码示例。

它是学习 Claude 最新特性的最快途径。

![Image 15](https://mmbiz.qpic.cn/sz_mmbiz_png/ePw3ZeGRruz6XfrlrjNsvYbU05QnNorxv7vJhD5knlmCzPlqAHyYwia6DmJELwKyuwD2bGzL9HFCLYJ8LJ7ibQaA/640?wx_fmt=png&from=appmsg#imgIndex=15)

**x402 支付协议**


x402 是由 Coinbase 开源的一个基于 HTTP 构建的互联网支付协议。现在已经 4.7K 的 Star 了。

命名灵感来源于 HTTP 状态码 "402 Payment Required"。

该项目的愿景是建立一个开放、原生的互联网支付标准，让数字货币支付像数据传输一样简单和普遍，消除传统信用卡支付的高门槛和高手续费。


![Image 17](https://mmbiz.qpic.cn/sz_mmbiz_png/ePw3ZeGRruz6XfrlrjNsvYbU05QnNorxkVx0vIJq6JThLIrYwXdWRJicHEGAglho0ZRpN2kz5VmRzLhfZ2HdoPA/640?wx_fmt=png&from=appmsg#imgIndex=17)

该协议非常适合需要极低成本、高频次交易的场景，例如 API 调用计费、AI  Agent 之间的自动付费、内容付费墙等。

通过 x402，开发者可以用极少的代码，号称仅需一行让自己的服务具备接收数字美元或其他加密资产的能力，极大地促进了机器对机器（M2M）经济网络的发展。

![Image 18](https://mmbiz.qpic.cn/sz_mmbiz_png/ePw3ZeGRruz6XfrlrjNsvYbU05QnNorxpNicff6s3HfTeZSpB9jdt4WyKXF8xoWq4SDQOcQ2eP8qOmEqZMQibj9g/640?wx_fmt=png&from=appmsg#imgIndex=18)

**对象存储系统：**MinIO


MinIO 是全球领先的高性能、S3 兼容的对象存储系统。现在已经  58.7K+ 的 Star 了。

![Image 19](https://mmbiz.qpic.cn/sz_mmbiz_png/ePw3ZeGRruz6XfrlrjNsvYbU05QnNorxjqmicBZKU23z96icSIzsqwCDbxGtfZfd8mF1CxxFoEWByCazLfOMml6w/640?wx_fmt=png&from=appmsg#imgIndex=19)

作为云原生数据基础设施的标杆，它专门为大规模 AI/ML、数据分析和应用数据工作负载而设计。

MinIO 完全开源，不仅可以作为公有云的替代方案，更是私有云和边缘计算场景下的首选存储解决方案。

![Image 20](https://mmbiz.qpic.cn/sz_mmbiz_png/ePw3ZeGRruz6XfrlrjNsvYbU05QnNorxH85hlpX2icrKEvUl4wQXKMGicebPEOpjF9rXz90cb5a0VjiceBgIQX9XQ/640?wx_fmt=png&from=appmsg#imgIndex=20)

它能够充分压榨硬件潜力，提供极高的读写吞吐量。

它完全兼容 Amazon S3 API，意味着现有的 S3 生态工具和应用可以无缝迁移。其架构轻量且去中心化，支持纠删码（Erasure Coding）和比特罗特（Bitrot）保护，确保数据的高可用性和持久性。

它还原生支持 Kubernetes，非常适合在容器化环境中部署。

```Plain Text
项目链接: https://github.com/minio/minio
```
