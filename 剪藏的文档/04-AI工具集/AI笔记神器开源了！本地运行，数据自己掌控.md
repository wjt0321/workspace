## 0x02：为什么我们需要“会思考”的笔记本？

先问一个问题：你现在是怎么做知识管理的？

大多数人可能走的是这几条路：

- 用浏览器收藏夹存网页 → 结果三年没打开过

- 用微信收藏文章 → 收藏即读完

- 用印象笔记/语雀记笔记 → 写完就扔进文件夹吃灰

- 用 Notion 搭知识库 → 搭了半天，最后只用来写周报

问题出在哪？

不是工具不好，而是这些工具都在解决“存储”问题，而不是“思考”问题。

而真正的思考是什么样的？

是这样的：

> 我正在看一篇关于 Transformer 架构的论文，突然想到昨天 B 站那个讲 Attention 机制的视频好像提到了类似观点……于是我去翻了一下 YouTube 笔记，发现确实有个时间点讲得很清楚，接着我又想起上周下载的那个 PyTorch 实战 PDF 里也有相关代码示例……我把这三个资源整合在一起，让 AI 帮我对比分析一下差异，然后生成了一个可视化的小应用来展示不同注意力权重的效果。

这个过程涉及：

- 多种媒体类型（论文 PDF、YouTube 视频、本地文档）

- 跨平台信息关联（从网页到本地文件）

- 实时调用 AI 进行推理

- 输出可交互的结果（不只是文字）

但现实中，你要完成这一套操作，得在至少五个软件之间跳来跳去，手动复制链接、截图、粘贴文本、查时间戳……

**思考还没开始，精力就已经耗尽了。**

所以，我们缺的不是一个更好的笔记软件，而是一个能跟上你思维节奏的“认知加速器”。

而这，正是 Deta Surf 想要解决的问题。

## 0x03：Surf 到底是个啥？一句话说不清的那种

官方给它的定位是：“Your AI Notebook”，你的 AI 笔记本。

但它其实更像是一个 **融合型认知操作系统**。

你可以把它理解为：

- 如果说 Obsidian 是“第二大脑”，那 Surf 就是“带 GPU 的大脑”

- 如果说 Cursor 是“AI 编程助手”，那 Surf 就是“全栈研究员搭档”

- 如果说 Logseq 强调双向链接，那 Surf 直接把链接变成了“活的数据源”

它的核心技术栈也很有意思：前端用 Svelte + TypeScript，后端关键模块用 Rust 打造，整体基于 Electron 构建桌面应用。

也就是说，既有现代 Web 框架的灵活性，又有系统级性能保障。

更重要的是：**完全开源、数据本地存储、支持私有化部署、AI 模型任你选（包括本地运行的大模型）！**

这意味着什么？

意味着你再也不用担心：

- 数据被上传到国外服务器

- 私密资料被训练进商业模型

- 功能受限于订阅套餐

- 被迫使用某个特定厂商的闭源 AI

相反，你可以：

- 把所有资料存在自己电脑上

- 用 Ollama 跑本地 Llama 3 做摘要

- 接入通义千问或讯飞星火做中文理解

## 0x04：核心功能一 —— 多媒体统一图书馆，打破信息孤岛

传统笔记软件最大的痛点是什么？

**它们只能处理“文本”。**

你想引用一段 YouTube 讲解？不好意思，请先手动记下时间戳，再去视频平台找对应片段，再复制描述，再贴进笔记。

你想分析一份 PDF 里的图表？抱歉，大多数工具只能提取纯文字，图片和表格直接丢失。

但在 Deta Surf 里，这一切都不是问题。

### 🌐 它有一个叫“Library”的多媒体图书馆

你可以往里面塞：

- 本地文件（PDF、Word、Excel、PPT、Markdown……）

- 网页链接（知乎回答、公众号文章、新闻报道）

- 视频资源（YouTube、Bilibili 页面链接）

- 社交内容（推文、豆瓣短评）

- 甚至可以直接在里面创建笔记、草稿、待办事项

关键是：这些资源不是简单地“收藏”进去，而是会被 **索引并结构化处理**，变成你可以随时调用的知识节点。

比如你把一个 YouTube 视频加进来，Surf 会自动抓取标题、描述，并且后续可以通过 AI 提取字幕内容，建立时间轴索引。

下次你在写笔记时输入 `@` 符号，就能像@人一样@这个视频，还能精确到某个时间段：

```Plain Text
正如 @YouTube视频[12:34-15:20] 中所说，注意力机制的本质其实是加权求和……
```

保存后点击这个引用，Surf 会直接跳转到该视频的指定时间点播放！

这不就是传说中的“非线性阅读”吗？

### 💾 数据格式也够硬核：SFFS 文件系统

Surf 使用了一个名为 **SFFS（Surf Flat File System）** 的本地存储引擎。

听名字像是操作系统级别的玩意儿，其实它是专门为个人知识管理设计的一套扁平化文件管理系统。

特点有几个：

1. **开放格式**：所有数据以标准 JSON、Markdown、HTML 等明文格式存储，不怕厂商跑路。

2. **易于备份**：整个知识库就是一个文件夹，拖进 iCloud/Dropbox/坚果云就能同步。

3. **可审计性强**：你可以随时打开文件夹查看每个资源的元数据、内容缓存、AI 处理记录。

4. **跨平台兼容**：MacOS、Windows、Linux 全支持。

举个例子，当你收藏一个网页时，Surf 不仅保存 URL，还会把页面主要内容快照下来，哪怕原网站关闭了，你依然能看到当时的内容。

这就解决了“链接腐烂”（link rot）的经典难题。

## 0x05：核心功能二 —— Smart Notes，让笔记自己“活”起来

如果说 Library 是仓库，那 **Smart Notes（智能笔记）** 就是工厂。

传统的笔记是什么？是一块空白画布，等着你一笔一划去填充。

而 Surf 的 Smart Notes 是什么？是一个可以主动获取信息、执行任务、生成结果的“智能代理”。

来看几个真实场景：

### 场景一：边看视频边提问

你在研究“如何优化 React 性能”，打开了一个技术分享视频。

传统做法：边看边暂停，记下关键词，回头再去搜资料验证。

Surf 做法：直接在旁边开个笔记窗口，输入：

> @这个视频，总结一下主讲人提到的三个性能优化技巧，并结合我的项目经验给出改进建议。

Surf 会：

1. 调用 AI 解析视频字幕

2. 提取关键观点

3. 结合你之前写的项目笔记

4. 生成一份定制化的建议报告

全程无需复制粘贴，也不用手动查找上下文。

### 场景二：PDF + Web 搜索联动

你正在读一篇机器学习论文，看到一句“近年来稀疏注意力取得显著进展”，但不太明白具体指哪些工作。

于是你在笔记中写下：

> search: 近年来有哪些代表性的稀疏注意力模型？请列出名称、提出年份、核心思想，并与这篇 PDF 第 8 页的方法做对比。

Surf 会：

1. 自动触发网络搜索

2. 抓取最新研究成果

3. 对比分析异同

4. 插入带引用的表格

输出效果长这样：

|模型|年份|核心思想|与本文对比|
|-|-|-|-|
|Sparse Transformer|2019|固定模式稀疏连接|更灵活，但计算复杂度高|
|Longformer|2020|滑动窗口+全局注意力|类似本文局部注意力设计|

每一条都有来源链接，点击直达原文。

### 场景三：自动插入富媒体内容

你在整理会议纪要，同事发来一张 Excel 表格截图。

传统方式：手动输入数据，或者想办法 OCR 识别。

Surf 方式：直接粘贴图片，然后右键选择“让 AI 解析这张图”。

AI 会：

- 识别表格结构

- 提取数值

- 转换成 Markdown 表格

- 甚至建议画成柱状图

然后你就可以继续追问：“根据这些销售数据，预测下季度趋势。”

整个过程就像有个助理坐在你旁边，随时帮你查资料、做计算、写总结。

## 0x06：最炸裂的功能 —— Surflets，零代码生成交互式应用

如果说前面的功能还算“合理范围内创新”，那接下来这个功能就有点“未来已来”的感觉了。

它叫：**Surflets（发音类似 surf-lets）**

翻译过来就是“小浪花”，寓意是从小小的想法泛起涟漪。

但实际上，它是 **AI 驱动的零代码应用生成器**。

什么意思？

就是你可以在笔记里写一段自然语言，Surf 就能自动生成一个可交互的小程序。

### 来个实际例子：

假设你是产品经理，正在调研用户反馈。

你在笔记里写道：

> 创建一个交互式词云，显示最近 100 条 App 评论中最常出现的词汇，点击某个词能查看包含它的原始评论列表。

回车之后，Surf 会在几秒钟内生成一个嵌入式词云组件，彩色气泡随鼠标浮动放大，点击“卡顿”二字，下方立刻弹出十条相关差评。

整个过程不需要写一行代码。

再比如：

> 做一个滑动条调节器，展示不同温度系数下 Softmax 输出的概率分布变化。

Surf 会生成一个带滑块的图表，左右拖动实时更新曲线，旁边还附带公式说明。

这已经不是笔记了，这是 **可执行的思想容器**。

背后的原理是：

- AI 解析你的需求

- 生成 HTML + CSS + JavaScript 代码

- 在沙盒环境中安全运行

- 嵌入到当前笔记中作为动态组件

你可以把它想象成 Jupyter Notebook 的平民版，但完全通过自然语言驱动。

而且这些 Surflets 还能保存复用，形成你的“个性化工具箱”。

比如：

- 数据清洗小工具

- API 测试调试器

- 学习进度追踪器

- 甚至是小游戏（有人真让 Surf 生成了个贪吃蛇）

这才是 AI 真正该有的样子：不是替代人类，而是增强人类创造力。

## 0x07：AI 模型自由选择，拒绝绑定，拥抱本地化

现在市面上很多所谓的“AI 笔记”，本质上都是某家大厂模型的前端壳子。

你用了它的产品，就得接受它的 API 调用规则、价格策略、数据政策。

但 Surf 完全反其道而行之：**AI 模型你说了算！**

它提供了三种接入方式：

### ✅ 方式一：自带 API Key（云端模型）

如果你愿意用 OpenAI、Anthropic、Google Gemini 等服务，可以直接填入自己的 Key。

Surf 不会中间截留请求，也不会额外收费。

### ✅ 方式二：添加自托管模型

支持 OpenAI 兼容接口的服务都可以接入，比如：

- Ollama（本地运行 Llama 3、Qwen 等）

- LocalAI

- vLLM 部署的私有模型

- FastChat 后端

配置方法也非常简单，在设置里添加一个新模型，填上地址和端口就行。

```Plain Text
name: "My Llama3"
base_url: "http://localhost:11434/v1"
model: "llama3"
api_key: "none"  # 本地不用key
```

从此以后，你的笔记 AI 就在自己电脑上跑了，断网也能用，隐私百分百可控。

### ✅ 方式三：混合模式 + 模型路由

更高级的玩法是设置“模型路由规则”。

例如：

- 中文任务 → 通义千问

- 英文写作 → GPT-4

- 数学计算 → DeepSeek-Math

- 代码生成 → CodeLlama

你可以根据不同场景自动切换最优模型，实现“AI 联邦制”。

这种设计思路非常符合国内开发者的需求——既要先进能力，又要数据安全。

光功能强还不够，用户体验也得跟上。

## 0x09：安全性与开源承诺，真正为用户服务

在这个数据泄露频发的时代，Surf 的安全理念值得点赞。

### 🔒 本地优先（Local First）

默认情况下，所有数据都存在你自己的设备上，不会上传到任何服务器。

即使你后来决定使用 Deta 官方提供的增强版（带云同步等功能），也是明确授权后的选择，而非强制绑定。

### 🛡️ 数据透明

所有内容以开放格式存储，不像某些商业软件用加密数据库锁死用户。

你想迁移？直接拷贝整个文件夹就行。

你想做自动化处理？写个 Python 脚本批量读取 Markdown 文件即可。

### 📜 开源许可证清晰

项目采用 **Apache 2.0 协议**，允许商用、修改、分发。

唯一的例外是用了 Ghostery 的反广告库补丁，遵循 MPL-2.0 许可，这也符合上游规定。

最关键的一点：**Deta 公司明确声明，Surf 的开源版本可以在没有他们服务器的情况下独立运行。**

这不是“伪开源”，而是真正的开源精神践行者。

## 0x0A：我能怎么玩？新手入门指南

说了这么多，你肯定想知道：我该怎么上手？

别急，我已经为你准备好了一份“Surf 极速入门四步曲”：

前往 GitHub Releases 页面： 👉 https://github.com/deta/surf/releases

选择适合你系统的版本（macOS / Windows / Linux）下载安装包。

> ⚠️ 注意：官方网站 deta.surf 提供的版本集成了云服务，属于商业增强版；如果你想纯粹体验开源本地版，请务必从 GitHub 下载。

推荐先试试这几个组合：

- 一个 YouTube 技术视频

- 一篇 PDF 论文

- 一个知乎高赞回答链接

- 几个本地 Markdown 笔记

全部拖进 Library，等待索引完成。

### 第三步：动手试 AI 功能

新建一个笔记，尝试以下命令：

```Plain Text
@YouTube视频，请总结前5分钟讲的核心概念。

search: 这篇论文提到的“动态稀疏化”和MoE架构有什么区别？
```

```Plain Text

```

感受一下什么叫“所想即所得”。

### 第四步：构建专属知识流

试着建立一个主题笔记本，比如“大模型微调实践”。

每天遇到相关内容就丢进去，定期用 AI 做一次聚合梳理：

> 总结本月收集的所有关于 LoRA 优化的经验，按数据准备、超参设置、常见问题分类输出。

你会发现，知识不再是零散的碎片，而是一条不断演进的河流。

Surf 目前还处于早期阶段（v0.x），但它的架构野心显然不止做一个“智能笔记”。

我们可以预见几个发展方向：

### 🧠 成为“个人 AI 操作系统”

整合日历、邮件、待办事项、代码编辑器，成为一个统一的认知工作台。

想想看，早上起床 AI 根据你昨晚的笔记+今日行程+天气情况，自动生成一份“今日作战计划”。

### 🤖 支持 Agent 工作流

引入 AutoGPT 式的自主代理，让它帮你完成一系列复杂任务：
