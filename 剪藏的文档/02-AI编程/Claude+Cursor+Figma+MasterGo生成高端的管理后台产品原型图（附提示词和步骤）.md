初始提示词

```Plain Text
现在我要开发一个企业级的设备管理系统，根据不同的角色设计提示词如下：我想开发一个企业级的设备管理系统要输出原型图，请通过以下方式帮我完成管理系统所有原型页面的设计。 1、作为产品经理先设计出这个管理系统实现哪些功能 2、作为UI设计师完成这些原型界面的设计 3、使用html在一个界面上生成所有的原型界面，使用Tailwind CSS创建高保真UI原型，可从Unsplash获取图片素材，使用FontAwesome等开源图标库，让原型显得更精美和接近真实 4、我希望这些界面可直接用于企业级项目开发 帮我把这个提示词更加丰富，达到做出一个非常高端企业级管理系统的要求，最后把提示词给我
```

完整提示词

```Plain Text
我要开发一个企业级的设备管理系统（Equipment Management System），需要输出完整的高保真原型图。请按照以下角色和要求，系统性地完成所有原型页面的设计：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

作为资深产品经理，请完成以下设计：
【1. 用户角色与权限体系】
- 定义完整的用户角色：超级管理员、系统管理员、设备管理员、维护人员、普通员工、访客等
- 设计基于RBAC的权限控制模型
- 规划各角色的功能访问矩阵
【2. 核心功能模块设计】
请设计以下模块的详细功能：
📊 工作台/仪表盘（Dashboard）
- 关键KPI指标卡片（设备总数、在线率、故障率、待维护数量）
- 实时设备状态监控地图/分布图
- 预警信息滚动展示
- 近期任务/待办事项
- 数据趋势图表（周/月/年维度）
🖥️ 设备管理模块
- 设备台账（设备列表、高级筛选、批量操作）
- 设备详情页（基本信息、运行参数、维护记录、关联文档）
- 设备分类管理（多级分类树）
- 设备位置/区域管理
- 设备二维码/条形码管理
- 设备生命周期管理（采购→使用→维护→报废）
- 设备调拨/借用管理
🔧 维护管理模块
- 维护计划制定（周期性/一次性）
- 维护工单管理（创建、派发、执行、验收）
- 维护记录查询
- 备件管理与消耗统计
- 维护成本分析
⚠️ 预警与告警模块
- 告警规则配置
- 实时告警列表
- 告警处理流程
- 告警统计分析
- 告警通知设置（短信/邮件/站内信/企微/钉钉）
📈 数据报表模块
- 设备运行报表
- 维护统计报表
- 成本分析报表
- 自定义报表生成器
- 报表导出（Excel/PDF）
- 报表订阅与推送
⚙️ 系统管理模块
- 组织架构管理
- 用户管理
- 角色权限管理
- 操作日志审计
- 数据字典管理
- 系统参数配置
📱 其他功能
- 消息中心/通知中心
- 个人中心（资料修改、密码修改、偏好设置）
- 帮助文档/使用指南
- 登录/注册/忘记密码页面

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- 主色调：专业蓝（#1890FF 或自定义品牌色）
```

### Figma

**使用 html.to.design 插件**

**1. 打开 Figma → 右上角菜单 → Plugins → 搜索 "html.to.design"
2. 安装插件后，在浏览器中打开 HTML 文件
3. 复制页面 URL（本地文件用 file:// 协议或启动本地服务器）
4. 回到 Figma，运行插件 → 粘贴 URL → Import
5. 插件会自动将页面转换为可编辑的 Figma 图层**

### MasterGo

**即时设计导入步骤：**

**1. 打开 即时设计 (js.design)
2. 使用「即时 AI」或「设计助手」功能
3. 上传 HTML 文件或输入在线 URL
4. 自动解析为可编辑图层**

**可以用于参考设计不同的管理后台，添加不同的功能进行优化**
