不仅仅是节点的使用数量不同，每个人想要实现的最终结果也有差异。 




比如，6个节点的视频主最终生成的视频URL（站点）链接，只会出现在当前工作流结果中，下次想调用的话，就得打开「N8N」再把当前工作流运行一遍才会有结果。 




8个节点的那个视频，主要实现的是视频生成后自动同步上传到各社交媒体账号上直接发布。 




而我的9个点的工作流中，有Google sheet的表格记录，所有生成的视频URL会被记录于表格内，同时包括生成视频时用到的Prompt（提示词）也会被自动记录。 




那么，要掌握什么情况下该用到什么节点，对于我们初学者来说，我觉得至关重要。 




接下来，我将分享自己整理出来的「N8N节点」中我们最好能看懂的60个节点（值得收藏）：




01

## **触发器类节点（Triggers）**







> 它们就像「开关」：一旦条件满足，就启动整个流程。

- 

    ### **⚡ Activation Trigger「激活触发」**

    ### **像「启动钥匙」，当你打开或停用工作流时，它第一个亮起。**

👉 **功能**：检测激活事件。

🔧 节点参数**设置**：选择何时触发、是否记录。




- 

    ### **⏰ Schedule Trigger「定时触发」**

就像闹钟一样，每隔一段时间就叫醒工作流。

👉 **功能**：定时执行任务。

🔧 节点参数**设置**：Cron 表达式、间隔时间。




- 

    ### **🧠 Manual Trigger「手动触发」**

手动按钮，一点就跑。

👉 **功能**：测试或临时触发流程。

🔧 节点参数**设置**：无。




- 

    ### **💬 Chat Trigger / MCP Server Trigger「消息触发/MCP服务器触发」**

像是群聊里的「唤醒词」或服务器事件监控。

👉 **功能**：消息输入即触发。

🔧 节点参数**设置**：关键词、来源、监听事件。




- 

    ### **📩 Email Trigger (IMAP)「邮件触发」**

邮箱侦探，有新邮件就出动。

👉 **功能**：检测新邮件触发工作流。

🔧 节点参数**设置**：邮箱账号、文件夹路径。




- 

    ### **🚨 Error Trigger「错误触发」**

这个触发就像火警报警器，哪里出错马上报警。

👉 **功能**：捕获错误并启动应急处理。

🔧 节点参数**设置**：错误来源、通知方式。




- 

    ### **🧲 Evaluation Trigger「条件触发」**

条件触发器，像“温度到达30℃就启动空调”。

👉 **功能**：当条件满足自动启动流程。

🔧 节点参数**设置**：监控字段、阈值。




- 

    ### **📥 Webhook / Respond to Webhook「外部请求触发」**

可以说像是智能门铃，有人按就触发。

👉 **功能**：接收外部请求，执行任务。

🔧 节点参数**设置**：路径、请求方法（GET/POST）、返回方式。




### **🧩 Workflow Trigger「工作流触发」**

像“内部连锁反应”，别的工作流触发当前工作流。

👉 **功能**：监听其他工作流事件。

🔧 节点参数**设置**：允许外部调用、工作流ID。




02

数据处理类节点（Data & Logic）




> 这些节点是「数据加工厂」：负责筛、合、算、改、组合。




- 

    ### **📊 Aggregate「聚合」**

有点像Excel里的“汇总表”。

👉 **功能**：按字段分组求和或计数。

🔧 节点参数**设置**：分组字段、聚合方式（Sum/Avg）。




- 

    ### **🧮 Compare Datasets「表单对比」**

两张表对比神器。

👉 **功能**：比出新增、缺失或不同记录。

🔧 节点参数**设置**：匹配字段、输出类型。




- 

    ### **🧹 Filter「过滤」**

筛子一样过滤数据。

👉 **功能**：筛选符合条件的项目。

🔧 节点参数**设置**：条件表达式、逻辑关系。




- 

    ### **🧩 Merge「合并」**

两条水流汇合成一条。

👉 **功能**：合并两组数据。

🔧 节点参数**设置**：合并模式（Append/Merge by Key）。




- 

    ### **🔁 Loop Over Items (Split in Batches)「循环」**

大任务分小批跑。

👉 **功能**：把大数据分成批次执行。

🔧 节点参数**设置**：每批数量、循环规则。




- 

    ### **✂️ Split Out「拆分」**

像拆箱机。

👉 **功能**：把数组拆成单条记录逐个处理。

🔧 节点参数**设置**：字段名。




- 

    ### **🧩 Sort「排序」**

排序员。

👉 **功能**：按字段升降序排列。

🔧 节点参数**设置**：字段、顺序方向。




- 

    ### **🧠 If / Switch「条件/分支」**

红绿灯 & 分流阀。

👉 **功能**：条件判断，选择分支。

🔧 节点参数**设置**：条件表达式、匹配值。




- 

    ### **🧍‍♂️ Limit「限制」**

像“节流阀”。

👉 **功能**：限制输出数据条数。

🔧 节点参数**设置**：最大数量。




- 

    ### **🧱 Edit Fields (Set) / Rename Keys「编辑」**

字段编辑器。

👉 **功能**：修改、重命名、删除字段。

🔧 节点参数**设置**：字段对应表。




- 

    ### **🧹 Remove Duplicates「去重」**

Excel里的“去重”功能。

👉 **功能**：删除重复记录。

🔧 节点参数**设置**：对比字段。




- 

    ### **🕰️ Date & Time「日期时间」**

时间管家。

👉 **功能**：计算日期、格式化时间。

🔧 节点参数**设置**：时区、格式。




- 

    ### **🔍 Evaluation「判断评估」**

判断器。

👉 **功能**：比较字段、输出True/False。

🔧 节点参数**设置**：逻辑表达式。




- 

    ### **⛔ Stop And Error「终止」**

紧急刹车。

👉 **功能**：终止流程并返回错误信息。

🔧 节点参数**设置**：自定义错误内容。




03

文件与系统类节点（File & System）




> n8n 的「文件管家 + 系统操作员」，负责在电脑、本地磁盘或服务器之间搬运、生成、监控和处理各种文件与命令，让数据流真正“落地执行”。

###  ****

- 

    ### **📦 Compression「压缩」**

像打包zip。

👉 **功能**：压缩/解压文件。

🔧 节点参数**设置**：操作类型、文件名。




- 

    ### **📄 Convert to File「转换至文件格式」**

把文字或JSON变文件。

👉 **功能**：生成TXT、CSV等文件。

🔧 节点参数**设置**：文件名、格式。




- 

    ### **📂 Read/Write Files from Disk「本地读写」**

文件搬运工。

👉 **功能**：读写本地文件。

🔧 节点参数**设置**：路径、操作方式。




- 

    ### **📑 Extract From File「从文件提取」**

像“数据挖掘机”。

👉 **功能**：从PDF/CSV提取信息。

🔧 节点参数**设置**：文件类型、提取字段。




- 

    ### **🗂️ Local File Trigger「文件触发」**

监控文件夹变化。

👉 **功能**：检测新文件或修改。

🔧 节点参数**设置**：路径、监听模式。




- 

    ### **🧰 FTP**

老派中转站。

👉 **功能**：上传或下载FTP文件。

🔧 节点参数**设置**：主机、用户名、路径。




- 

    ### **💻 Execute Command / SSH「远程控制」**

像远程控制台。

👉 **功能**：执行系统命令或远程操作。

🔧 节点参数**设置**：命令、主机、认证方式。




04

网络与接口类节点（Integration）




> n8n 的「数据信使与桥梁」，负责把你的工作流和外部世界连接起来——发请求、收数据、对接 API，让不同系统之间自由“对话”。




- 

    ### **🌍 HTTP Request「站点请求」**

最万能的“数据快递员”。

👉 **功能**：向任意网站发GET/POST请求。

🔧 节点参数**设置**：URL、头信息、Body内容。




- 

    ### **🧭 GraphQL**

聪明的HTTP版本。

👉 **功能**：发GraphQL请求并获取结构化数据。

🔧 节点参数**设置**：查询语句、变量。




- 

    ### **🔐 JWT**

数字身份生成器。

👉 **功能**：生成或验证JWT令牌。

🔧 节点参数**设置**：密钥、算法、有效期。




- 

    ### **🧬 LDAP**

公司通讯录。

👉 **功能**：读取LDAP用户信息。

🔧 节点参数**设置**：服务器地址、凭证。




- 

    ### **🧩 Git**

版本控制器。

👉 **功能**：拉取/提交代码仓库。

🔧 节点参数**设置**：仓库URL、分支、密钥。




- 

    ### **📡 RSS Read / RSS Feed Trigger**

新闻订阅器。

👉 **功能**：读取或监听RSS源更新。

🔧 节点参数**设置**：RSS URL、刷新时间。




- 

    ### **🧱 HTML / Markdown / XML**

文本格式化三剑客。

👉 **功能**：互转HTML、Markdown、XML。

🔧 节点参数**设置**：输入内容、转换方向。




05

内容与 AI 类节点（Content & AI）

> 这些节点像是「聪明脑袋 + 表达高手」，能写、能理解、还能帮你处理语言类任务。




- 

    ### **🤖 AI Transform**

AI 智囊团。

👉 **功能**：改写、总结、分类文本。

🔧 节点参数**设置**：模型类型、Prompt 内容。




- 

    ### **🧠 AI Agent**

像你的“私人智能助理”，能听懂上下文、自动调用工具、执行复杂任务。

👉 **功能**：与多节点互动，让 AI 根据任务自动决策，比如调用 API、搜索数据、发送消息。

🔧 节点参数**设置**：

- 选择 AI 模型（如 GPT、Claude 等）

- 配置可用工具（如 HTTP Request、Data Table 等）

- 设置记忆模式（是否保留上下文）




- 

    ### **🧾 Summarize**

AI 总结助手。

👉 **功能**：提炼文章要点。

🔧 节点参数**设置**：摘要长度、语气风格。




- 

    ### **🧩 Data Table (Beta)**

迷你数据库。

👉 **功能**：建表、查数据、更新字段。

🔧 节点参数**设置**：表名、操作类型。




- 

    ### **📸 Edit Image**

小型 PS。

👉 **功能**：裁剪、旋转、加文字。

🔧 节点参数**设置**：操作类型、尺寸。




- 

    ### **💌 Send Email / Respond to Chat**

智能客服。

👉 **功能**：自动发邮件或聊天回复。

🔧 节点参数**设置**：收件人、内容、模板。




06

系统辅助与调试类（Helpers）




> 像 n8n 工作流里的“幕后军师”和“修理工”，专门帮你检查流程、传话递值、排错补丁，确保整个自动化系统顺畅高效地运转。




- 

    ### **🧰 Debug Helper**

放大镜。

👉 **功能**：查看节点输出数据。

🔧 节点参数**设置**：是否显示完整值。




- 

    ### **🔄 Execution Data**

录像机。

👉 **功能**：回放历史执行记录。

🔧 节点参数**设置**：执行ID、时间段。




- 

    ### **🧩 Execute Sub-workflow**

调用子程序。

👉 **功能**：在一个流程中运行另一个。

🔧 节点参数**设置**：子流程ID、输入参数。




- 

    ### **🕐 Wait**

暂停键。

👉 **功能**：延迟或等待事件发生。

🔧 节点参数**设置**：时间、触发条件。




- 

    ### **🔢 TOTP**

动态验证码生成器。

👉 **功能**：生成一次性密码。

🔧 节点参数**设置**：密钥、时间间隔。




- 

    ### **🧍‍♂️ No Operation (Do nothing)**

名字说明一切，什么也不用操作。

👉 **功能**：不执行任何动作，常用于调试。

🔧 节点参数**设置**：无。
