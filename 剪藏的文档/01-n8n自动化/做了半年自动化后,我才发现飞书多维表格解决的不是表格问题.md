# 做了半年自动化后,我才发现飞书多维表格解决的不是表格问题

Original 林月半子聊AI [林月半子的AI笔记](javascript:void(0);)*2025年11月4日 17:30* *浙江*

上个月在 WaytoAGI 做 n8n 共学的时候，我以为大家会被“节点连接”、“逻辑判断”这些概念卡住。




结果不是。




大家真正卡住的地方，是数据往哪儿存。

有个学员做小红书自动化，抓到的数据不知道怎么存；另一个想做定时提醒，表单提交的内容存哪儿也懵了。

有人用 Google Sheets，连不上；有人试 n8n 自带的 Data Tables，结果发现团队其他人想看数据还得教他们怎么进 n8n，太麻烦了。

共学期间，好多学员私下找我，问的都是：“老师，那个飞书多维表格是干啥的？我看小红书案例里你用了，但不知道怎么上手😭”

![Image](https://mmbiz.qpic.cn/mmbiz_png/ZFDQVGCJicKCjR7FfZaM69l3HTM2uw80VjIibdmmFJjXjgsmh28Jib9hNfwpsgjwMlwiaPlSSWf4EKRcGwAaUcXFNQ/640?wx_fmt=png&from=appmsg#imgIndex=0)

看到这些消息的时候，我愣了一下。因为我自己用飞书多维表格快一年了，在我的认知里，它就是——n8n 最好的数据存储搭档。




但我突然意识到，或许对很多人来说，飞书多维表格 这六个字，可能只是个模糊的概念。




## 那么，什么是飞书多维表格?




如果你用过 Excel，那你可以这样理解:

🎯

飞书多维表格 = 会自动干活的 Excel + 能直接用的数据库




听起来很玄?其实很简单 ——

你在 Excel 里要手动复制粘贴、手动筛选、手动提醒自己的事，在飞书多维表格里，全都能自动搞定。而且它还能:

- 用表单收集数据，自动进表(不用你一个个复制粘贴)

- 跟 n8n、API 这些工具无缝对接(不用折腾导入导出)

- 权限管理简单，分享链接就能让团队协作(不用教队友怎么用)




最关键的是——你不需要懂数据库，也不需要写代码

以前我做自动化，最头疼的就是数据存储。现在有了飞书多维表格，我只需要设计好字段，剩下的事情它都能帮我搞定。

而且，飞书多维表格和飞书剥离了，访问 https://base.feishu.cn 就能直接用，不需要下载飞书。

看起来挺方便的，对吧?




点几下配置，提交就能生成，确实省事。

但用了几天后，我发现了一个问题 —— 字段捷径只管生成，生成完了就结束了。




如果我想:

- 生成后自动通知用户"你的卡片好了" → 字段捷径做不到

- 失败了自动重试几次，而不是手动重新点 → 字段捷径做不到

- 把生成的卡片自动发到飞书群里 → 字段捷径做不到




但如果换成 飞书多维表格工作流 + n8n，就不一样了 ——




你可以完全控制整个流程:

- 生成前做什么(下载图片、校验格式)

- 生成中做什么(调用API、错误重试)

- 生成后做什么(上传结果、发送通知)

    


这才是真正的"全自动"。




### 方案二:自动化 + n8n 工作流




而且配置起来，比你想象的简单。

飞书多维表格有个强大的功能 —— 工作流，它可以在表格里发生某件事时，自动触发后续动作。




比如:

- 当有新记录添加时 → 触发 n8n 处理

- 当某个字段被修改时 → 发送通知

- 当满足某个条件时 → 执行某个操作

    


我们要做的，就是让"表单提交时"自动触发 n8n，然后由 n8n 来控制整个卡片生成流程。




#### 第一步:在飞书多维表格里创建工作流

点击左下角的 "+" 号，选择"工作流":

#### 第二步:设置触发条件

现在我们要告诉飞书多维表格：什么时候触发工作流?

在"当发生"这里，选择 "添加新记录时":

![Image 9](https://mmbiz.qpic.cn/mmbiz_png/ZFDQVGCJicKCjR7FfZaM69l3HTM2uw80VaNYsLylGkrMJC8CwibUCuj22QD5UDjcFkicUTclJ7ypKCQGPeicP0ickIQ/640?wx_fmt=png&from=appmsg#imgIndex=15)

然后设置触发条件:

![Image 10](https://mmbiz.qpic.cn/mmbiz_png/ZFDQVGCJicKCjR7FfZaM69l3HTM2uw80V9LWbD4ibHCT4MndwSKe3Wvj6XXbVg0M7G5LIMxMHuyDwPQazvvWSxBA/640?wx_fmt=png&from=appmsg#imgIndex=16)

这里我勾选了 "X账号截图" 这个字段，意思是:

只有当用户上传了 X 截图时，才触发工作流。

这样可以避免用户提交了空表单，导致工作流白跑。




#### 第三步:添加执行动作 —— 发送 HTTP 请求

触发条件设置好了，接下来要告诉飞书多维表格：触发后要做什么?

在"就执行"这里，点击下拉菜单，选择 "发送 HTTP 请求":

![Image 11](https://mmbiz.qpic.cn/mmbiz_png/ZFDQVGCJicKCjR7FfZaM69l3HTM2uw80Vtg3bRCFF39q7yX47tAky0gnbdZ5C54X8s9eOztXnjbOuEOH3FDIichQ/640?wx_fmt=png&from=appmsg#imgIndex=17)

这一步很关键 —— 它是飞书多维表格连接 n8n 的桥梁。

简单理解:

- 当有新记录添加时，飞书多维表格会"打电话"给 n8n

- "电话号码"就是 n8n 的 Webhook URL

- n8n 接到"电话"后，就开始执行后续的卡片生成流程

现在我们来配置这个"电话":

![Image 12](https://mmbiz.qpic.cn/mmbiz_png/ZFDQVGCJicKCjR7FfZaM69l3HTM2uw80VibqWvDVI1Aoqp1tPb5TibNjG85QDdY0Tl9pLM5xm9au5TSbAu7Ycskbw/640?wx_fmt=png&from=appmsg#imgIndex=18)

请求地址(URL): 这里填的是 n8n 提供的 Webhook URL。格式类似:

```Plain Text
https://n8n-xxxxxx.com/webhook/34ec37ea-930d-4162-8e87-b7652df8fdec
```

请求头(Headers)：为了安全起见，我们约定：请求头必须包含 api-key: foobar 才允许访问。

这样可以防止别人随便调用你的 n8n 工作流。




配置完成后，别忘了点"保存并启用"这个工作流。

![Image 13](https://mmbiz.qpic.cn/mmbiz_png/ZFDQVGCJicKCjR7FfZaM69l3HTM2uw80V4NDzkzcmkvmyc34xa6xBk70bBgxdqmOR9yg9JibRJXibdcmtK4wNboOQ/640?wx_fmt=png&from=appmsg#imgIndex=19)

然后你可以在飞书多维表格的"工作流 → 运行日志"里，看到每次触发的记录:

![Image 14](https://mmbiz.qpic.cn/mmbiz_png/ZFDQVGCJicKCjR7FfZaM69l3HTM2uw80VBicKia22mvAVLGsx3icZLgX1JfaZqZfn462dRDoZIAFoQSrEgNzWQIKBQ/640?wx_fmt=png&from=appmsg#imgIndex=20)

这个日志很重要 —— 它能告诉你:

- ✅ 工作流有没有被成功触发

- ✅ HTTP 请求有没有发送成功

- ✅ n8n 有没有收到数据

如果日志显示"运行成功"，就说明飞书多维表格和 n8n 已经打通了!

## 那 n8n 那边怎么配置?




简单说一下流程(具体配置就不展开了):

1. 1.在 n8n 里创建一个 Webhook 节点，它会自动生成一个 URL

2. 2.把这个 URL 复制到飞书多维表格的"发送 HTTP 请求"里

3. 3.在 n8n 里配置后续流程:

4. 

    - 接收飞书多维表格发来的数据

    - 下载用户上传的 X 截图

    - 调用即梦 4.0 API 生成未来风卡片

    - 把生成的卡片链接写回飞书多维表格

    - (可选)发飞书消息或邮件通知用户

整个流程长这样:

![Image 15](https://mmbiz.qpic.cn/mmbiz_png/ZFDQVGCJicKCjR7FfZaM69l3HTM2uw80VCmZibYBDPibyqodpvZRNsz3sMibclwxR1aOAibpFE8qXIibCvNDkRnSiauWQ/640?wx_fmt=png&from=appmsg#imgIndex=21)

看起来有点复杂?

别担心，这个工作流里有20多个节点，如果每个都展开讲，又是一篇万字长文了😂  

而且这篇文章的重点是飞书多维表格，不是 n8n。




所以这里我只说重点:

飞书多维表格在这个流程里做了什么?

1. 接收数据 - 用户填表单，数据自动进多维表格 

2. 触发自动化 - 有新记录时，自动"打电话"给 n8n 

3. 存储结果 - n8n 生成完卡片，写回多维表格  




飞书多维表格既是起点，也是终点，还是数据中心。




具体 n8n 每个节点怎么配置，我在 WaytoAGI 的 n8n 零基础训练营第三课里讲得很详细，这个案例就是那节课的变种。感兴趣的同学可以看这里：AI训练营第三课：商单case拆解，带你搭建复杂工作流系统

📌

关键是 —— 飞书多维表格在这里扮演了什么角色?

- 它是数据的起点(表单提交 → 触发工作流)

- 它是数据的中心(所有信息都存在这里，随时可查)

- 它是自动化的触发器(新记录 → 自动调用 n8n)




这就是我说的"n8n 最好的数据存储搭档"。
