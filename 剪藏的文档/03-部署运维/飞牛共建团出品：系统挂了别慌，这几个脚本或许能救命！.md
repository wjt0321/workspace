# 飞牛共建团出品：系统挂了别慌，这几个脚本或许能救命！

Original 知新坊 [知新坊](javascript:void(0);)*2025年11月9日 18:50* *浙江*

在折腾 NAS、服务器或 Linux 系统的路上，总会遇到各种稀奇古怪的问题，DIY 用户，在玩飞牛 NAS 的过程中，更新挂了、依赖坏了、Docker 崩了、GPU 不认了，这些情况都不算稀罕。

今天就给大家推荐几个实用脚本——由飞牛共建团的几位大佬提供，能帮你快速解决常见的棘手问题，也能在一定程度上减轻官方和共建团成员的支持压力。

另外遇到问题可以先去社区搜索关键词一般都有解决办法的，社区已经整理了很多问题点和处理教程，社区资源要利用起来哦！

脚本

**1️⃣（推荐）飞牛依赖修复脚本 作者：谢佬**

最顶的一个脚本，官方出品修复依赖的脚本，如果你更新不了，FN ID开启不了，经常出现小问题请执行它

```Plain Text
curl http://qdnas.icu/fixapt.sh | bash
```

**2️⃣ GPU编解码能力检测 作者：青团&友人**


此脚本可以直接查询你的NAS硬件和软件支持的视频格式

```Plain Text
curl -sSL http://qdnas.icu/check_gpu.sh | sed 's/\r$//' | bash
```

**3️⃣ 飞牛docker重置脚本 作者：七月七夕**

docker出现问题的可以使用这个脚本，虽然删除掉/etc/docker/daemon.json的配置文件也能重置，但是总归没脚本操作方便

```Plain Text
curl http://qdnas.icu/docker_reset.sh | bash
```

4️⃣重新开启SWAP脚本 **作者：七月七夕**

重新开启SWAP，这个脚本是给用服务器做飞牛的人用的，理论上其他debian系的都可以用。

```Plain Text
curl http://qdnas.icu/swap_manage.sh | bash
```

**5️⃣飞牛聚合脚本 作者：又菜又爱玩的小朱猪**

一个比较齐全执行脚本的集合库，可以修复图中描述的问题如IOMMU直通、挂载CD/DVD、阵列状态和修复，推荐使用他的一键开启IOMMU直通和网络诊断

```Plain Text
git clone https://gitee.com/xiao-zhu245/fnscript.git
cd fnscript/
python3 menu.py
```
