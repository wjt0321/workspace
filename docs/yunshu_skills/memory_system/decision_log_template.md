# 决策日志模板 (Decision Log Template)

此文档用于记录项目开发过程中的关键技术决策（Architectural Decision Records - ADR），以防止“决策断层”。

## ADR-[序号]: [决策标题]
*   **日期**: YYYY-MM-DD
*   **背景**: [简述做出该决策前的背景、痛点或问题]
*   **决策**:
    *   [决策点1]
    *   [决策点2]
*   **后果**: [该决策带来的影响，包括正面的改进和负面的代价/重构需求]

---
*(示例)*
## ADR-001: 采用模块化架构
*   **日期**: 2024-01-01
*   **背景**: 单一脚本文件过大，难以维护。
*   **决策**: 将功能拆分为 `skills/` 和 `scripts/` 两个目录。
*   **后果**: 提高了代码复用性，但增加了文件跳转的复杂度。
