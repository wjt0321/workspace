# 状态快照模板 (State Snapshots Template)

此文档用于记录项目在关键节点的完整状态快照。
旨在解决“上下文丢失”问题，不仅记录代码，还记录运行时的关键数据、环境变量配置及临时文件状态。

## 框架结构 (Framework Structure)

### 1. 快照触发条件 (Trigger Conditions)
*   **里程碑完成时**: 如“完成核心功能模块开发”。
*   **复杂调试中断时**: 当需要暂停调试任务，第二天继续时。
*   **环境变更时**: 升级依赖库或更改配置。

### 2. 记录模板 (Snapshot Template)

#### [日期] 快照标题
*   **当前任务**: [正在进行的任务 ID 或描述]
*   **关键文件状态**:
    *   `main.py`: [Hash/Version] - 修改了...
    *   `config.json`: [关键配置项]
*   **运行时数据**:
    *   上次成功运行的命令: `python ...`
    *   输出结果: [文件路径或日志摘要]
    *   关键变量值: `debug=True`, `threshold=0.5`
*   **遗留问题 (Pending Issues)**:
    *   [ ] 问题1...
    *   [ ] 问题2...
