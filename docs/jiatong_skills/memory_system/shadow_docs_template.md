# 影子文档模板 (Shadow Documentation Template)

此文档作为代码的"影子"，记录那些无法直接写在代码注释里，但对理解代码至关重要的**隐性知识**。
旨在捕捉"代码背后的故事"，特别是那些"为什么不这样做"的反直觉知识。

## 框架结构 (Framework Structure)

### 1. 记录原则
*   **不重复**: 不要复制粘贴代码注释。
*   **解释"为什么"**: 重点解释设计背后的权衡 (Trade-offs)。
*   **记录"坑"**: 记录尝试过但失败的路径 (Anti-patterns)。

### 2. 知识库 (Knowledge Base)

#### KB-[序号]: [知识点标题]
*   **关联代码**: `[path/to/file.py]`
*   **现象**: [描述遇到的奇怪现象或问题]
*   **隐性知识**: [解释背后的深层原因、库的限制或业务的特殊逻辑]
*   **当前方案**: [我们是如何绕过这个坑的]

---
*(示例)*
#### KB-001: 外部库的奇怪行为
*   **关联代码**: `utils/helper.py`
*   **现象**: 函数 A 在参数为 None 时会报错，而不是返回默认值。
*   **隐性知识**: 这是一个已知 Bug，官方文档未说明。
*   **当前方案**: 在调用前必须显式检查 `if arg is not None`。
