## 📋 9步标准流程

### 🔍 Step 0: 操作前准备 - **预查重步骤**
**优先级最高**：此步骤必须在任何其他步骤之前执行

**执行要求**：
- ✅ 必须检查总索引：仔细阅读`美文总索引.md`的完整内容
- ✅ 预判重复风险：了解各文件夹现有文章，避免收集重复内容
- ✅ 记录潜在重复：识别可能重复的作品

**检查内容**：
- 各文件夹现有文章：查看5个文件夹的收录情况
- 作家分布：了解已收录作家的作品分布
- 重复关系：查看跨文件夹重复文章的记录
- 数量统计：了解各文件夹的完成度

**🚨 果果的教训**：
- 必须先读取`美文总索引.md`
- 必须确认拟收录作品的现有数量（不超过2篇）
- 不可跳过此步骤

---

### 🚀 Step 1: 收集片段 (prose-snippet-collector)

**执行要求**：
- 片段长度标准：
  - 小学生：150-300字
  - 初中生：200-400字
  - 高中生：300-500字
  - 大学生：300-500字
  - 成人：300-500字
- 开头Emoji要求：必须添加2-3个主题相关Emoji组合
- 正文穿插Emoji：每2-3句话1个，与内容高度贴合（所有级别统一标准）
- 句子定义：以任何标点符号结尾的完整语义单元

**📂 输出文件**：
- `d:\workspace\{级别}美文\{级别}美文精选100篇.md`
- 格式必须100%符合skill示例

**⚠️ 特别注意**：
- 🚨 **必须先用Glob查找是否存在标准文件名**
- 🚨 **不可自创文件名**
- 正确示例：`高中生美文/高中生美文精选100篇.md`
- 错误示例：`高中生美文/美文收集.md`（这是果果犯的错误）

**格式示例**：
```markdown
### {序号}. 《{作品名}》- {作者}

**经典片段：**

> {开头Emoji组合}
>
> {穿插Emoji的片段内容...}

**推荐理由：**

{推荐语}

**完整文章：**
[[{序号}.{作品名}-{作者}]]

**文学价值：**
- {价值点1}
- {价值点2}

---
```

### 📋 Step 2: 更新索引 (prose-index-manager)

**执行要求**：
- 必须更新内容：
  - 总计篇数
  - 已完成篇数
  - 最后更新日期（当前日期）
  - 片段列表（包含文件位置、行号范围、主题、特色）

**📂 输出文件**：
- `d:\workspace\{级别}美文\美文收集索引.md`
- 格式必须与标准流程完全对齐

**必须更新字段**：
```markdown
## 📊 收录统计

- **总计篇数**：{最新数量}
- **片段收集**：{最新数量}个片段
- **完整文章**：{最新数量}篇
- **双向链接**：✅ 已建立
- **最后更新**：{YYYY-MM-DD}
```

---

### 🔍 Step 3: 搜索原文 (prose-fulltext-hunter)

**执行要求**：
- **强制执行三步验证法**：
  1. 搜索完整版关键词
  2. **必须**搜索"课文与原文区别"
  3. 版本验证（字数、结构、特征）
- **铁律**：宁可不收录，也不能收录删减版
- **高风险作品必查**：丰子恺《白鹅》、郑振铎《燕子》等

**三步验证法**：
```yaml
第一步: 搜索完整版关键词
  - "{作品名}" "{作者}" 原文完整版
  - "{作品名}" "{作者}" 未删减版
  - "{作品名}" "{作者}" 文集版

第二步: 版本对比（最关键）
  - "{作品名}" 课文与原文区别
  - "{作品名}" 教材删改
  - "{作品名}" 课文版 vs 原文版

第三步: 版本验证
  - 字数验证
  - 结构验证
  - 特征验证
```

**🚨 核查清单（每篇必须完成）**：
- [ ] 已搜索"课文与原文区别"
- [ ] 已确认是否存在删改
- [ ] 已对比字数
- [ ] 确认为完整版 ✅

---

### 📋 Step 4: 格式化文章 (prose-article-formatter)

**执行要求**：
- **必须先读取模版**：对应级别的美文赏析通用模版
- **100%符合模版要求**：
  - 完整的YAML元数据（title、source、created、tags）
  - H1标题（含Emoji+完整版标注）
  - H3作者署名
  - 原文正文（开头2-3个Emoji，H3+Emoji分段，正文穿插Emoji）
  - 所有核心板块（根据级别要求）
  - 所有必需Emoji

**文件命名**：`{序号}.{作品名}-{作者}.md`

**模版使用规范**：
- **占位符**：`{{...}}`格式的内容必须替换为实际内容并保留在最终文章中
- **操作指南**：以`**`开头的规则说明**不得出现在最终正文中**

**正文Emoji穿插规范**：
- 开头Emoji组合：在正文开始前添加2-3个主题相关Emoji
- 段落内穿插：在关键句子、情感表达、事物描述后添加Emoji
- **密度统一**：每2-3句话添加1个Emoji（所有级别统一标准）
- **句子定义**：以任何标点符号结尾的完整语义单元
- 贴合内容：Emoji必须与句子内容紧密相关

---

### 📋 Step 5: 去重检查 (prose-deduplicator)

**执行要求**：
- **总索引预检**：收录前必须查看`美文总索引.md`，确认重复情况
- **同一文件夹规则**：绝对不允许同一文件夹内出现相同文章
- **跨文件夹数量限制**：同一作品（作品名+作者）在全库中不得超过2篇，包含2篇
- **超限处理**：如发现某作品已达2篇上限，必须选择其他作品替代，不得强行收录
- **四级去重检查**：
  1. 总索引核查
  2. 文件名去重
  3. 作品去重
  4. 内容去重

**🚨 违规处理**：
- 发现超限：立即停止收录，选择替代作品
- 遗漏链接：补建所有跨文件夹重复文章的双向链接
- 索引不符：更新`美文总索引.md`，确保信息准确

---

### 🔗 Step 6: 建立链接 (prose-link-weaver)

**执行要求**：
- **链接类型**：
  1. **片段→完整文章**：在片段下方添加`[[完整文章的文件名]]`
  2. **完整文章→片段**：在文章开头添加返回片段的链接
  3. **文章→文章**：在相关作品中添加同主题文章的链接（如需要）
  4. **索引链接**：更新索引文件的所有链接
- **链接格式**：使用标准Obsidian链接格式`[[文件名]]`
- **链接规范**：仅使用文件名，不包含路径，确保可点击跳转

**📂 需要更新的文件**：
- `{级别}美文精选100篇.md`
- 所有完整文章的`.md`文件
- `美文收集索引.md`

---

### ✅ Step 7: 质量检查与修复

**执行要求**：
- **格式化完整性检查**：验证所有文章是否符合模版要求
- **正文Emoji穿插专项检查**：确保Emoji使用符合标准
- **板块完整性检查**：确认所有必需板块都已创建
- **链接有效性验证**：测试所有链接是否可正常跳转
- **问题修复**：修复任何不符合标准的内容

**✅ 质量检查清单（每篇必须完成）**：
```markdown
## 质量检查清单

### 必需元素(不得缺项)
- [ ] YAML 元数据(title, source, created, tags)
- [ ] H1 标题(含Emoji +完整版标注)
- [ ] H3 作者署名
- [ ] 📖 原文正文(分段+Emoji)
- [ ] 🌟 美文赏析(3个子板块)
- [ ] 🎭 朗读指导(2个子板块)
- [ ] 🌈 小练笔(步骤式)
- [ ] 📚 知识小卡片

### 正文Emoji穿插专项检查
- [ ] 开头添加了2-3个主题相关Emoji组合
- [ ] 每个段落都有Emoji穿插
- [ ] Emoji密度统一(每2-3句话1个)
- [ ] **句子定义**：**任何标点符号**分隔的文本均视为独立的"一句话"
- [ ] 所有Emoji都与句子内容相关
- [ ] 没有影响阅读流畅性
```

### 🚨 Step 8: 总索引更新 (prose-master-index-updater) - **铁律步骤**

**🚨 铁律声明**：此步骤为**强制性要求**，任何收录工作如果没有完成总索引更新，都视为未完成。无论收录多少篇文章，都必须执行此步骤！

**执行要求**：
- **🚨 强制更新总索引**：收录完成后必须更新`美文总索引.md`
- **同步各库信息**：确保总索引反映各文件夹的最新状态
- **维护统计数据**：更新总体统计、分库统计、重复关系等信息
- **检索功能维护**：维护按作者、题材、首字母的检索功能
- **质量保证验证**：确保所有新增链接可正常跳转

**✅ 总索引更新检查清单**：
