# 核心概念

---

## 1. Workspace

Agent 的"家"，持久化存储所有配置、记忆、会话和项目。

```
~/.myagent/
├── soul.md              # Agent 个性/人格定义
├── user.md              # 用户画像
├── identity.md           # Agent 身份
├── memory/              # 记忆文件
│   ├── user/            # 用户偏好
│   ├── feedback/         # 用户反馈
│   ├── project/          # 项目上下文
│   └── reference/        # 外部参考资料
├── skills/              # Agent 技能
├── projects/             # 项目工作区
├── sessions/             # 会话历史
├── logs/                 # 日志
└── workspace/           # Agent 可访问的根目录
```

**记忆文件格式**：Markdown + YAML Frontmatter

```markdown
---
type: user
tags: [preference, coding-style]
created: 2026-04-23
updated: 2026-04-23
---

# User Preference

用户偏好简洁回复，不要冗余总结。
```

## 2. Memory System（记忆系统）

两层设计：

- **自动提取**（Auto-extraction）— 从对话中自动抽取重要信息写入记忆文件
- **RAG 检索**（RAG retrieval）— 回答问题前自动检索相关记忆

四种记忆类型：

| 类型 | 说明 |
|------|------|
| `user` | 用户偏好、习惯、语言风格 |
| `feedback` | 用户对 Agent 输出的反馈 |
| `project` | 项目上下文、技术栈、决策记录 |
| `reference` | 外部参考资料、文档链接 |

## 3. Task Engine（任务引擎）

**核心循环**：

```
pending → planning → planned → executing → executed → reviewing → done
                    ↑__________________________________________|
                    （评审不通过则重新执行）
```

状态说明：

| 状态 | 说明 |
|------|------|
| `pending` | 等待启动 |
| `planning` | Planner 正在分解任务 |
| `planned` | 计划已生成，等待审批 |
| `executing` | 执行中（Executor/团队工作）|
| `executed` | 执行完成，等待评审 |
| `reviewing` | Reviewer 评审中 |
| `done` | 通过评审，任务完成 |
| `failed` | 执行失败 |
| `cancelled` | 用户取消 |

**快照机制**：每个阶段自动保存快照，支持重试（retry）和恢复（restore）。

---

## 4. Agent Teams（多 Agent 协作）

内置四种默认角色：

| 角色 | Agent ID | 职责 |
|------|----------|------|
| Planner | `plan` | 分解任务，制定执行计划 |
| Explorer | `explore` | 调研代码库，查找相关信息 |
| Executor | `worker` | 执行具体开发任务 |
| Reviewer | `reviewer` | 代码评审，质量把关 |

典型工作流：

```
用户 → Planner(分解) → Explorer(调研) → Executor(执行) → Reviewer(评审)
                                                    ↓ 不通过
                                              Executor(重试)
```

---

## 5. Context Compression（上下文压缩）

自动压缩长对话，保持关键决策点：

- 动态阈值：超过 N 轮自动压缩
- 保留关键决策点（用户明确的重要结论）
- 语义保留：压缩后语义不走形

---

## 6. Session（会话）

三层隔离：

| 维度 | 说明 |
|------|------|
| per-user | 每个用户独立会话 |
| per-group | 每个群组独立会话 |
| per-thread | 每个线程独立会话 |

会话持久化绑定，WebSocket 实时同步会话摘要。

---

## 7. Gateway（多通道网关）

内置适配器：

| 平台 | 适配器 | 特殊能力 |
|------|--------|---------|
| 飞书 | `feishu.py` | Webhook + WebSocket，签名验证 |
| Slack | `slack.py` | Socket Mode，Block Kit |
| Discord | `discord.py` | 斜杠命令 `/ask` `/reset` `/agent`，线程创建 |
| Telegram | `telegram.py` | 内联权限申请 |
| QQ | `qq.py` | — |
| 微信 | `weixin.py` | — |
| GitHub | `github.py` | Webhook，服务端 Secret 验证 |

所有适配器均支持 **inline permission request**（权限申请弹窗）。

---

## 8. Security（安全机制）

| 机制 | 说明 |
|------|------|
| JWT 认证 | Web UI 登录保护 |
| WebSocket Token | 会话隔离，服务端验证 |
| 文件 API 路径限制 | 只能在 cwd/workspace 内访问 |
| GitHub Webhook 签名 | 服务端 Secret 验证 |
| 消息来源信任 | 默认未知发送者需配对，推荐使用 allowlist |
| 沙箱执行 | Code Interpreter 隔离 Python 执行环境 |
