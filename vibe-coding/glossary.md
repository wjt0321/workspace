# 📚 术语表

本文档包含《Vibe Coding 从零开始》教程中使用的所有专业术语及其解释。

---

## A

### Agent 模式
**英文：** Agent Mode  
**解释：** AI 编程工具的一种工作模式，AI 可以自主执行多步骤任务，如创建文件、运行命令、修改代码等，而不仅仅是回答问题。代表工具：Cursor Composer、Windsurf Cascade。

### API (Application Programming Interface)
**中文：** 应用程序编程接口  
**解释：** 软件系统之间交互的接口，定义了请求和响应的格式。在 Vibe Coding 中常用于集成第三方服务（如 OpenAI API）。

### AST (Abstract Syntax Tree)
**中文：** 抽象语法树  
**解释：** 源代码的结构化表示，AI 编程工具使用 AST 来理解和操作代码结构。

---

## B

### Backend
**中文：** 后端  
**解释：** 应用程序的服务器端部分，负责数据处理、业务逻辑和与数据库交互。

### Boilerplate
**中文：** 样板代码  
**解释：** 重复性高、模式固定的代码片段。Vibe Coding 特别适合生成这类代码。

### Branch
**中文：** 分支  
**解释：** Git 版本控制中的独立开发线，用于并行开发和功能隔离。

---

## C

### CI/CD (Continuous Integration/Continuous Deployment)
**中文：** 持续集成/持续部署  
**解释：** 自动化构建、测试和部署的流程。Vibe Coding 可以辅助编写 CI/CD 配置文件。

### CLI (Command Line Interface)
**中文：** 命令行界面  
**解释：** 通过文本命令与计算机交互的界面。许多 Vibe Coding 工具提供 CLI 版本。

### Context
**中文：** 上下文  
**解释：** AI 模型能够看到的代码和对话历史。有效的上下文管理是 Vibe Coding 的核心技能。

### Context Window
**中文：** 上下文窗口  
**解释：** AI 模型一次能处理的最大 token 数量。超出部分会被截断或忽略。

### CRUD
**中文：** 增删改查  
**解释：** Create（创建）、Read（读取）、Update（更新）、Delete（删除）的缩写，是大多数应用程序的基础操作。

---

## D

### DDD (Domain-Driven Design)
**中文：** 领域驱动设计  
**解释：** 一种软件设计方法，强调以业务领域为核心进行架构设计。

### Debug
**中文：** 调试  
**解释：** 查找和修复代码中错误的过程。Vibe Coding 工具可以辅助分析错误信息。

### Dependency
**中文：** 依赖  
**解释：** 项目运行所需的外部库或模块。Vibe Coding 可以自动管理依赖安装。

### DevOps
**中文：** 开发运维一体化  
**解释：** 软件开发人员和 IT 运维人员之间的协作文化和实践。

---

## E

### ESLint
**中文：** -  
**解释：** JavaScript/TypeScript 的静态代码分析工具，用于检查代码风格和潜在问题。

---

## F

### Frontend
**中文：** 前端  
**解释：** 应用程序的用户界面部分，运行在浏览器或客户端设备上。

### Full-stack
**中文：** 全栈  
**解释：** 同时涉及前端和后端开发的完整技术栈。

---

## G

### Git
**中文：** -  
**解释：** 分布式版本控制系统，用于跟踪代码变更和协作开发。

---

## H

### Hook (React)
**中文：** 钩子  
**解释：** React 16.8 引入的特性，允许在函数组件中使用状态和其他 React 特性。

### HTTP
**中文：** 超文本传输协议  
**解释：** Web 浏览器和服务器之间通信的基础协议。

---

## I

### IDE (Integrated Development Environment)
**中文：** 集成开发环境  
**解释：** 提供代码编辑、调试、构建等功能的完整开发工具，如 VS Code、Cursor、Trae。

### Inline Editing
**中文：** 内联编辑  
**解释：** AI 工具直接在代码编辑器中修改代码的功能，如 Cursor 的 Ctrl+K。

### Iterative Development
**中文：** 迭代开发  
**解释：** 将开发过程分解为多个小周期，每个周期都产生可工作的软件增量。

---

## J

### JWT (JSON Web Token)
**中文：** JSON 网络令牌  
**解释：** 用于安全传输信息的开放标准，常用于用户认证。

---

## L

### LLM (Large Language Model)
**中文：** 大语言模型  
**解释：** 基于大量文本数据训练的人工智能模型，如 GPT-4、Claude，是 Vibe Coding 的核心技术基础。

### Linting
**中文：** 代码检查  
**解释：** 自动检查代码中潜在错误和风格问题的过程。

---

## M

### Middleware
**中文：** 中间件  
**解释：** 处理 HTTP 请求和响应的函数，常用于认证、日志记录等横切关注点。

### Mock
**中文：** 模拟  
**解释：** 在测试中创建模拟对象或数据，用于隔离测试单元。

---

## N

### NLP (Natural Language Processing)
**中文：** 自然语言处理  
**解释：** 使计算机能够理解和处理人类语言的技术，是 AI 编程助手的基础能力。

### Node.js
**中文：** -  
**解释：** 基于 Chrome V8 引擎的 JavaScript 运行时，用于构建服务器端应用。

---

## O

### ORM (Object-Relational Mapping)
**中文：** 对象关系映射  
**解释：** 在面向对象编程语言和关系型数据库之间转换数据的技术，如 Prisma、TypeORM。

---

## P

### PR (Pull Request)
**中文：** 拉取请求  
**解释：** 请求将代码变更合并到主分支的机制，常用于代码审查。

### Prettier
**中文：** -  
**解释：** 代码格式化工具，自动统一代码风格。

### Prompt
**中文：** 提示词  
**解释：** 向 AI 模型输入的指令或问题。编写有效的 Prompt 是 Vibe Coding 的关键技能。

### Prompt Engineering
**中文：** 提示工程  
**解释：** 设计和优化 Prompt 以获得更好 AI 输出的技术和方法。

---

## R

### RBAC (Role-Based Access Control)
**中文：** 基于角色的访问控制  
**解释：** 根据用户角色分配权限的安全模型。

### Refactoring
**中文：** 重构  
**解释：** 在不改变外部行为的前提下改进代码内部结构的过程。

### Repository
**中文：** 仓库  
**解释：** 存储代码和版本历史的地方，可以是本地的或远程的（如 GitHub）。

### REST API
**中文：** 表述性状态转移接口  
**解释：** 基于 HTTP 协议的 API 设计风格，使用 GET、POST、PUT、DELETE 等方法。

---

## S

### Schema
**中文：** 模式/架构  
**解释：** 数据库或数据结构的设计定义，描述数据的组织方式。

### SEO (Search Engine Optimization)
**中文：** 搜索引擎优化  
**解释：** 提高网站在搜索引擎中排名的技术和策略。

### Server Component (Next.js)
**中文：** 服务器组件  
**解释：** Next.js 13+ 的特性，组件在服务器端渲染，可以减少客户端 JavaScript 体积。

### SLA (Service Level Agreement)
**中文：** 服务等级协议  
**解释：** 服务提供商和客户之间的协议，定义服务标准和责任。

### SQL Injection
**中文：** SQL 注入  
**解释：** 通过输入恶意 SQL 代码攻击数据库的安全漏洞。

### SSR (Server-Side Rendering)
**中文：** 服务器端渲染  
**解释：** 在服务器上生成 HTML 的技术，有利于 SEO 和首屏加载速度。

### State Management
**中文：** 状态管理  
**解释：** 管理应用程序数据状态的方法和工具，如 Redux、Zustand。

---

## T

### Token
**中文：** 令牌  
**解释：**
1. 身份验证令牌：用于验证用户身份的字符串（如 JWT）
2. AI 模型中的 token：文本处理的基本单位，影响上下文窗口和 API 费用

### TypeScript
**中文：** -  
**解释：** JavaScript 的超集，添加了静态类型系统，提高代码质量和可维护性。

---

## U

### UI (User Interface)
**中文：** 用户界面  
**解释：** 用户与软件交互的视觉和操作界面。

### UX (User Experience)
**中文：** 用户体验  
**解释：** 用户使用产品时的整体感受和体验。

---

## V

### Vibe Coding
**中文：** 氛围编程/直觉编程  
**解释：** 一种新兴的编程方式，开发者通过与 AI 对话来描述需求，由 AI 生成代码，开发者专注于创意和逻辑而非语法细节。

### Virtual DOM
**中文：** 虚拟 DOM  
**解释：** React 使用的编程概念，在内存中维护 DOM 的表示，通过差异算法高效更新实际 DOM。

---

## W

### WebSocket
**中文：** -  
**解释：** 在单个 TCP 连接上提供全双工通信通道的协议，适用于实时应用。

### Worker Thread
**中文：** 工作线程  
**解释：** 在后台运行脚本的技术，不阻塞主线程，用于处理 CPU 密集型任务。

---

## X

### XSS (Cross-Site Scripting)
**中文：** 跨站脚本攻击  
**解释：** 在网页中注入恶意脚本的攻击方式，是常见的 Web 安全漏洞。

---

## Z

### Zustand
**中文：** -  
**解释：** 轻量级的 React 状态管理库，API 简洁，易于使用。
