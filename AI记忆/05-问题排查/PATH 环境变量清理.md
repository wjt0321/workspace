# PATH 环境变量清理

> [!info]
> 记录 Windows PATH 审计、备份、清理与回滚边界。

## 核心原则
- 修改前必须先备份
- 区分 **用户 PATH** 与 **系统 PATH**
- 系统 PATH 需要管理员权限，不可越权修改
- 清理后要能解释“删了什么、保留了什么、为什么”

## 优先清理对象
1. 死链目录
2. 重复目录
3. 只有 DLL、没有 CLI 价值的目录
4. 明确已废弃或已迁移的旧工具目录

## 操作顺序
1. 备份当前 PATH
2. 盘点用户 PATH 与系统 PATH
3. 由保守到激进清理用户 PATH
4. 系统 PATH 若遇权限问题，立即停止并说明边界
5. 回报清理前后状态，保留回滚依据

## 已验证经验
- 用户 PATH 通常可直接修改
- 系统 PATH 若报 `WinError 5`，说明需要管理员权限
- `.exe` / `.cmd` / `.bat` 的命中顺序会影响实际入口
- PATH 问题不只是“有没有安装”，还包括“先命中了谁”

## 常见误区
### 只给建议，不真正执行
- 当用户已明确授权时，应直接完成：备份 → 清理 → 汇报

### 混淆用户 PATH 与系统 PATH
- 这是 Windows 环境排障的常见坑

### 不做备份就改
- 风险过高，回滚困难
