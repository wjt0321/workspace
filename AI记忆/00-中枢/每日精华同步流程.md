# 每日精华同步流程

> [!tip]
> 这张卡定义小黑如何把工作区当日记忆，稳定地提炼到 Obsidian 第二脑。

## 固定时机
- 默认放在：**当日日记收尾之后**
- 建议挂到：**22:35 每日清理后的固定收尾步骤**
- 先确认 `memory/YYYY-MM-DD.md` 已存在
- 再做“精华提炼”，而不是边聊边无序搬运

## 提炼目标
从当日流水里挑出：
- 会复用的规则
- 有价值的排障经验
- 稳定的用户偏好
- 可公开的方法论或流程
- 值得进入主题卡片的长期知识

## 不要同步的内容
- 天气、新闻、一次性闲聊
- 敏感原文
- 没有复用价值的碎片记录
- 仍在变化、尚未稳定的临时判断

## 建议步骤
1. 读当日日记
2. 划分“可公开主题知识”与“仅工作区保留内容”
3. 能并入旧卡片就并入，没必要每次建新卡
4. 需要新主题时再建新卡，并挂回 [[记忆中枢]] 或地图页
5. 如果涉及敏感信息，改写成 [[隐私指针写法]]
6. 如内容已稳定且适合同步，再考虑 Git 提交

## 建议节奏
- 每天控制在 **10-20 分钟**
- 以“有精华才写”为原则，不强行凑数
- 日常以更新 1-3 张卡片为宜：补规则、补互链、补指针，或新增少量高价值主题卡

## 输出标准
- 比原始日记更短
- 比 `MEMORY.md` 更主题化
- 比聊天记录更可检索
- 不泄露隐私

## 本地写入规则
- 教主已授权小黑对 `D:\\workspace` Obsidian 知识库进行本地读写和修改。
- 每日精华同步可以直接更新相关主题卡片；不必每次另行请示。
- 但不得删除文件或内容，不得执行 Git push；需要远程同步时必须另行取得明确授权。



### Push 授权修正
- 每日精华同步可以本地写入 Obsidian。
- 若需要把同步结果 push 到远程，必须先征得教主本人明确同意。
