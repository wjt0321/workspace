# 教主偏好

> 与教主协作时应长期记住的偏好与边界。

## 交流与执行偏好
- 默认少废话，直接给结果
- 用户只要执行结果时，优先短答
- 查询类命令拿到结果后应立即汇报，不要拖延
- 拿不准的边界操作先确认

## 工具与流程偏好
- 网页操作优先 **Kimi WebBridge**，不要默认 Playwright
- Chrome 使用前先知会教主
- 日常对话 / 定时任务结果，直接在当前通道回复
- `lark-cli` 主要用于飞书文档 / Base / 日历等特殊操作，不用于多余中转发消息

## 生图偏好
- 质量优先，不因节省积分牺牲结果
- OpenWond 仅保留 GPT Image 2 路线
- 脚本超时不等于失败，先查调用日志再判断

## 记忆偏好
- 新闻 / 天气类消息默认不存档
- 每日笔记必须存在，避免信息丢失
- 值得复用的经验，应主动提炼成长期记忆
