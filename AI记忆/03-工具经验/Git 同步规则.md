# Git 同步规则

> [!info]
> 记录与 `D:\\workspace` 知识库同步远程仓库相关的稳定做法。

## 基本原则
- 对知识库有稳定改动后，再做 `git add -> commit -> push`
- 不只提交自己新增目录；若用户要求整体提交，要以 `git status` 为准统一处理
- 提交前先确认隐私边界，避免把不该公开的内容带进仓库

## 推送规则
1. 先检查网络是否可直连 GitHub
2. 若直连失败，启用 V2Ray
3. 需要时为仓库临时设置：
   - `git config http.proxy http://127.0.0.1:10808`
   - `git config https.proxy http://127.0.0.1:10808`
4. push 成功后，关闭 V2Ray，并清理临时代理配置

## 提交策略
- 日常小改可按主题提交
- 若用户明确要求“整体提交”，则把工作区中相关未提交变更一起纳入
- commit message 以可读、可回溯为准
