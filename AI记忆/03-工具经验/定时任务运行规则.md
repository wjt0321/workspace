# 定时任务运行规则

> 关于 cron / 日报 / 清理 / 对账的一组稳定规则。

## 基本原则
- 新增定时任务必须同步更新 `jobs.json` 与说明表
- 改完后要跑对账，确保任务清单一致
- 例行任务结果优先直接在当前通道汇报

## 汇报规则
- 报告应说明：跑了什么、是否成功、关键结果、是否有异常
- 查询类/定时类结果不要拖延，拿到就报
- 每周备份审计类任务，即使“无变化”也要汇报

## 清理规则
- 每日结束前必须确认当日笔记存在
- 清理与删除操作要走安全围栏
- `tool_results/`、截图、缓存可以清；关键目录不能误删

## 每日清理并入 Obsidian 精华同步
- 22:35 每日清理收尾时，除运行 `tools/cleanup-daily.py` 与 `tools/check-cron-consistency.py` 外，还应确认 `memory/YYYY-MM-DD.md` 存在。
- 读取当日日记后，只筛选会复用的规则、排障经验、工具经验、稳定偏好；不要把流水账、新闻/天气、一次性结果机械同步进 Obsidian。
- 优先更新 `AI记忆/` 下已有主题卡片，能并入旧卡就并入，不强行新建。
- Obsidian 知识库允许本地写入/修改，但严禁删除文件或内容；如需 git push，只在汇报中建议，等待教主本人明确同意。

## 2026-06-07 补充：严格目录约束任务
- 用户明确限定执行目录时，除用户点名的跨目录脚本外，所有命令都必须固定 `cwd` 到指定目录。
- 不读取、不修改、不写入、不推送用户禁止触碰的路径；即使只是核查，也要遵守“只读/不可碰”边界。
- 若发现额外未提交改动但不在用户指定提交范围内，先汇报，不擅自一起提交。
- 长流程任务若任一步失败，应汇报已完成步骤、失败步骤与原因，禁止静默结束。
