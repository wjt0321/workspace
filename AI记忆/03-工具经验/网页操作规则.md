# 网页操作规则

> [!info]
> 记录网页相关任务在当前环境中的默认工具选择与执行边界。

## 默认选择
- 网页操作优先 **Kimi WebBridge**
- 不要默认用 `browser_use` / Playwright
- 登录态页面、SPA、长表格、真实站点操作优先 WebBridge

## 健康检查
- 优先检查：`C:\Users\wxb\\.kimi-webbridge\\bin\\kimi-webbridge status`
- daemon 启动不等于扩展已连接
- 若 `daemon.pid` 残留且未监听，先清理再重启

## 使用原则
1. 先确认是否需要真实浏览器登录态
2. 能复用现有浏览器 Cookie，就不要切隔离浏览器
3. 用户若只要结果，优先交付结果，不堆过程
4. 涉及对外提交、发布、发送前要确认边界

## 常见坑
### 只看 shell/curl 壳页面
- SPA 站点要继续查真实 API 或登录后后台

### daemon 已起但扩展没连
- 这是 WebBridge 常见误判

### 用 Playwright 代替真实浏览器
- 对登录态/真实站点场景往往会走偏
