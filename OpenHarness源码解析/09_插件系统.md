# 09 - 插件系统

## 核心文件

- `src/openharness/plugins/types.py` — LoadedPlugin 类型
- `src/openharness/plugins/schemas.py` — PluginManifest 清单格式
- `src/openharness/plugins/loader.py` — 插件发现与加载

---

## 1. PluginManifest 清单

```json
// plugin.json 或 .claude-plugin/plugin.json
{
    "name": "my-plugin",
    "version": "0.1.0",
    "description": "插件描述",
    "enabled_by_default": true,     // 默认启用
    "skills_dir": "skills",         // skills/*.md 目录
    "hooks_file": "hooks.json",     // hooks 配置
    "mcp_file": "mcp.json"          // MCP 服务器配置
}
```

---

## 2. LoadedPlugin 贡献物

```python
@dataclass(frozen=True)
class LoadedPlugin:
    manifest: PluginManifest
    path: Path
    enabled: bool
    skills: list[SkillDefinition]           // 贡献的 Skill
    hooks: dict[str, list]                 // 贡献的 Hooks
    mcp_servers: dict[str, McpServerConfig] // 贡献的 MCP 服务器
    commands: list[SkillDefinition]         // 贡献的命令（agent 子目录）
```

**一个插件可以贡献：**
- Skills（`skills/` 目录下的 `.md` 文件）
- Hooks（`hooks.json`）
- MCP 服务器（`mcp.json`）
- Commands（`commands/` 目录下的 `.md`）
- Agents（`agents/` 目录下的 `.md`）

---

## 3. 插件发现路径

```python
# 用户级插件（全局生效）
~/.openharness/plugins/<name>/plugin.json

# 项目级插件（仅当前项目生效）
<cwd>/.openharness/plugins/<name>/plugin.json
```

**两种 manifest 路径兼容：**
- `<plugin_dir>/plugin.json`
- `<plugin_dir>/.claude-plugin/plugin.json`

## 4. 插件加载流程

```python
def load_plugins(settings, cwd) -> list[LoadedPlugin]:
    for path in discover_plugin_paths(cwd):  # 扫描两个路径
        plugin = load_plugin(path, settings.enabled_plugins)
        if plugin:
            plugins.append(plugin)
    return plugins

def load_plugin(path, enabled_plugins) -> LoadedPlugin | None:
    manifest = PluginManifest.model_validate_json(read("plugin.json"))
    enabled = enabled_plugins.get(manifest.name, manifest.enabled_by_default)

    # 加载 skills
    skills = _load_plugin_skills(path / manifest.skills_dir)

    # 加载 commands
    commands = _load_plugin_skills(path / "commands")

    # 加载 agents
    agents = _load_plugin_skills(path / "agents")

    # 加载 hooks
    hooks = _load_plugin_hooks(path / manifest.hooks_file)

    # 加载 MCP
    mcp = _load_plugin_mcp(path / manifest.mcp_file)

    return LoadedPlugin(manifest, path, enabled, skills, hooks, mcp, commands)
```

## 5. 插件隔离的 Skill 解析

```python
def _load_plugin_skills(path: Path) -> list[SkillDefinition]:
    for skill_path in path.glob("*.md"):
        content = skill_path.read_text()
        name, description = _parse_skill_markdown(skill_path.stem, content)
        skills.append(SkillDefinition(
            name=name,
            description=description,
            content=content,
            source="plugin",  # ← 标记来源
            path=str(skill_path),
        ))
    return skills
```

**复用 `skills/loader.py` 的 `_parse_skill_markdown`**，完全兼容 SKILL.md 格式。

## 6. 插件 MCP 配置

```python
def _load_plugin_mcp(path: Path) -> dict[str, McpServerConfig]:
    raw = json.loads(path.read_text())
    parsed = McpJsonConfig.model_validate(raw)
    return parsed.mcpServers
```

插件可以自带 MCP 服务器配置，随插件启用。

## 7. SkillRegistry 如何合并插件 Skill

```python
def load_skill_registry(cwd, settings, plugins):
    registry = SkillRegistry()

    # 1. 内置 skills
    for skill in get_bundled_skills():
        registry.register(skill)

    # 2. 用户 skills (~/.openharness/skills/*.md)
    for skill in load_user_skills():
        registry.register(skill)

    # 3. 插件 skills
    for plugin in plugins:
        if plugin.enabled:
            for skill in plugin.skills:
                registry.register(skill)

    return registry
```

**插件 skill 和内置 skill 完全平等**，都在同一个 SkillRegistry 里，通过 `source="plugin"` 区分来源。

## 9. Hooks 如何合并插件 Hook

```python
def load_hook_registry(settings, plugins=None) -> HookRegistry:
    registry = HookRegistry()

    # 1. Settings 中的 hooks
    for raw_event, hooks in settings.hooks.items():
        event = HookEvent(raw_event)
        for hook in hooks:
            registry.register(event, hook)

    # 2. 插件 hooks
    for plugin in plugins or []:
        if not plugin.enabled:
            continue
        for raw_event, hooks in plugin.hooks.items():
            event = HookEvent(raw_event)
            for hook in hooks:
                registry.register(event, hook)

    return registry
```

**插件 hooks 和 settings hooks 合并到同一个 HookRegistry**，事件相同则依次执行。

## 11. 与 OpenClaw 对比

| 方面 | OpenHarness | OpenClaw |
|------|------------|---------|
| 插件目录 | `~/.openharness/plugins/` | 无对应 |
| 清单格式 | `plugin.json` 声明式 | 无 |
| 插件贡献 | Skill/Hook/MCP/Agent/Command | 无完整插件系统 |
| 禁用粒度 | per-plugin | 无 |
| 路径兼容 | `plugin.json` / `.claude-plugin/` | 无 |
| 复用 Skill 解析 | `_parse_skill_markdown` 直接复用 | 无 |

**OpenClaw 的 skills 加载是扁平的**，没有"插件"这层封装。

## 12. 核心启发

1. **清单驱动** — `plugin.json` 让插件完全自描述，无需运行代码就能知道插件贡献了什么
2. **目录约定** — `skills/`、`commands/`、`agents/`、`hooks.json`、`mcp.json` 约定清晰，插件作者零学习成本
3. **来源标记** — `SkillDefinition(source="plugin")` 让 registry 能区分 skill 来源，可用于权限控制或展示
4. **启用/禁用粒度** — `enabled_by_default` + 显式 `enabled_plugins` 字典，允许插件默认启用、用户按需关闭
