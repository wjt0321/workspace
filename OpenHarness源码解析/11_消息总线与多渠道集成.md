## 2. InboundMessage / OutboundMessage

```python
@dataclass
class InboundMessage:
    channel: str          # "feishu" | "discord" | "telegram" ...
    sender_id: str        # 用户 ID
    chat_id: str         # 会话 ID（DM 或群）
    content: str          # 消息文本
    timestamp: datetime
    media: list[str]      # 媒体文件路径列表
    metadata: dict        # 渠道特定数据

    @property
    def session_key(self) -> str:
        return self.session_key_override or f"{self.channel}:{self.chat_id}"
```

## 3. MessageBus

```python
class MessageBus:
    def __init__(self):
        self.inbound: asyncio.Queue[InboundMessage] = asyncio.Queue()
        self.outbound: asyncio.Queue[OutboundMessage] = asyncio.Queue()

    async def publish_inbound(self, msg: InboundMessage): ...
    async def consume_inbound(self) -> InboundMessage: ...

    async def publish_outbound(self, msg: OutboundMessage): ...
    async def consume_outbound(self) -> OutboundMessage: ...
```

**设计亮点：**
- 纯内存队列，不需要外部消息中间件
- 渠道和 Agent 核心完全解耦
- 两个队列（入站/出站）各自独立

## 4. ChannelBridge（核心桥接）

```python
class ChannelBridge:
    async def _loop(self):
        while self._running:
            msg = await self._bus.consume_inbound()  # 阻塞等消息
            await self._handle(msg)

    async def _handle(self, msg: InboundMessage):
        reply_parts = []
        async for event in self._engine.submit_message(msg.content):
            if isinstance(event, AssistantTextDelta):
                reply_parts.append(event.text)  # 实时收集增量文本
            elif isinstance(event, AssistantTurnComplete):
                pass  # 回合结束，用已收集的文本作为回复

        reply_text = "".join(reply_parts)
        outbound = OutboundMessage(
            channel=msg.channel,
            chat_id=msg.chat_id,
            content=reply_text,
        )
        await self._bus.publish_outbound(outbound)
```

**关键设计：**
- `submit_message()` 返回 `AsyncIterator[StreamEvent]`
- `AssistantTextDelta` 实时收集到 `reply_parts`
- 完整回合后才发一条 `OutboundMessage`

## 5. BaseChannel 基类

```python
class BaseChannel(ABC):
    name: str

    def __init__(self, config, bus: MessageBus):
        self._bus = bus

    @abstractmethod
    async def start(self): ...

    @abstractmethod
    async def stop(self): ...

    async def send(self, msg: OutboundMessage):
        raise NotImplementedError

    async def _handle_message(self, sender_id, chat_id, content, media, metadata):
        await self._bus.publish_inbound(InboundMessage(
            channel=self.name,
            sender_id=sender_id,
            chat_id=chat_id,
            content=content,
            media=media,
            metadata=metadata,
        ))
```

## 6. 飞书实现要点（FeishuChannel）

### WebSocket 长连接

```python
async def start(self):
    # 创建 Lark 客户端
    self._client = lark.Client.builder() \
        .app_id(self.config.app_id) \
        .app_secret(self.config.app_secret) \
        .build()

    # 创建事件处理器（只处理消息接收事件）
    event_handler = lark.EventDispatcherHandler.builder(...) \
        .register_p2_im_message_receive_v1(self._on_message_sync) \
        .build()

    # WebSocket 长连接（不需要公网 IP / Webhook）
    self._ws_client = lark.ws.Client(
        self.config.app_id,
        self.config.app_secret,
        event_handler=event_handler,
    )

    # 在独立线程运行（避免 asyncio 事件循环冲突）
    def run_ws():
        ws_loop = asyncio.new_event_loop()
        _lark_ws_client.loop = ws_loop  # 注入 loop
        self._ws_client.start()

    self._ws_thread = threading.Thread(target=run_ws, daemon=True)
    self._ws_thread.start()
```

### 消息去重

```python
self._processed_message_ids: OrderedDict[str, None] = OrderedDict()

def _on_message(self, data):
    message_id = message.message_id
    if message_id in self._processed_message_ids:
        return  # 跳过重复消息
    self._processed_message_ids[message_id] = None
    # 超过 1000 条时清除旧条目
    while len(self._processed_message_ids) > 1000:
        self._processed_message_ids.popitem(last=False)
```

### 消息格式智能选择

```python
@classmethod
def _detect_msg_format(cls, content: str) -> str:
    # 复杂 Markdown（代码块/表格/标题）→ interactive
    if cls._COMPLEX_MD_RE.search(content):
        return "interactive"
    # 长度 > 2000 → interactive
    if len(content) > 2000:
        return "interactive"
    # 有粗体/斜体/删除线 → interactive
    if cls._SIMPLE_MD_RE.search(content):
        return "interactive"
    # 有列表项 → interactive
    if cls._LIST_RE.search(content):
        return "interactive"
    # 有链接 → post（支持 <a> 标签）
    if cls._MD_LINK_RE.search(content):
        return "post"
    # 短文本 → text
    return "text"
```

### Markdown → 飞书格式转换

| Markdown | 飞书格式 |
|----------|---------|
| 代码块 ` ``` ` | `interactive` 卡片 |
| 表格 `\|...\|` | `interactive` 表格元素（1表/卡片） |
| 标题 `#` | `div` + `**粗体**` |
| 链接 `[text](url)` | `post` 富文本 `<a>` |
| 粗体 `**text**` | `interactive` 卡片 |
| 图片/音频/文件 | `image`/`media`/`file` 类型 |

## 7. 完整消息处理流程（飞书）

```
用户发送消息
    │
    ▼
Feishu WebSocket 接收 (lark.ws.Client)
    │
    ▼
_on_message_sync()  [同步线程回调]
    │
    ▼
asyncio.run_coroutine_threadsafe(_on_message, loop)  [调度到主循环]
    │
    ├─ 添加 reaction 表情
    ├─ 解析 content（text/post/image/file/share_card...）
    ├─ 下载媒体文件到本地
    │
    ▼
_bus.publish_inbound(InboundMessage)
    │
    ▼
ChannelBridge._handle(msg)
    │
    ├─ _engine.submit_message(content)
    │   └─ AsyncIterator[StreamEvent]
    │       ├─ AssistantTextDelta → 实时收集到 reply_parts
    │       └─ AssistantTurnComplete → 回合结束
    │
    ▼
_bus.publish_outbound(OutboundMessage)
    │
    ▼
FeishuChannel.send(msg)
    │
    ├─ 检测消息格式 (_detect_msg_format)
    ├─ 上传媒体（_upload_image_sync / _upload_file_sync）
    └─ 发送消息（text/post/interactive）
```

## 9. 与 OpenClaw 对比

| 方面 | OpenHarness | OpenClaw |
|------|------------|---------|
| 消息总线 | MessageBus（asyncio.Queue 内存队列） | 无对应结构 |
| 渠道抽象 | BaseChannel ABC | 无统一渠道基类 |
| 消息格式 | Inbound/OutboundMessage 标准化 | 各渠道直接处理 |
| 桥接器 | ChannelBridge（StreamEvent → 累积回复） | 无对应 |
| 飞书连接 | WebSocket 长连接（无需公网 IP/Webhook） | OpenClaw 用 HTTP Callback |
| 多表格 | 自动拆分（1表/卡片） | 无对应 |
| 消息去重 | OrderedDict 滑动窗口 1000 条 | 无对应 |
| 格式选择 | 智能检测 text/post/interactive | 无对应 |

## 10. 核心启发

1. **MessageBus 解耦** — 渠道层和 Agent 核心完全不依赖，渠道可以随时增删
2. **WebSocket 长连接** — 飞书不需要公网 IP、不需要配置 Webhook，比 OpenClaw 的 HTTP Callback 更优雅
3. **消息格式智能选择** — `text` / `post` / `interactive` 三级降级，保证每条消息都能发送
4. **OrderedDict 去重** — 滑动窗口 1000 条，防止内存泄漏同时保证幂等
5. **run_coroutine_threadsafe** — WebSocket 线程回调通过这个方法安全地调度到 asyncio 主循环，设计干净
