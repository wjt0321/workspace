## 三、技术难点与风险评估

### 🔴 高风险 / 高难度

| 难点 | 挑战 | 缓解措施 |
|------|------|----------|
| **AI 生成质量** | 依赖第三方 AI 模型，生成质量不可控 | Prompt 调优、用户可编辑结果 |
| **长篇连贯性** | 50+ 章节的故事逻辑一致性 | 故事骨架 + 记忆系统 |
| **Prompt 工程** | 需要精心设计各类提示词模板 | RTCO 框架是好的起点 |
| **智谱API兼容性** | Anthropic协议兼容可能有微小差异 | 预留原生Claude切换选项 |

### 🟡 中等风险 / 可控

| 难点 | 挑战 | 缓解措施 |
|------|------|----------|
| **向量数据库集成** | ChromaDB 嵌入模式 + 中文支持 | 选择中文优化的 Embedding 模型 |
| **SSE 流式传输** | 实时推送 + 错误处理 | FastAPI 原生支持 SSE |
| **3-5人并发** | 轻量级场景 | 简单信号量限制即可 |

### 🟢 低风险 / 标准实现

| 模块 | 说明 |
|------|------|
| 用户认证 | 简化本地密码认证 |
| 数据库 | SQLite + SQLAlchemy |
| 前端 UI | React + Ant Design 组件库 |
| Docker 部署 | 单容器配置 |
| AI接口 | Anthropic SDK + 智谱兼容端点 |
